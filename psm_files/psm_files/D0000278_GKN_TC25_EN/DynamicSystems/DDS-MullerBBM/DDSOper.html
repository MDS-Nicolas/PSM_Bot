Dynamic Data System (DDS) - Muller-BBM PAK MKII

Operation

The Dynamic Data System (DDS) is a Muller-BBM PAK MKII Data Acquisition System (referred to as the PAK MKII in this section). The PAK MKII is a compact, mobile, multi-channel measurement system for technological acoustic and vibration analyses. The client system measures the following:

  • Vibration
  • Speed
  • Bridge Pressure

The PAK MKII is connected to proDAS through the Real-Time network and data is displayed on a Real-Time Display Computer screen and a PAK Workstation monitor. The PAK MKII communicates with proDAS using the Modbus serial communications protocol. The channels are defined within the Generic Modbus Subsystem Drivers and the channels are configured with the Channel Editor.

proDAS scans the PAK MKII channels up to 100 Hz.

Vibration

The PAK MKII monitors vibration levels in a test engine to detect degradation or imminent failure from:

  • rotational imbalance
  • internal debris
  • turbine blade corrosion
  • bearing damage lubrication deficiencies

Vibration is measured using three CHG42S G2 modules installed on a SC42 Signal Conditioning Board. Each module has four input channels for a total of 12 vibration measurement channels. The CHG module accepts inputs from 12 piezoelectric accelerometers installed on the test engine and the adapter.

The 12 vibration channel are scanned at up to 204.8 kSa/s with 24 bit resolution by the PAK MKII Data Acquisition System.

Speed

The rotational speed of the engine is measured using an ICT42 G2 Input Module installed on a SC42 Signal Conditioning Board. The module accepts 2 inputs from the engine tachometer generators.

The two speed channels are scanned at up to 204.8 kSa/s with 24 bit resolution by the PAK MKII Data Acquisition System.

Bridge Pressure

Bridge pressure is measured using two WSB42X G2 Input Modules installed on a SC42 Signal Conditioning Board. The input module has 4 strain bridge/voltage input channels for a total of 8 strain bridge/voltage input channels which accept inputs from 8 bridge pressure transducers installed on the fuel and oil systems of the RM12 engines.

The 8 bridge pressure channels are scanned at up to 102.4 kSa/s by the PAK MKII Data Acquisition System.

Fast Fourier Transform (FFT) Analysis

The PAK MKII Data Acquisition System can perform calculations of real or complex spectra according to the variant with amplitude/phase or real/imaginary values such as FFT, APS, CPS, or PRS on all channels simultaneously using the PAK Signature software installed on the PAK Workstation.

Any waveform in the time domain can be represented by the weighted sum of pure sine waves of all frequencies. If the signal in the time domain is periodic, then its spectrum is probably dominated by a single frequency component. The Signature software represents the time domain signal by its component frequencies. FFT analysis is provided in real-time to proDAS through the Real-Time network.

Post analysis data from the PAK Data Server can be converted to data formats ASAM-ODS, ASAM Transport Format/XML, universal files, sdf, matlab binary and RPC.

Results from an FFT analysis can be printed directly to any printer attached to a computer with a Windows Operating System or saved in PDF, SVG, EMF and PNG formats, as well as directly printed to the Windows-based laser printer in the Control Room.

Calibration

The PAK MKII Data Acquisition System does not require any regular user calibration. It has a self-calibration feature which is performed by the PAK software whenever the system is powered on. For details, see the PAK MKII User Manual.

The unit must be calibrated by the manufacturer every two years. If an alternate schedule is required, it must be done in agreement with the manufacturer.
