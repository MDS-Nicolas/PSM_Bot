Dynamic Data System (DDS) - Muller-BBM PAK MKII

PAK Embedded Analyzer

The PAK Embedded Analyzer is the main communication interface between the proDAS software and the PAK MkII Mainframe. The communication is sent using the protocol Modbus over Ethernet.

The PAK Embedded Analyzer allows the PAK MkII Mainframe to operate in a standalone state. Using the PAK workstation, the system can configure the hardware channels as well as the virtual channels. The hardware channel configuration controls grounding, gains, filters, and other measurement parameters. The virtual channel configuration controls what information is sent to proDAS over the Modbus connection. The virtual channel configuration also controls what analysis are used on the channels being sent to proDAS.

proDAS sends the following commands to the PAK Embedded analyzer:

Command Description
Start Scanning The embedded analyzer begins scanning. This requires the PAK Mainframe to be powered on and not scanning.
Stop Scanning The embedded analyzer stops scanning. This requires the PAK Mainframe to be scanning.

It is recommended to have the system not recording and/or scanning. Pre-trigger Buffer is acceptable.
Restart Mainframe Restarts the PAK Mainframe. This command can be sent at any time.

It is recommended to have the system not recording and/or scanning.
Start Recording The embedded analyzer starts recording. This requires the system to be scanning and not recording.
Stop Recording The embedded analyzer stops recording. This requires the system to be scanning and recording.
Enable Pre-trigger Creates a buffer to record scanned data prior to starting a scan. This requires the system to be scanning, not recording, and with pre-trigger off.

Bridge Balancing and auto zero functions must not be in operation.
Disable Pre-trigger Disables the pre-trigger buffer. This requires the system to be scanning, not recording, and with pre-trigger on.
Bridge Balance Creates a true zero for scanning bridge channels. This requires the system to be scanning, not recording, and with pre-trigger off.
Auto Zero Creates a true zero for all other channels. This requires the system to be scanning, not recording, and with pre-trigger off.

Buffer Function of System

The system includes a 150 GB solid state drive contained within the PAK Mainframe. Measurement recordings taken within the PAK Mainframe must be transferred to a separate computer using a file transfer protocol. Details to set up the file transfer protocol are found within the PAK User Manual.

Due to the limitation of the PAK 5.8 software pre-trigger function, it is best to use the pre-trigger function within the PAK Embedded Analyzer to act as a circular buffer. In the event of a critical event, the PAK Embedded analyzer can be triggered to start a recording, which will automatically include the configured pre-trigger time. The system has been commissioned with a 2 hour (7200 second) pre-trigger. Longer pre-trigger times can be configured with consultation with the supplier. Details on setting up the pre-trigger time can be found in the PAK User Manual.

Care should be given to the start and stop order of the embedded analyzer system and the PAK software system. The PAK software cannot receive data from the embedded analyzer unless it is scanning.
