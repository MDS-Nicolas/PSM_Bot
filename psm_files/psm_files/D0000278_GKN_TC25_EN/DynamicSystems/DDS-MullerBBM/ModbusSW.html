Dynamic Data System (DDS) - Muller-BBM PAK MKII

ModBus2PAK Software

The Modbus2PAK software is the main communication interface between the proDAS software and the PAK 5.8 software. The communication is sent using the protocol Modbus over Ethernet. The communication interface between proDAS and the PAK Embedded Analyzer is located in the PAK Embedded Analyzer.

When the Modbus2PAK application is running, the Modbus communication is active between proDAS and the PAK software. “Start Server” must be pressed to enable the server and allow proDAS to control the PAK software. When the server is active and the PAK software is accepting commands from proDAS, a heartbeat signal is visible on proDAS. While the server is active, most features of the PAK software are unavailable.

To use the PAK software manually to take measurements or modify its configuration, the server must be stopped using the "Stop Server" command. For more information, see Vendor Documentation.

proDAS sends the following commands to the PAK software:

Command Description
Start Scanning The Modbus2PAK software starts scanning. This requires the Modbus2PAK software to be running and the server enabled. The system must not be scanning.
Stop Scanning The Modbus2PAK software stops scanning. This requires no recording in progress.

Any recordings in progress will be discarded.
Start Recording The Modbus2PAK software starts recording. This requires the system to be scanning.
Stop Recording The Modbus2PAK software stops recording. This requires the system to be recording.

For more details on the Modbus2PAK commands, refer to the Modbus Actions and Status Channels document.

The software also sends the project number and test number which are used by the PAK software for the D-Rec naming. The PAK software provides proDAS the number of the recording, the duration of the recording and the remaining disk space available for recording.

proDAS sends the commands to the PAK software multiple times to allow the test number and project number data to be available.

Care should be given to how the commands are being sent from proDAS to the PAK Software to prevent duplicate commands.

Care should be given to the start and stop order of the embedded analyzer system and the PAK software system. The PAK software cannot receive data from the embedded analyzer unless it is scanning.
