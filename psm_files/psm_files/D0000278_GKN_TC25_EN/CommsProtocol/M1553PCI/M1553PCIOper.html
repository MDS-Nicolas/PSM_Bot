M1553 PCI System

Operation

The M1553 PCI System provides a means of receiving and transmitting electronic information between the Electronic Engine Control (EEC) and proDAS on the PCI bus located in the Linux Host Computer, located in Equipment Rack CM1, in the Computer Room. The M1553 PCI channels are defined in the M1553 Subsystem Driver and are configured with the Channel Editor.

The M1553 PCI System includes:

  • Ballard Omnibus PCI M1553 Interface Card
  • M1553 Subsystem Driver

The level "C" M1553 modules provide the following functionality:

  • software selectable bus termination
  • configurable Remote Terminal (RT) response times

Each databus on all OmniBus M1553 modules has a 75 ohm termination resistor that can be switched across the direct-coupled terminals under software control.

When a transformer coupling is used, the direct-coupled termination resistance must be off, and external couplers and terminators are required.

The RT response time is measured from the mid-set zero crossing of the parity bit to the mid-bit zero crossing of the status word. The RT response time may be set through software using the BTI1553_RTResponseTimeSet function. For more details on the operation of the M1553 PCI card see MIL-STD-1553 Programming Manual for BTIDriver-Compliant Devices and OmniBus PCI/cPCI User's Manual Interface Card to Avionics Databuses.

Troubleshooting

See the user manuals for details on troubleshooting the M1553 PCI System.

The following special tools and equipment are required to perform troubleshooting.

  • Mi-1553B Bus reader
  • Ballard OmniBus Box and CoPilot software
