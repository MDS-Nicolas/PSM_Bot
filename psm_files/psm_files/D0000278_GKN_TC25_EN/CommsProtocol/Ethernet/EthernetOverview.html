Ethernet

Overview

Ethernet (IEEE 802.3 standard) is a link layer protocol in the TCP/IP stack which describes how networked devices communicate by transferring data using packets and frames on the same local area network (LAN).

There are Four main Ethernet communication networks in a typical test cell facility:

  • proDAS User Network that connects the computers, printers and peripheral equipment
  • proDAS Real-Time Network that connects to engine adapter temperature/pressure scanners, facility scanners and the PLC
  • Dynamic Data System (DDS) Network
  • Test facility client network that connects to client supplied equipment

User Network

The User Network connects the Data Acquisition System (proDAS), the throttle display computer, and the client network. The User Network includes administrative and maintenance computers, such as the Host Computer, Real-Time Display computers, the Management GUI Computer, the Configuration Computer and the NAS backup. Each node on the network has an Internet Protocol (IP) address that is unique. This address is a 4-byte logical address used by the IP protocol on the network to uniquely identify each node. Each data packet sent over the network contains the IP address of the destination node. Most nodes have a host table that associates symbolic names and aliases to the addresses of the nodes on the network, allowing the network device to refer to hosts by name rather than by number.

The network uses Ethernet and TCP/IP with 1000 Mb/s interface cards. Each node on the network has an Internet Protocol (IP) address used by the IP protocol on the network to uniquely identify each node. Each data packet sent over the network contains the IP address of the destination node. Most nodes have a host table that associates symbolic names and aliases to the addresses of the nodes on the network, allowing the network device to refer to hosts by name rather than by number. The Client assigns the network address, and MDS assigns the host address on the network. There is a single connection between the facility Test Cells and the client network, with a router, supplied by the client, regulating traffic flow over the connection.

Major switches in the network are:

  • a proDAS User-Network Switch
  • a proDAS Real-Time Network Switch.

These switches separate and isolate traffic into distinct segments, allowing the segments to reside on the same network, and providing better scalability when adding new segments to the network.

User Network Switch

The proDAS User Network switch typically connects to:

  • the Host Computer that is also connected to the Real-Time Network
  • the Real-Time Display (RTD) Computer(s)
  • a Management GUI Computer
  • an engine control computer
  • a printer

User Network Switch Ports

The User Network switch has multiple ports. The switch typically provides ports for the following:

  • one port for the Host Computer
  • ports for the RTD Computers, the Management GUI Computer, the NAS backup, and the Throttle Display Computer
  • a limited number of spare ports

Real-Time Network

The Real-Time Network connects the Host Computer running the proDAS Real-Time Engine (RTE) with all of the time-critical components used in collecting data such as the:

  • facility and engine scanners (temperature and pressure scanners)
  • the FCS PLC
  • the Throttle System
  • the vibration monitoring system

Real-Time Network Switch

proDAS can receive data from multiple pressure/temperature scanners mounted on the engine and in the Test Cell. To reduce communication collisions and simplify cabling, these scanners are often served by smaller switches, each with 10/100 baseT ports for communication with the scanners and 2 1000 baseT ports for communication with the Real-Time network switch. The switch is also used to connect the facility scanners, the FCS PLC, and the Host Computer in the test cell.

Dynamic Data System (DDS) Network

The DDS Network connects the DDS computers and components to the Host Computer. For more details, see DDS - Operation.
