Host Computer

Overview

The Host Computer runs a Real-Time multi-tasking Linux operating system which allows task priorities to be set in parallel, and ensures that important tasks are performed in a deterministic time frame.

The Host Computer manages the proDAS database and executes the proDAS real-time data acquisition software. The Host Computer is networked to all other proDAS computers and interfaces with the hardware subsystems to acquire and process data in real-time. The computer also makes the data available for display in real-time.

proDAS collects test data and stores the test data on the Host Computer. Steady state data (fullsets) is stored in relational tables in the Oracle database. Transient data is stored as transient or critical logs in binary files on the system hard disk. The real-time data is displayed on the Real-Time Display Computer.

Linux Operating Levels

The Host Computer uses six different operating (run) levels, each with a corresponding operating mode. The operating mode is specified in the init program.

The operating (run) levels are configured as follows:

Run Level Descriptions
0 Power-down mode.
1, S Single-user mode is used to maintain the system (install or remove software, run back-ups, and check system files). Access to the Host Computer in this mode is through the Host Computer monitor only.
2 Multi-user mode, without Network File System (NFS).
3 Full multi-user mode.
5 Multi-user mode with X11, the normal operating mode.
6 Reboot mode (brings the system down and then brings it back up again).

For detailed information on Linux operating (run) levels, see the on-line documentation that is provided with the software.
