Your proDAS

Overview

The proDAS documentation is presented in a generic manner to encompass all aspects of the software. Sections within the proDAS documentation that are marked as *Option are not part of a standard installation.

proDAS is configured based upon customer requirements. For this reason modules located in the proDAS Subsystem Configuration and Custom Applications sections of the proDAS Integrated Help are classified as optional.

  • Post Test Functions

The default Test Information Page is displayed if an engine-specific page is unavailable.

Test Information Page

The default Test Information Page is displayed by the Management GUI Computer if an engine-specific page is unavailable, or if the engine-specific Test Information Page returns an error. The page displays any available data for the current test and contains fields for entering specific information such as:

  • engine serial number
  • build number
  • engine type
  • engine standard
  • test cell
  • configuration
  • test name
  • description

Page functions include:

  • updating test information that is contained in the Test Header table of the Test Results and Sensor Calibration Database (TRSCDB)
  • resetting all fields to the original data which existed for the current test
  • reconfiguring the Real-Time Engine (RTE)

The default Test Information page cannot set channels in the Current Value Table (CVT) nor update the Customer Specific Table.

The Customer Test Information Pages are displayed below and can set channels in the CVT as well as update the Customer Specific Table.
