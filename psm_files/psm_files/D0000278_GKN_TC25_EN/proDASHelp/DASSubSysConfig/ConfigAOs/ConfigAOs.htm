Configuring Analogue Output Signals (Option*)

Overview

The General Analogue Subsystem Analogue Output (GASSAO) hardware comprises one or more E1418 VXI cards. Each VXI card can be configured with 8 or 16 ports, representing individual analogue output channels. The channels are numbered from 0 to 15.

Operation

The General Analogue Subsystem Analogue Output (GASSAO) in proDAS communicates through the Host Computer. Communication with the VXI chassis is handled via the National Instruments (NI) PCI-MXI-2 Interface Card and 777178-01 VXI-MXI-2 Extender Card.

Configuring an Analogue Output Channel

To configure an Analogue Output Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the Analogue Output Subsystem from the list of available subsystems.

  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.
  5. Set the Pinout, Chassis, Slot, Channel Number, and Output Type columns.
  6. Set the Slope column.
  7. Set the Offset column.

Configuration File Structure

Channel configuration information (e.g. hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import a GASSAO channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import a GASSAO channel, see Tab-Delimited File Structure.

Subsystems

The subsystem definitions are stored in Subsystems.xml files in the configuration directories.

A GASSAO Subsystem contains generic information, described in Subsystem Editor - Generic Subsystem Parameters - XML File Structure, and the specific attributes:

  • Cards
  • Pinouts

The specific attribute Cards defines a collection of the GASS cards implemented in the subsystem.
Each Card has the following properties:

Column Name XML Attribute Name Tab-Delimited Attribute Name Description Type Acceptable Values
Chassis Chassis CHASSIS The VXI chassis number Integer 1, 2 or 3
Slot Slot SLOT the VXI slot number in the chassis Integer 1 to 12
Card type Type CARD_TYPE The GASS card type implemented String 1418A

The specific attribute Pinouts defines a collection of the physical channels i.e. pinouts, for the card currently selected in the Subsystem. Up to 16 channels can be configured. Each Pinout has the following properties:

Column Name XML Attribute Name Tab-Delimited Attribute Name Description Type Acceptable Values
Name Name PINOUT The pinout name associated with the hardware channel being defined String Unique, non-empty
Channel Number ChannelNumber CARD_CHANNEL_NO The physical channel number on the card Integer 0 to 15
Description Description n/a Specifies the description of the pinout String Any String

Tab-Delimited File Structure

The GASSAO channel definition includes all generic channel definition columns and the specific GASSAO input parameters defined in the tab-delimited configuration file.

The rules, syntax and guidelines for creating GASSAO channels are described in the following table:

Column Name Description Syntax (data type expected in the column) Multiple Values Allowed (See Note*) Data Required (See Note**)
PINOUT PINOUT is a unique identifier given to each connection point.

Either this field or all of the following fields must be specified:
  • CHASSIS
  • SLOT
  • CARD_CHAN_NO
Either can uniquely identify the channel location. If all four fields are entered, they must match (i.e.; specify the same physical location). Each PINOUT uniquely identifies an SCP type.		 40 character alphanumeric, (case sensitive) No See Description.

SLOT SLOT identifies the slot in the VXI chassis that contains the card with the channel. Number 1 - 12 No See Description.
CARD_CHAN_NO CARD_CHAN_NO identifies the channel on the card in the slot in the VXI chassis that contains the channel. Number 1 - 16 No See Description.
CUR_VOLT Specify C for current, V for voltage. C or V No Yes
SLOPE Specify the slope of the linear equation to use to convert the Engineering Units data into output voltage or current. Number No Yes
OFFSET Specify the offset of the linear equation to use to convert the Engineering Units data into output voltage or current. Number No Yes

"Yes" indicates that more than one column value may be specified for a single channel. In this case, additional lines follow the first line of a channel definition.

An additional line is identified by a "*" in its first column (the generic NAME column). Only fields that cause a channel to be defined on more than one line (like LIMIT_TYPE when a channel has more than one limit defined) are qualified as multi-line. Fields that appear multiple times as attributes of a multi-line field (like VALUE), are not qualified as multi-line.

"No" means that the channel is restricted to a single line of the spread sheet.

"Yes" indicates that the field must contain data (it cannot be left blank or filled with an asterisk).

"No" indicates that the field is always optional.

Subsystem Editor for Analogue Output Channels

To edit the GASSAO Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.

To add a new subsystem:

  1. From the Edit menu, select Append Data Record or, in the left pane of the splitter window, click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select GASSAO in the dialogue box to select the type of subsystem.

A new line in the Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to GASSAO. The user must then assign a name and description to the subsystem.

Adding an Additional Interface Card

To add an additional Interface Card to the Subsystem:

  1. Select the Cards tab in the right hand pane of the splitter window where Cards and Pinouts are defined.
  2. From the Subsystem Editor, select Subsystem GASS/Card/Append. A new line for a new Card is added to the list of Cards in the subsystem.
  3. Define the card attributes Chassis (1, 2 or 3), Slot (1 to 12) and Card type (E1418A).

RTE Initialisation File

The GASSAO Subsystem is configured by the RTE, along with all the other subsystems, as a request from the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file would include the following GASSAO Subsystem information:

#--------------------------------------------------------------------------
# GASS Analogue Output
#--------------------------------------------------------------------------
SS GASSAO TRUE
CI card 2 10 0xF000
