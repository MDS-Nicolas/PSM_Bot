Configuring Analogue and Frequency Input Signals (Option*)

Overview

The proDAS General Analogue Subsystem VME Extended for Instrumentation (GASSVXI), GASS Analogue Input (GASSAI) and GASS Totalizer Counter (GASSTC) Subsystem communicates with the VXI chassis which provides Analogue and Frequency Input signals. In addition the GASS VXI Subsystem includes provisions for Digital I/O. Each of these subsystems is described below.

Configuring the subsystem hardware using the Subsystem Editor is identical for all ;GASS subsystems, only the selection of Cards and SCPs available for each subsystem differ.

The channel configuration for each subsystem is also identical, only the channel attributes differ between an analogue input channel and a frequency input channel.

The following subsystems are described below:

  • GASSVXI
  • GASSAI
  • GASSTC

GASS VXI Subsystem

The General Analogue Subsystem VME Extended for Instrumentation (GASSVXI) hardware comprises one or more VXI cards. Each GASS VXI card can have up to eight Signal Conditioning Plug-on (SCP) modules attached. Each SCP contains eight channels. The channels are numbered from 0 to 63 with channels 0 to 7 on one SCP, channels 8 to 15 on another SCP, and so on. The SCPs currently being used are:

  • VXI Technologies VT1501A Eight-Channel Direct Input SCP
  • VXI Technologies VT1502A Fixed-Gain Low-Pass Filter SCP
  • VXI Technologies VT1503A Eight-Channel Programmable Filter Input SCP
  • VXI Technologies VT1505A Eight-Channel Current Source SCP
  • VXI Technologies VT1508A Eight-Channel Fixed Filter and Amplifier SCP
  • VXI Technologies VT1509A Eight-Channel Fixed Filter and Amplifier SCP
  • VXI Technologies VT1512A Eight-Channel Low Pass Filter SCP
  • VXI Technologies VT1536A Isolated 8-bit Digital I/O SCP
  • VXI Technologies VT1538A Enhanced Frequency/Totalize/PWM SCP

Operation

The GASSVXI Subsystem communicates with the VXI chassis which provides Analogue and Frequency Input signals. The GASSVXI Subsystem can be replaced by the GASSAI and/or the GASSTC, but the driver allows control of both analogue and frequency modules on the same cards.

Configuring a VXI Channel

To configure a VXI Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the Analogue Input Subsystem from the list of available subsystems.
  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.
  5. Set the Pinout column or, the Chassis, Slot and Channel Number columns, for both Analogue Input, Frequency Input and Discrete Input/Output channels.

    Each pinout uniquely describes a chassis, slot and card number.

  6. If available, set the Gain column.
  7. If available, set the Range column.

    8. The range is based on the input of the sensor, not the output.

  8. Enter Calibration Sensor information in the Generic Columns.
  9. If desired, define a fixed sensor.
  10. Define a Calibration Group.
  11. If desired, set OTD On.
  12. If available, set the Input column to reflect the desired input frequency of the SCP.
  13. If a thermocouple channel is required, set the Reference RTD column to the name of the reference RTD channel.

Configuring a Digital Input/Output Channel

To configure a Digital Input/Output Channel:

  1. Set the Polarity to either Positive or Negative.
  2. Set the Debounce time of the switch.

VXI Configuration File Structure

Channel configuration information (e.g. hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import a GASSVXI channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import a GASSVXI channel, seeTab-Delimited File Structure.

Subsystems

The subsystem definitions are stored in files Subsystems.xml in the configuration directories.

A GASS VXI Subsystem contains generic information, described in Subsystem Editor - Generic Subsystem Parameters - XML File Structure, and the specific attributes:

  • Cards
  • SCPs
  • Pinouts

The specific attribute Cards defines a collection of the GASS cards implemented in the subsystem.
Each Card has the following properties:

Column Name XML Attribute Name Tab-Delimited Attribute Name Description Type Acceptable Values
Chassis Chassis CHASSIS The VXI chassis number Integer 1, 2 or 3
Slot Slot SLOT the VXI slot number in the chassis Integer 1 to 12
Card type Type CARD_TYPE The GASS card type implemented String 1413, 1415 or 1419

The specific attribute SCPs defines a collection of SCPs, implemented on the card currently selected in the subsystem. Each SCP has the following properties:

Column Name XML Attribute Name Tab-Delimited Attribute Name Description Type Acceptable Values
Location Location SCP LOCATION The location of the SCP on the card which implicitly defines the channels configured by the SCP Integer 0 to 7
SCP type Type SCP TYPE The SCP type implemented String 1501, 1502, 1503, 1505, 1508, 1509, 1512, 1536 or 1538

Since the SCPs configured are the SCPs configured for the currently selected Card in the Subsystem, it follows that each Card has its own collection of SCPs.

The specific attribute Pinouts defines a collection of the physical channels i.e. pinouts, for the card currently selected in the Subsystem. Each Pinout has the following properties:

Column Name XML Attribute Name Tab-Delimited Attribute Name Description Type Acceptable Values
Name Name PINOUT The pinout name associated with the hardware channel being defined String Unique, non-empty
Channel Number ChannelNumber CARD_CHANNEL_NO The physical channel number on the card Integer 0 to 63
Description Description n/a Specifies the description of the pinout String Any String

GASSTC Subsystem

For legacy purposes, proDAS supports configuring frequency channels independently of the analogue channels. The GASSTC Subsystem supports:

  • VXI Technologies VT1538A Enhanced Frequency/Totalize/PWM SCP

Configuring a Frequency Input Channel

To configure an Frequency Input Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the Frequency Input Subsystem from the list of available subsystems.
  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.
  5. Set the Pinout column or, the Chassis, Slot and Channel Number columns, for Frequency Input channels.

    Each pinout uniquely describes a chassis, slot and card number.

  6. Set the Measurement Type column to either Frequency, Period or Totalizer.
  7. Set the Polarity to either Positive for a rising edge trigger or Negative for a trailing edge trigger.
  8. Set the Threshold column.
  9. If the Measurement Type is Frequency, set the Minimum Frequency column to reflect the minimum expected input frequency.
  10. If the Measurement Type is Period, set the Duration column to reflect the number of cycles to average.
  11. Enter Calibration Sensor information in the Generic Columns.
  12. If desired, define a fixed sensor.
  13. Define a Calibration Group.

Subsystem Editor for VXI Channels

To edit the GASS VXI Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.

To add a new subsystem:

  1. From the Edit menu, select Append Data Record or, in the left pane of the splitter window, click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select GASSVXI in the dialogue box to select the type of subsystem.
    A new line in Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to GASSVXI. The user must then assign a name and description to the subsystem.

Adding an Additional SCP

To add an additional SCP to the Subsystem:

  1. Select the Cards tab in the right hand pane of the splitter window where Cards, SCPs and Pinouts are defined.
  2. From the Subsystem Editor, select Subsystem GASS/SCP/Append. A new line for a new SCP is added to the list of SCPs in the subsystem.
  3. Define the SCP attributes Location (0 to 7) and SCP type (1501, 1502, 1503, 1505, 1508, 1509, 1512, 1536 or 1538).

Adding an Additional Interface Card

To add an additional Interface Card to the Subsystem:

  1. Select the Cards tab in the right hand pane of the splitter window where Cards, SCPs and Pinouts are defined.
  2. From the Subsystem Editor, select Subsystem GASS/Card/Append. A new line for a new Card is added to the list of Cards in the subsystem.
  3. Define the Card attributes Chassis (1, 2 or 3), Slot (1 to 12) and Card type (1413, 1415 or 1419).
