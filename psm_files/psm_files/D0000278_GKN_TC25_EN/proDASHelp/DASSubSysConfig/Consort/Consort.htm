CONSORT Subsystem (Option*)

Overview

The proDAS CONSORT Subsystem provides proDAS with steady state data from the PW2000, PW6000 and V2500 engines via the CONSORT Engine Control Monitor. The CONSORT computer is a Pratt and Whitney supplied computer that processes the ARINC data provided by the PW2000, PW6000 and V2500 engines. proDAS communicates with the CONSORT computer using the UDP protocol over an Ethernet communication.

Configuring a CONSORT Channel

To configure a CONSORT Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the CONSORT Subsystem from the list of available subsystems.

  3. Fill the category independent columns as described in the Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.

  5. Set the Label column to the value of the ARINC label in octal.
  6. If the channel data type is Boolean, set the Bit column.
  7. Set the Lane column.
  8. If the channel data type is Float, set the Factor column.

Configuration File Structure

Channel configuration information (e.g.: hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from an XML file (a tab-delimited text file will be available in the future).

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import a CONSORT channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import a CONSORT channel, see Tab-Delimited File Structure (currently not available).

Lane Lane LANE Specifies the Lane. Combo Box "A" or "B".
Factor Factor FACTOR Specifies the factor for decimal point shift when processing SCALED INTEGER or the scale factor when processing RAW DATA. Float Any float value.

Subsystems

The subsystems are stored in Subsystems.xml files in the configuration directories.

A CONSORT Subsystem contains generic information, described in Subsystem Editor - Generic Subsystem Parameters - XML File Structure and the specific attributes:

Column Name XML Attribute Name Tab-Delimited Attribute Name Description Type Acceptable Values
Host Name HostName HOST_NAME Specifies the name of the CONSORT computer. String The name of the CONSORT computer as defined in the system host file with the appropriate IP-Address.
Steady State Period SteadyStatePeriod STEADY_STATE_PERIOD Specifies the period in seconds for the CONSORT system to average the data. Float A float between 1.0 and 60.0
Default value 5.0
Lane A PortLaneA LANE_A Specifies the port number expected for Lane "A". Integer An integer between 0 and 7
Default value 0
Lane B PortLaneB LANE_B Specifies the port number expected for Lane "B". Integer An integer between 0 and 7
Default value 5

The structure of a CONSORT Subsystem is shown in the following example:

Tab-Delimited File Structure (Future Implementation)

The CONSORT Subsystem channel definition includes all generic channel definition columns and the specific CONSORT Subsystem parameters that are defined in the tab-delimited configuration file.

The rules, syntax and guidelines for creating CONSORT Subsystem channels are described in the following table:

Column Name Description Syntax (data type expected in the column) Multiple Values Allowed (See Note*) Data Required (See Note**)
LABEL A unique identifier for the data word in octal. Octal 0 - 377 No Yes
BIT The bit number in the Status or Maintenance word as defined in the ARINC protocol. Only valid for Discrete channels. Integer 14 - 29 No See Description.
LANE A string that defines the Lane Identifier. A or B No Yes
FACTOR The factor for shifting the decimal point when processing SCALED INTEGER data or the scale factor when processing RAW DATA. Only required for Float channels. Float No See Description.

"Yes" indicates that more than one column value may be specified for a single channel. In this case, additional lines follow the first line of a channel definition.

An additional line is identified by a "*" in its first column (the generic NAME column). Only fields that cause a channel to be defined on more than one line (like LIMIT_TYPE when a channel has more than one limit defined) are qualified as multi-line. Fields that appear multiple times as attributes of a multi-line field (like VALUE), are not qualified as multi-line.

"No" means that the channel is restricted to a single value.

"Yes" indicates that the field must contain data (it cannot be left blank or filled with an asterisk).

"No" indicates that the field is optional.

Subsystem Editor for CONSORT Channels

To edit the CONSORT Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.

To add a new subsystem:

  1. From the Edit menu, select Append Data Record or, in the left pane of the splitter window, click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select CONSORT in the dialogue box to select the type of subsystem.
    A new line in Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to CONSORT. The user must then assign a name and description to the subsystem.
  3. Enter in the Host Name, Steady State Period, Lane A and Lane B values for this subsystem.

RTE Initialisation File

The CONSORT Subsystem is configured by the RTE, along with all the other subsystems, on request from the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file would include the following CONSORT Subsystem information:

# -----------------------------------------------------
CONSORT Subsystem
# -----------------------------------------------------
SS CONSORT FALSE
# the timeout in msec to receive data from the CONSORT computer
CI timeout 60
# the CONSORT computer UDP service name
CI service_name ConsortService
