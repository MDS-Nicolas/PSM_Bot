Process Write Monitoring (PWM) Subsystem (Option*)

Overview

The Process Write Monitoring (PWM) Subsystem is a software driver that communicates with the stand-alone PWM and transfers data between the PWM and the proDAS Current Value Table (CVT). proDAS and the PWM have a client/server relationship (i.e.: the proDAS PWM driver is the client and the PWM is the server). The PWM Subsystem scans up to 128 PWM channels at rates up to 100 hertz.

The PWM Subsystem is also referred to as the EMS (Engine Monitoring System).

Configuring a PWM Channel

To configure an PWM Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the PWM Subsystem from the list of available subsystems.

  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.

Configuration File Structure

Channel configuration information (e.g.: hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import an Ext channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import an Ext channel, see Tab-Delimited File Structure.

XML File Structure

Channels

The channels are stored in Channels.xml files in the configuration directories.

A Process Write Monitoring Channel only contains the generic information, described in Channel Editor - Generic Channel Parameters - XML File Structure.

The structure of an Process Write Monitoring Channel complies with the following example:

<Channels LastModificationDate="2015-06-11 11:05:55" Version="1.1">
<Channel Name="PWM1"
Description=""
Enabled="true"
LastModificationDate="2015-06-11 11:05:55"
SubsystemName="ZZPWM"
SubsystemType="PWM"
AlternateName=""
DataType="Float"
InitialValue="-999999"
Format="4.2f"
UpdateRate="1"
QualityCeiling="GOOD"
Critical="false"
EngineeringUnit="-"
DisplayMin="0"
DisplayMax="100"
DeadBand="0"
Output="false"
ReferenceChannel=""
Group=""
SignalId=""
CalSensor=""
CalGroup=""
FixedSensor=""
CustomerSensor=""
FullsetStability="false"
ContinuousLog="true"/>
</Channels>

Tab-Delimited File Structure

The Process Write Monitoring (PWM) Subsystem channel definition includes all generic channel definition columns. No subsystem specific configuration information is required.

Subsystem Editor for PWM Channels

To edit the PWM Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.


To add a new subsystem:

  1. From the Edit menu, select Append Data Record or, in the left pane of the splitter window, click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select PWM in the dialogue box to select the type of subsystem.
    A new line in Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to PWM. The user must then assign a name and description to the subsystem.

RTE Initialisation File

The PWM Subsystem is configured by the RTE, along with all the other subsystems, as a request from the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file would include the following PWM Subsystem information:

#--------------------------------------------------------------------------
# PWM Subsystem
#--------------------------------------------------------------------------
SS PWM TRUE
CI cmd_host dpwmw2
CI cmd_service pwm_ws
CI data_host daiut1
CI data_service pwm_vax
CI TRENT TRENT_700
CI BRR BR_710
