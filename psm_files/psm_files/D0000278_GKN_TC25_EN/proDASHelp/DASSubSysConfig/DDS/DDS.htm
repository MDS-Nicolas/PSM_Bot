DDS Subsystem (Option*)

Overview

The DDS Subsystem provides the interface to control the following Dynamic Data System (DDS) operations:

  • load and select the test definition file
  • command the DDS to select the next available disk set
  • place the DDS into MONITORING mode
  • place the DDS into READY mode

During proDAS configuration, or upon a request from the Management Interface, the DDS Subsystem transmits the DDS configuration file from the proDAS system to the DDS.

DDS - proDAS Communication

The DDS software driver communicates with the Dynamic Data System (DDS) through a TCP/IP Ethernet connection. The driver provides an interface between the systems for file transfer. During configuration of the DDS System, or if requested from the Management Interface, proDAS transmits a configuration file to the DDS that is based upon information in the proDAS database.

The proDAS driver notifies the operator of communication problems with the DDS, including:

  • a broken Ethernet connection
  • configuration errors in DDS
  • file transfer errors

The DDS Subsystem receives unsolicited status messages from the DDS to maintain the state of the DDS and for error tracing purposes. UEL messages include:

  • "Failed to connect to the DDS"
  • "Established connection to the DDS"
  • "Lost connection to the DDS"
  • "Recovered connection to the DDS"
  • "Failed to configure the DDS"
  • "Failed to transition the DDS into MONITORING mode"
  • "Failed to transition the DDS into READY mode"
  • "Request to reconfigure the DDS ignored, another request is already in progress"
  • "Failed to reconfigure the DDS: Not connected"
  • "Failed to reconfigure the DDS: Internal error"

For successful messages, the UEL messages include:

  • "DDS successfully transitioned into READY mode"
  • "DDS successfully transitioned into MONITORING mode"
  • "File transfer of dynamic file <file name> to the DDS succeeded"

Operation

The DDS operates as a server to the DDS Subsystem. Functions within the DDS Subsystem send requests to the DDS to perform the required operations.

The following information is read from the RTE .config file:

  • Hostname of the DDS
  • TCP Service Name for DDS communication
  • Timeout for communication
  • Test Definition File remote path for file transfers
  • Time Out for transition to Monitoring Mode

Files that contain configuration definitions for the Dynamic System are imported through the Configuration Import / Export Utility.

DDS Configuration Definitions

proDAS collects and stores engine test data by channels. Channels are grouped by subsystem. A channel definition file (sometimes referred to as a configuration file) is created for the Dynamic Data System (DDS). The difference with this file is that it contains multiple sections, none of which include the generic channel definitions.

Each section starts with a line containing only the word START, followed by a "space" (not a "tab") and the label of the section. For example, the device header section begins with START DEVICE_HEADER. The end of a section is indicated by a line containing only the word END, followed by a "space" (not a "tab") and the label of the section. For example, the device header section ends with END DEVICE_HEADER.

The file must contain all sections in the order below. Each section contains specific columns as noted in the section descriptions below:

  • Device Header(DEVICE_HEADER)
  • DAS333 Channels(DASS333)
  • Telemetry Channels (TELEMETRY)
  • DDART Channel Definition (DDART_CHANNELS)
  • DDART Feature Definition (DDART_FEATURE)
  • DDART Selection Definition (DDART_SELECTION)
  • DDART Oscilloscope Definition (DDART_SCOPE)
  • DDART Display Definition (DDART_DISPLAY)
  • Shaft Speed Definition (SHAFT_SPEED)
  • Telemetry Control (CHINCA) (CONTROL_TELEMTERY)
  • Telemetry Control (STINCA) (STINCA_CONTROL)
  • Telemetry Control (TINCA) (TINCA_CONTROL)
  • Telemetry Control (FBOSTINCA) (FBOSTINCA_CONTROL)
  • Telemetry Control (FBOCHINCA) (FBOCHINCA_CONTROL)
  • Dynamic Configuration File Additional Fields
    • DAS333 and TELEMETRY Channel Calibration Fields

DDART Feature Definition

The label for this section is DDART_FEATURE. The DDART Feature Definition Table describes a list of features; each identified by their FEATURE_NAME. These features can be associated with any of the channels, but are only used if the channel is selected for Fast Fourrier Transform (FFT) analysis (see DDART Selection Definition for details). The WARNING_LEVEL parameters either define a constant warning level (if only one point is specified) equal to the WARNING_LEVEL_LIMIT value, or a break point table consisting of a list of WARNING_LEVEL_RPM and WARNING_LEVEL_LIMIT ordered pairs. These speed-limit pairs define a staircase profile, where the speed for a particular value pair defines the lowest speed at which the limit is applicable, and the limit to be applied is constant between value pairs (i.e. not interpolated between limit values).
