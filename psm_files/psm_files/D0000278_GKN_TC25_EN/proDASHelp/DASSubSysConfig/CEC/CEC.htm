CEC Vibration Subsystem (*Option)

Overview

The CEC C-CATS 8000 is a modular, programmable vibration signal conditioning system. It accepts inputs from a variety of vibration transducers (including single ended and differential accelerometers, velocity transducers and displacement transducers).The system can also process up to four tachometer signals for tracking and filtering purposes.

The C-CATS can be configured to run as a stand-alone system using an optional Manual Operation Module (M.O.M.) or as an integrated system where each module is programmed via proDAS through a serial link.

Up to 14 single channel vibration modules can be installed in a C-CATS system. These modules combine transducer inputs and programmable filters with AC, DC and proportional outputs to provide a flexible system.

Each module can be programmed to provide a fixed or variable bandpass filter, a tracking filter or a sweep filter. The module can also be set to balance mode to provide a phase output for trim balancing applications.

Configuring a CEC Channel

To configure a CEC Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the CEC Vibration Subsystem from the list of available subsystems.
  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.
  5. Set the Pinout column.
  6. Set the appropriate columns depending on the pinout and other parameters: Input Type, Sensitivity, Unit Type, Transd. Scaling, Filter Mode, Backplane No, Filter On, Output Range, Output Mode, Output Scaling, Output Range AC, Output Range DC, Fixed Filter Table, Low Pass Freq, High Pass Freq, Bandwidth Type, Bandwidth, Min Out Freq, Max Out Freq, Var Filter Freq, Min Bandwidth, Max Bandwidth, Tach Input No, Track Filter Order, Sweep Freq Step, Sweep Freq Delay, Sweep Type, Min Sweep Freq , Max Sweep Freq, Tach Channel No, Tach Ratio and Tach Edge.

Configuration File Structure

Channel configuration information (e.g. output ranges, filter modes etc.) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import a CEC channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import a CEC channel, see Tab-Delimited File Structure.

Subsystem Editor for CEC Channels

To edit the CEC Vibration Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.

To add a new subsystem to the current configuration:

  1. Select Append Data Record from the Edit menu or click in the left pane of the splitter window and then click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select CEC in the dialogue box to select the type of subsystem.
    A new line in Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to CEC. The user must then assign a name and description to the subsystem.

RTE Initialisation File

The CEC Vibration Subsystem is configured by the RTE, along with all the other subsystems, as a request from the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file would include the following CEC Vibration Subsystem information:

#--------------------------------------------------------------------------
# CEC-8000 subsystem
#--------------------------------------------------------------------------
SS CEC False 04/26/05
#Chassis information
CI chassis 1 /dev/ttyM3 19200 1 8 N
#Time out in ms. Must be greater than 3000 ms>
CI timeout 3000
#Maximum number of retry before give up
CI retry  

#--------------------------------------------------------------------------
#
#--------------------------------------------------------------------------
