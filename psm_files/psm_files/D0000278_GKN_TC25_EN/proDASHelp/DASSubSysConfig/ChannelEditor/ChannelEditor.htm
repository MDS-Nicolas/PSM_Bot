Channel Editor

Overview

proDAS collects and stores engine test data by channels. Each channel has a unique channel name and definition that describes how the channel data (engine signal) is scanned and processed by proDAS.

All channel definitions are stored in the Configuration Database and must be referenced to a subsystem and a configuration.

The Channel Editor is the graphical user interface used to edit the channel data of the Configuration Database. The Channel Editor allows creating and editing of channel configuration data.

Channel Types

There are five types of proDAS channels:

  • float channels (variable value)
  • discrete channels (one / zero condition)
  • integer channels (truncated float channels)
  • date channels
  • time channels

Based on the selected channel type for a given channel name, the corresponding fields within the tab-delimited configuration files may require additional data.

Float Channels

A float channel monitors a measured or calculated floating point variable value. For a float channel, the units and the minimum and maximum values are included in the channel definition. The minimum and maximum values, used for displays, are applied to the ActiveX controls used to display the channel value on an operator page.

The deadband parameter is used to dampen spurious alarm signals. When no deadband is applied, a channel value which crosses an alarm threshold, and then fluctuates near that value, would cause the alarm to be triggered repeatedly for the same condition. To eliminate repeated alarms, a deadband is defined that allows the signal amplitude to fluctuate in or near the alarm zone without a new alarm being registered. The alarm is activated only when the value passes beyond the deadband threshold. The same deadband value is defined for all limits defined for a given channel.

Deadband values are used only if the channel is defined with limits.

Discrete Channels

A discrete channel monitors only On (non-zero) and Off (0) conditions. Additional parameters will need to be defined when:

  • the channel has related limits and actions
  • the channel is a calculated channel, where the equation must be defined
  • subsystem dependent information is required (e.g. digital subsystem requires a card and a channel number.)

Discrete channels can also have text attached to each state (e.g.: 0= DOOR OPEN, 1= DOOR CLOSED). This text can be displayed on the operator page using a dataview, see "Creating and Modifying Display Pages".

Integer Channels

An integer channel has all the characteristics of a Float Channel with the following exceptions:

  1. The channel value that is passed from the RTE to the RTDD is a float value, however, for the purpose of display the value will be truncated (i.e.: a value of 2.7 will be displayed as 2).
  2. When formatting an integer channel it is possible to display text strings that represent integer values (i.e.: values of 0, 1 and 2 could be displayed as zero, one, and two or as Normal, Ignition, and Crank).

The RTE is limited to 16-bit integer channel support (stored as a float value).

Alarm Limits

Limits can be applied to all channel types. A float or integer channel limit is the threshold value at which an alarm or warning is triggered.

A discrete channel has only two operating states (0 or 1). In this case a limit is applied to the state which is defined as a not normal operation. When the discrete channel enters this state the limit will trigger an alarm or warning.

Actions

Up to five actions (responses) can be defined for each limit. For example, the action for a warning could be to write a message to the Event Handler window and to start a transient log. When corrective action is taken by the operator, and the value of the channel is again within its normal range, a different message (action) could be written to the Event Handler window and the transient log could be stopped as a result.

Actions can be applied to float, integer and discrete channels.

Channel Editor

The Channel Editor uses the Configuration Database to retrieve and save the configuration information required to perform an Engine Test. The Channel Editor allows the user to define and to manipulate channel configuration information (e.g. hardware connections, calculations, limits or actions).

When selecting multiple line items of a filtered list in the spreadsheet portion of the Channel Editor (the left hand panel), clearing the filter increases the selection to include any items between the selected items that were not part of the filter.

The Delete Data Records option is a generic menu option. When deleting channels, a reference check is performed to determine if the channels are referenced to a subsystem. When deleting more than 10 channels at a time, a message informs the user that the operation may take a long time, and an option is provided to avoid the reference check. When deleting/disabling records from the Channel Editor, proDAS checks to determine if the channels are referenced elsewhere. If they are referenced elsewhere, a warning message appears with information on where they are referenced. Selecting 'Cancel' aborts the deletion/disabling process. Selecting 'OK' deletes/disables only the channels that are not referenced elsewhere. Referenced channels must be deleted/disabled from each reference before doing so in the Channel Editor.

Extended Check

The Extended Check item of the Edit menu enables a more time consuming check of the configuration. It verifies if the sensor exists in the Test Results and Sensor Calibration Database (TRSCDB).

Selecting Extended Check in the Edit menu activates an Extended Check. To perform an Extended Check, the user must navigate to the Edit menu and select Check (ensuring there is a check mark beside the Extended Check option).

Reference Check

The reference check option (not in edit menu) performs a check on references to other configuration editors (e.g. Subsystem Editor). The reference check operation is enabled/disabled through the ChannelEditor.ini file through the EnableReferenceCheck parameter. The default entry is true.

Check

Consistency checks are performed by the Configuration Server. A consistency check is performed on explicit requests of the user or on a save request.

Saving the Subsystem data is not permitted if the configuration is considered as inconsistent. The Channel Editor will display the error messages received from the Configuration Server after a consistency check in the list of messages.

The Channel Editor allows the user to generate some common types of calculated channels.

The general information, required to generate the Calculated Channels automatically, is configured using the Options property sheets.

The description of each automatically generated channel contains the text Automatically generated followed by the type (e.g.: the description of an automatically generated Absolute Pressure channel reads Automatically generated Absolute Pressure).

Convert Engineering Unit (Automatic Channel Generation)

The Automatically Generated Calculated Channels for the Engineering Unit Conversion is named using the name of the selected channel followed by a configurable Channel Name Suffix as defined in the Convert Engineering Unit Property Page.

The equation of the Automatically Generated Calculated Channel for the Engineering Unit Conversion is defined as Slope * Selected Channel Name + Offset. Where, the Slope and Offset are defined in the Engineering Unit Editor.

The Minimum, Maximum and Dead Band values of the generated Channel are going to be updated accordingly. See XML File Structure for more information.

Absolute Pressure (Automatic Channel Generation)

The Automatically Generated Calculated Channels for the Absolute Pressure Conversion is named using the name of the selected channel followed by a configurable Channel Name Suffix defined in the Absolute Property Page.

The equation of the Automatically Generated Calculated Channel for the Absolute Pressure is defined as Selected Channel Name + Reference Channel Name.

Differential Pressure (Automatic Channel Generation)

The Automatically Generated Calculated Channels for the Differential Pressure Conversion is named using the name of the selected channel followed by a configurable Channel Name Suffix defined in the Differential Property Page.

The equation of the Automatically Generated Calculated Channel for the Differential Pressure is defined as Selected Channel Name - Reference Channel Name.

Validated Average (Automatic Channel Generation)

The Automatically Generated Calculated Channels for the Validated Average is defined as a group of calculated channels derived from a list of selected channels to be averaged. The name of the automatically calculated channels are defined as follow:

  • a channel containing the Sum of all Valid Values to be averaged
  • a channel containing the Tolerance Value
  • a channel containing the Number of Valid Values
  • a channel containing the Average of all Valid Values ...
  • as many Valid Flags as selected channels

The Validated Average channel is named using the name entered in the Validated Average Dialogue.

To create the average channel the user enters a value between 0 and 1 for the minimum tolerance as part of display range and a value for the relative tolerance of the same range.

The equation of the validated average is defined as avg_t ( max (Name of Average Channel * Relative Tolerance, Minimum Tolerance), comma-separated list of selected channels).

Fullset Filtering (Automatic Channel Generation)

Fullset Filtering enables the definition of an automatically generated calculated channel to smooth an existing channel by applying filtering (allows a user to have a continuous fullset like channel defined). A suffix is defined for the automatically generated channel which is appended to the existing channel name.

The Automatically Generated Calculated Channels for the Fullset Filtering is named using the name of the selected channel followed by a configurable Channel Name Suffix defined in the Fullset Filtering Property Page.

The existing channel is smoothed by applying the following equation (Factor * Fullset Filtering Name + (1.0 - Factor) * Selected Channel Name, where the Factor is a configurable constant between 0.0 and 1.0 typically set to 0.9. The Factor default value is set in the Fullset Filtering Property Page.

Where a fullset averages the measured value at 100% over a pre-defined duration, in this instance, the average occurs in real time and allows the user to scale the averaged value and the acquired value in a rolling average resulting in a “smoothed” channel.

Site Specific Channel Attributes

The channels are stored in the Configuration Database. Each site specific channel attribute is generic (i.e. valid for all channels) regardless of their subsystem type.

Each site specific channel attribute has the following properties:

  • Name - mandatory, to be used in the COM interface of the Configuration Server and in the channels.xml file
  • Display name - optional, default: the name of the site specific channel attribute; to be used in the channel editor
  • Data type - optional (default is String; or can also be Enumeration, Boolean, Float or Engineering Unit)
  • Default value; mandatory for enumeration type only, otherwise “”, FALSE, 0.0
  • Display format; mandatory for enumeration type only, otherwise %s or %f (as used in printf)
  • Description; optional

When an ampersand is to be included in the column name of a site specific channel attribute then the naming convention must be as follows:

DisplayName Incorrect Syntax: "Transfer to P&W"

DisplayName Correct Syntax: the text "amp;" must be added after the ampersand as follows: "Transfer to P&amp;W".

Creating Tab-Delimited Files

The Channel Editor allows the user to Create Tab-Delimited Files using the Import and Export functionalities.

When importing and exporting tab-delimited files using the legacy option, only the columns in the legacy format will be included. Other columns only present in the generic format may not be included. For example, if a legacy file is exported, then imported into generic format, the extra columns in the generic format will be left empty. If a generic file is exported, then imported into legacy format, the extra columns in the generic file will not be included in the legacy format. When a generic file is imported into the legacy format, an error message informs the user which columns are not included.

Extras -- Complete TAB-delimited Export/Import

When selected, the Complete TAB-delimited Export/Import allows the user to export/import the entire configuration.

Extras -- Import TAB-delimited

A Tab-Delimited file can be imported using the menu item Import TAB-delimited file ... from the Menu Extras.

The Import TAB-delimited file dialogue allows the user to:

  • select a directory
  • select one or many file names
  • select the Configuration Level
  • enter a description
  • specify if the imported data will Append to or Replace the existing configuration

The name of the Tab-Delimited file must be the same as the subsystem that it will be associated with (i.e.: if the desired subsystem name is Calculated then the Tab-Delimited file must be named Calculated.txt).

Importing TAB-delimited files supports importing non-English characters.

An entire configuration of Tab-Delimited files can be imported by selecting the Import all TAB-delimited files... menu item.

Extras -- Export TAB-delimited

A Tab-Delimited file can be created using the menu item Export TAB-delimited file ... from the Menu Extras.

The Export TAB-delimited file dialogue allows the user to:

  • select a directory
  • enter a file name
  • select the Configuration Level
  • select one or all Subsystems
  • specify if all or part of the configuration is going to be exported

When exporting a Tab-Delimited file, there is an option to export the GASS Subsystem hardware location, which is detailed in the configserver.ini file.

All Tab-Delimited files can be exported by selecting the Export all TAB-delimited files... menu item.

If the user chooses to export the channel definitions for all of the Subsystems, the Channel Editor creates a directory for each subsystem, a File named Subsystem Name.txt and places it in a subdirectory named Subsystem Name.
