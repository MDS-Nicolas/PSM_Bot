Scanivalve HyScan Subsystem (Option*)

Overview

The Scanivalve HyScan Subsystem is a pressure measurement system. When the Scanivalve Hyscan channels measure pressures, an intermediate Hyscan PC is used to digitize the signal and pass it to the Real-Time Engine (RTE).

Configuring a HyScan Channel

To configure a Scanivalve HyScan Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the Scanivalve HyScan Subsystem from the list of available subsystems.

  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.
  5. Set the Measurement Type column to either differential or absolute.
  6. Set the HyScan column to A, B, or C, to indicate the Hyscan PC.
  7. Set the Module column to an integer between 1 and 64.
  8. Set the Port column to an integer between 1 and 64.
  9. If the Type is set to Absolute, set the High-Line Reference to the name of the reference pressure channel.
  10. Set the Factor column to the desired conversion factor for the channel.

Configuration File Structure

Channel configuration information (e.g.: hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import a HyScan channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import a HyScan channel, see Tab-Delimited File Structure.

Subsystems

The subsystems are stored in files Subsystems.xml in the configuration directories.

A Scanivalve HyScan Subsystem contains generic information, described in Subsystem Editor - Generic Subsystem Parameters - XML File Structure, and the specific attribute Number of PCs.

Tab-Delimited File Structure

The Scanivalve HyScan Subsystem channel definition includes all generic channel definition columns and the specific Scanivalve HyScan input parameters defined in the tab-delimited configuration file.

The rules, syntax and guidelines for creating Scanivalve HyScan channels are described in the following table:

Column Name Description Syntax (data type expected in the column) Multiple Values Allowed (See Note*) Data Required (See Note**)
CHAN_TYPE Specifies the type of channel being used, where DIFF represents a differential pressure reading and ABS represents an absolute pressure reading. The HYSCAN returns differential pressure readings from the ZOC modules. The Subsystem calculates the absolute reading by adding the associated reference reading to the differential pressure reading. DIFF or ABS No Yes
HYSCAN Specifies the HYSCAN unit of the channel. A, B, or C No Yes
MODULE Specifies the ZOC module number of the channel. Number 1 to 64 No Yes
PORT Specifies the port module number of the channel on the ZOC module. Number 1 to 16 No No
UNITS_FACTOR An optional conversion factor to apply to the pressure reading. Pressures are nominally returned in PSI by the HYSCAN system. If not specified, the default value will be 1.0 (no conversion). Required for ABS channel types. May be specified for DIFF channel types if they have no corresponding ABS channel. Number No See Description
ABS_REF Specifies the Hy-Line reference channel name used to convert a different channel reading into an absolute channel reading. Required for ABS channel types. Same as Generic NAME field No See Description

"Yes" indicates that more than one column value may be specified for a single channel. In this case, additional lines follow the first line of a channel definition.

An additional line is identified by a "*" in its first column (the generic NAME column). Only fields that cause a channel to be defined on more than one line (like LIMIT_TYPE when a channel has more than one limit defined) are qualified as multi-line. Fields that appear multiple times as attributes of a multi-line field (like VALUE), are not qualified as multi-line.

"No" means that the channel is restricted to a single line of the spread sheet.

"Yes" indicates that the field must contain data (it cannot be left blank or filled with an asterisk).

"No" indicates that the field is always optional.

Subsystem Editor for Scanivalve HyScan

To edit the Scanivalve HyScan Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.


To add a new subsystem:

  1. From the Edit menu, select Append Data Record or, in the left pane of the splitter window and then click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select HyScan in the dialogue box to select the type of subsystem.
    A new line in Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to HyScan. The user must then assign a name and description to the subsystem.

Adding a New HyScan PC

The number of PCs configured for the Scanivalve HyScan Subsystem can be edited in the right pane of the splitter window of the Subsystem Editor screen.

Since this parameter is defined using a textbox dialogue, the EditSpecific menu item is greyed out and the textbox is edited directly.

RTE Initialisation File

The Scanivalve HyScan Subsystem is configured by the RTE, along with all the other subsystems, as a request from the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file would include the following Scanivalve HyScan Subsystem information:

#--------------------------------------------------------------------------
# HPS Subsystem
#--------------------------------------------------------------------------
SS HPS TRUE
CI hyscan a 4 acatgpib1 1
CI hyscan b 5 acatgpib2 1
CI hyscan c 6 acatgpib3 0
CI tol_zd_lt12 0.5
CI tol_zd_gt12 2.3
CI tol_hiline 0.7
CI gpib_retry 1
CI verify_calib TRUE
CI hps_calcoef_log HPS_calcoefs.log
CI hps_zero_report HPS_zerodiff.log
CI hps_hiline_rpt HPS_highline.log
CI gpib_service gpib_tcp
CI conf_setting CALIBRATION_FILE_EXTENSION 000
CI conf_setting CALIBRATION_FILE_NAME calcoefs
CI time_calib 350
CI time_zero 100
CI time_hizero 120
CI auxiliary_purge C
CI aux_purge_zone 1 1 34.0
CI aux_purge_zone 1 2 69.0
CI aux_purge_zone 1 3 344.0
CI aux_purge_zone 4 3 137.0
