Aircraft Vibration Monitoring (AVM) Subsystem (Option*)

Overview

The Aircraft Vibration Monitoring (AVM) Subsystem monitors vibration from transducers that are mounted on the engine. Excessive vibration levels may indicate rotational imbalances, component wear or damage.

The AVM signal conditioner receives signals from vibration transducers (accelerometers) that are mounted on the engine. The signal conditioner charge amplifier input converts accelerometer outputs from a charge to a voltage.

The signal conditioner includes:

  • charge amplifier cards, each with a selectable signal input type, integrators, selectable bandpass filter frequencies and AC-to-DC converter
  • four-channel Tracking Filter cards

The charge amplifiers output to integrators that convert the acceleration (mV/g) signals to velocity (mV/ips) and displacement (mV/ins) levels. The desired velocity or displacement output from the integrators is selected through the configuration options. The integrator output is a broadband signal that includes all vibration frequencies. The selected output is divided for processing between four tracking filters and a Bandpass filter. After filtering, the output is converted to a 0-10 vdc signal that can be interpreted by proDAS.

Each Tracking Filter output is a vibration level around a frequency that shifts with engine speed. The frequency and bandwidth settings used for the filters depend upon the location of the transducer. For example, a transducer that is placed near a fan would produce vibration signals that occur at a lower frequency than signals from a compressor transducer, since the fan and compressor are rotating at different rates. A frequency setting for a fan transducer filter might therefore be 100 Hertz, and a compressor transducer filter would be set to 3000 Hertz

As the frequency of vibration events is proportional to rotational speed, the initial preset frequency of the filter must increase to properly track each event. The filters shift frequency by receiving a rotation speed signal from a probe that is installed on the engine. The probe produces a 1/rev (once per engine revolution) speed signal that is extracted by the tracking filter. The tracking filter calculates the phase angle from the time between the 1/rev signal and a vibration and peak.

The ENDEVCO 68222-3 Test Facility Amplifier System (TFAS), consisting of an ENDEVCO 4970 chassis with associated plug-in cards, is a typical Aircraft Vibration Monitoring (AVM) system. The plug-in cards may include:

  • ENDEVCO Model 26752-9 Digital Controller card
  • ENDEVCO 6728M5 Tracking Filter cards
  • ENDEVCO Model 6647 Analogue cards

Configuring AVM Channels

To configure an Aircraft Vibration Monitor Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the Aircraft Vibration Monitoring Subsystem from the list of available subsystems.

  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.
  5. Set the Number column to an integer between 1 and 99.
  6. Set the Input Type column to one of Single Ended PE, Differential PE, Velocity or Isotron.
  7. Set the Integration column to Acceleration, Velocity or Displacement.
  8. Set the Filter column to None, Low, High or band pass filter.
  9. Set the Order column as four pole or six pole.
  10. Set the Low Freq. column.
  11. Set the High Freq. column.
  12. Set the Sensitivity column.
  13. Set the Range column to represent the fullscale output in engineering units.
  14. Set the Time Constant column to an integer between 1 and 6. If left blank the default time constant is 3.

Configuring Tracking Filter Channels

To configure a Tracking Filter Channel the user has to perform to following actions:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the Aircraft Vibration Monitoring Subsystem from the list of available subsystems.
  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.
  5. Set the Number column to an integer between 1 and 99.
  6. Set the Input Type column to Tracking Filter.
  7. Set the Vibration Filter column to match the number of the vibration channel to be tracked.
  8. Set the Tacho Channel column to an integer between 1 and 4.
  9. Set the Tacho Ratio column.
  10. Set the Tacho Gain column.
  11. Set the Tracking Mode column to constant Q or constant BW.
  12. Select the Bandwidth from the drop down list of available bandwidths.

Configuration File Structure

Channel configuration information (e.g.: hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import a AVM channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import a AVM channel, see Tab-Delimited File Structure.

Subsystem Editor for AVM Channels

To edit the AVM Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.

To add a new subsystem to the current configuration:

  1. Select Append Data Record from the Edit menu or click in the left pane of the splitter window and then click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select AVM in the dialogue box to select the type of subsystem.
    A new line in Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to AVM. The user must then assign a name and description to the subsystem.

Adding an ENDEVCO Card

To add a new card to the Subsystem:

  1. Select the spreadsheet for the collection of cards in the right hand pane of the splitter window.
  2. From the Subsystem Editor, select Subsystem AVM/Append. A new line for a new Card is added to the list of Cards in the subsystem.
  3. Define the card attributes card type (either a Tracking Filter, Vibration Charge Amplifier or Tracking Filter Input Assembly), chassis (the number of the chassis being configured beginning with 1) and slot (slot number where the card is inserted in the chassis).
    Possible definitions for the above attributes are available via a spin button when editing the specific cell.

    The controller card is not configured since it is always inserted in Slot 0. Consequently, slot 0 is always reserved for the controller card and is therefore not defined here in the configuration.

    Tracking Filter cards can only be located in slots 1 and 2. If there are no Tracking Filter cards in the system, all slots can be configured as Vibration Charge Amplifieres.

    If a Tracking Filter card exists in slot 1 or slot 2, slots 3 and slot 4 must be left empty. Slots 5 to 12 may have a Vibration Charge Amp or a Tracking Filter Input Assembly card configured.

    Any slot, except slot 0 which is reserved for the controller, may be left empty.

RTE Initialisation File

The AVM Subsystem is configured by the RTE, along with all the other subsystems, as a request from the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file may include the following AVM Subsystem information:

#--------------------------------------------------------------------------
# AVM Subsystem
#--------------------------------------------------------------------------
SS DYNAMIC_AVM TRUE
CI host_name GPIB_ENET_0
CI gpib_server gpib_tcp
CI avm_info 260
CI avm_low_freq 21
CI avm_high_freq 217
CI avm_acc_sens 50
CI avm_vel_sens 500
CI avm_tf_mode 0
CI avm_connection 1
CI avm_ch 1
CI avm_ch 2
