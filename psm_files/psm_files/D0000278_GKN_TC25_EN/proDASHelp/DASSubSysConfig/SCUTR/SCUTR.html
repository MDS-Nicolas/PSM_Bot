Scanning Uniform Temperature Reference (SCUTR) Subsystem (Option*)

Overview

The SCanning Uniform Temperature Reference (SCUTR) Subsystem provides proDAS with engine temperature data measured from thermocouples and resistance temperature detectors (RTDs). The temperatures, in millivolts and ohms, respectively, are scanned at a rate of 33 hertz and transmitted to proDAS for engineering unit conversions and storage in the proDAS current value table (CVT).

RTDs that are read by the SCUTR Subsystem are only used to calculate thermocouple temperatures. RTDs used for process measurement are not measured on SCUTR.

proDAS supports the following two configurations:

  • VME Radstone Card Configuration
  • DynEng PCI SCUTR Card Configuration

VME Radstone Card Configuration

The typical VME Radstone Card Configuration based SCUTR Subsystem consists of:

  • SCUTR units (each with one SCUTR Card)
  • Radstone 68-44 SCUTR Interface Card (supporting four ports) installed in the VME chassis
  • a 24 VDC power supply for the SCUTR units

The power supply, distribution panel, and SCUTR interface terminals are mounted in the rack assemblies. The power supply provides 24 vdc through a power distribution panel to interface terminals on the SCUTR units (the power supply is not dedicated to the SCUTR units).

The SCUTR Subsystem is monitored through the standard proDAS display functions for floating point channels, including digital displays, strip charts, gauges, bar charts, and the Event log. Thermocouple channel and RTD data is transmitted in serial format over a serial cable for each SCUTR unit.

The VME chassis has a power ON/OFF switch (on the back of the chassis) and a RESET switch (on the front of the chassis).

Do not push the power OFF switch when proDAS is running as this will cause the Host Computer to crash.

Do not push the RESET switch when proDAS is running as this may cause the proDAS and Host Computer to crash.

Configuring a SCUTR Channel

To configure a SCUTR Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the SCUTR Subsystem from the list of available subsystems.

  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.

  5. Set the Serial Number column.
  6. Set the Measurement Type column.
  7. Set the Channel Number column.
  8. If the Measurement Type of the channel is "Thermocouple" or "Transient", then the corresponding Fixed Sensor field in the Channel Specific Columns must be set to the thermocouple sensor name, as defined in the Calibration database.

Configuration File Structure

Channel configuration information (e.g.: hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import a SCUTR channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import a SCUTR channel, see Tab-Delimited File Structure.

"Yes" indicates that more than one column value may be specified for a single channel. In this case, additional lines follow the first line of a channel definition.

An additional line is identified by a "*" in its first column (the generic NAME column). Only fields that cause a channel to be defined on more than one line (like LIMIT_TYPE when a channel has more than one limit defined) are qualified as multi-line. Fields that appear multiple times as attributes of a multi-line field (like VALUE), are not qualified as multi-line.

"No" means that the channel is restricted to a single line of the spread sheet.

"Yes" indicates that the field must contain data (it cannot be left blank or filled with an asterisk).

Subsystem Editor for SCUTR Channels

To edit the SCUTR Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.


To add a new subsystem to the current configuration:

  1. Select Append Data Record from the Edit menu or click in the left pane of the splitter window and then click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select SCUTR in the dialogue box to select the type of subsystem.
    A new line in the Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to SCUTR. The user must then assign a name and description to the subsystem.

VME SCUTR Interface Card

The Radstone 68-44 SCUTR Interface Card is located in the VME chassis. The card converts SCUTR data to TTL levels and transmits the data with VME protocols to the Host Computer.

PCI SCUTR Interface Card

The DynEng PCI SCUTR Interface Card is installed in the Host Computer. The card converts SCUTR data to TTL levels and transmits the data to the Host Computer.

SCUTR Unit

Each SCUTR unit serial number is transmitted within the data stream to the card. The card identifies the unit from the serial number. The SCUTR Subsystem applies engineering unit conversions to the data. Conversion is performed at the proDAS level to allow SCUTRs to use various types of thermocouples.

A serial communication controller within the Radstone SCUTR interface card / PCI SCUTR Interface Card unpacks and decodes the Manchester data. The card transition module converts RS485 levels to TTL levels. The engineering unit converted temperatures are then transmitted to proDAS at a rate of 33 scans-per-second for a maximum of 60 thermocouple channels. During the first proDAS scan, thermocouple temperature data from the SCUTR units is verified for required starting temperature limits.

The front panel of the card provides:

  • four SCM connectors
  • two serial connectors SC1 and SC2
    • SC1 connector for normal serial communication with a terminal used for debug purposes (using PLUM) or the Boot Monitor
    • SC1 and SC2 are configured as DTE (9.6 kbaud, 8-bits/character, 1 stop bit, parity disabled)
  • reset/abort toggle switch for internal reset (RST) or abort interrupt (ABT). The switch is enabled following a power-up or a reset

    Do not use the RESET/ABORT switch when proDAS is running as it may cause the Host computer to reboot.

  • bicolour RUN LED to indicate operation (green), stop (red) or reset (red and green)
  • user-programmable PL bicolour LED

For the location and switch settings for the Radstone SCUTR Interface card, see the applicable drawing. There are no switch settings for the PCI SCUTR Interface card.

SCUTR Box

The SCUTR box (unit) generates temperature data for proDAS by comparing voltages from external thermocouples to reference voltages from two internal RTDs. Two 30-channel UTRs convert up to 60 temperature readings to voltages that are sent to the SCUTR card and transmitted to the Radstone SCUTR Interface card / PCI SCUTR Interface Card. The Radstone card / PCI SCUTR Card transmits the data to proDAS.

Eight calibration signals (two signals for every 15 thermocouple channels) are generated by the SCUTR card. The card converts the analogue thermocouple voltages, RTD voltages and the calibration voltages to digital values. The values are packaged into a serial string and are Manchester-encoded and transmitted from the SCUTR unit at RS485 levels. The SCUTR unit samples the thermocouple channels at a rate of 33 scans-per-second and transmits the serial data at a rate of 192 kHz to the Radstone SCUTR interface card / PCI SCUTR Interface Card.

Adding an Additional SCUTR Box

To add an additional SCUTR Box to the Subsystem:

  1. Select the Boxes spreadsheet in the right hand pane of the splitter window.
  2. From the Subsystem Editor, select Subsystem SCUTR/Append. A new line for a new Box is added to the list of Boxes in the subsystem.
  3. Define the Box attributes Serial Number (the serial number of the box in the form SCUTR_XXX where XXX defines the serial number of the box), and Transient (a flag defining whether transient thermocouples are supported for this box. The values are either true or false depending on whether the box is checked or not).

RTE Initialisation File

The SCUTR Subsystem is configured by the RTE, along with all the other subsystems, as a request from the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file would include the following SCUTR Subsystem information:

#--------------------------------------------------------------------------
# SCUTR Subsystem
#--------------------------------------------------------------------------
SS SCUTR TRUE
CI card 1 0x15000000
