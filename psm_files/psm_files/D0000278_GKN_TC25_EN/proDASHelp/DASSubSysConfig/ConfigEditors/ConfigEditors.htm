Configuration Editors

Overview

The configuration editors are the user interfaces that allow the operator to setup and control proDAS configurations. All of the configuration editors share some common features. An overview of these features is provided in this module.

The following editors are available to the operator:

  • Channel Editor - used to create and edit channel definitions (the definitions include the channel name, limits, and associated subsystems)
  • Subsystem Editor - used to create and edit subsystem definitions (the definitions include the subsystem name, type, and specific hardware data)
  • Engineering Unit Editor - used to create and edit engineering unit definitions (the definitions include the unit name, group, and its relationship to the group primary unit)
  • Polynomial Editor - used to create and edit polynomial definitions (the definitions include the polynomial name and its associated coefficients)
  • Breakpoint Table Editor - used to create and edit breakpoint table definitions (the definitions include the breakpoint table name, engineering units, and table entries)
  • Transient Log Definition Editor - used to create and edit transient log definitions (the definitions include the transient log name, scan rate, and associated channels)
  • Macro Editor - used to create and edit macro definitions (the definitions include the macro name and text)
  • User Functions - used to create and edit user function definitions (the definitions include the user function name and text)
  • Text Output Pages - used to create and edit the text output page definitions (the definitions include the name, size, and layout of the output page)

User Security

A configuration editor can be started in one of two ways:

  • via the Management GUI Control Panel where the username and password are passed automatically
  • as an independent editor where the username and password are not passed automatically

To start a configuration editor from the Management GUI Control Panel:

  1. From the Tools menu, select the applicable editor. The selected editor is started and a Processing... dialogue box is displayed to prevent the user from doing anything until the configuration data is loaded. Once loaded, the main screen for the editor is displayed.

    The operator is automatically logged into the editor with the same user name and privileges as entered for the Management GUI.

To start a configuration editor as an independent editor:

  1. Double-click the executable file, from your directory structure, for the applicable editor.
    The Login window is displayed.
  2. Enter a valid Username and Password.
    The user name and password must be a minimum of three characters and five characters long, respectively, to enable the Login button.
  3. Select Login.
    The selected editor is started and a Processing... dialogue box is displayed to prevent the user from doing anything until the configuration data is loaded. Once loaded, the main screen for the editor is displayed.
  4. To close the Login window, select Cancel.

Graphical Elements

The screen comprises the following graphical elements:

  • pull-down menus
  • buttons
  • navigation bar if the editor is configuration dependent

  • splitter windows:
    • list of data records as a spreadsheet on the left
    • detailed information of the specific data record selected on the left f the data record has specific data (i.e.: data that is not common for all data records of a specific data category)
  • window for the list of messages

The generic functionalities are accessed by the following menus and columns:

  • File
  • Edit
  • View
  • Configuration
  • Help
  • Index, Level, Enabled, Active, LastModDate, Name and Description columns

Find... (Ctrl+F) Displays the Find window which allows the user to search for a string. Another find criteria is a regular expression. For this, the user selects the Regular expression check box and enters a regular expression (e.g.: ^m finds only words beginning with m, but not words beginning with M). For more information about regular expressions, see Regular Expressions.

Using the check boxes, the user can define the search conditions (Match whole word only, Match case). Furthermore, the user can select the search range (All, Selection) and search direction (Up, Down).
  1. To start the search and/or search for the next position, select Find Next.
  2. Select Mark All, all text passages matching the regular expression are coloured.
  3. To remove all marks, select Remove Marks.
  4. To close the Find window, select Cancel.
Replace... (Ctrl+H) Displays the Replace window, which allows the user to search for a string or regular expression and replace it with any other string.
Using the check boxes, the user can define the search conditions (Match whole word only, Match case or Regular Expression). Furthermore, the user can select the search range (All, Selection) and search direction (Up, Down).
  1. To start the search and/or search for the next position, select Find Next.
    The cursor marks the position if the find text exists and the Replace dialogue is set to the foreground. Otherwise a message box is displayed, that the text was not found.
  2. To replace the text at the current cursor position with the Replace with text , select Replace.
  3. To replace all text passages matching the find text without further inquiry, select Replace All.
  4. To remove all marks, select Remove Marks.
  5. To close the Replace window, select Cancel.

Select the appropriate cell with the mouse, then select the simple Filter button . To remove the filter, select the simple Filter button again.

Undelete Data Record Revokes the last deletion of a data record and appends it at the end of the list of data records. This menu item is only available if Delete Data Record has been used before.

Regular Expressions

A regular expression is a series of characters that define a pattern. The pattern is then compared to a target string to see whether there are any matches of the pattern in the target string. Often, the characters in a pattern will simply match themselves in the target string, such as looking for all occurrences of the pattern the in "the quick brown fox chased the lazy dog".

Special characters, called meta-characters, can also be used to indicate character positioning, grouping, and repetition (e.g.: the use of the asterisk (*) as a wildcard for matching any character when doing a directory search in DOS is an example of a regular expression meta-character).

It is also possible to search for pattern sequences using regular expressions (e.g.: the regular expression [a-c] will match any a, b, or c in the target string.

Regular expressions include several special meta-character and sequences to allow more complex pattern matching. Negative character, digit sets, and ranges can also be expressed (e.g.: [^abc] will match any characters not enclosed in brackets). An expression (e.g.: [^m-q]) can also be written for a negative range.

Symbol Description Usage and Examples
* used as a wild-card character t*e will match the, trench, stringer
^ indicates the beginning of a string ^i will match is, into, identity but not mi
$ indicates the end of a string $i will match MI but not is, into, identity
+ used to match the preceding character one or more times fo+ will match foo but not f
? used to match the preceding character zero times or just one time a?ve? will match the ve in never
. used to match any single character except the new line character a.b will match aab and a3b, but not ab
| used for alternative matching a|b will match either a or b
{ } parentheses { } are used to express (enclose) the number of matches against the target string

the number of matches may be entered as an exact number (e.g.: {n}) or as a minimum and maximum number of matches (e.g.: {n,m}) 		e{2} will match feed but not fed
o{1,3} will match all the o's in food or sod, but will only match the first three o's in soooooie)
[ ] square brackets [ ] are used to express (enclose) character sets, digit sets, and ranges [abcd] will match any of the enclosed characters in the target string
[a-z] will match the whole lower-case alphabet
[A-Z] will match the whole upper-case alphabet
[a-zA-Z] will match both the whole lower- and upper-case alphabet
[0-9] will match digits in a string
