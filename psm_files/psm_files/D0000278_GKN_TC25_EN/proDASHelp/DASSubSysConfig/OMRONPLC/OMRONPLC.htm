OMRON PLC Subsystem (Option*)

Overview

The OMRON PLC Subsystem communicates with the OMRON C200H PLC System through a RS232 serial connection to get the digital input channels and to set the digital output channels. The OMRON PLC Subsystem scans all its channels at the maximum scan rate of the configured channels. The OMRON PLC Subsystem output channels are transmitted to the PLC when they change state.

The scan rate is limited by the speed of the serial connection.

Configuring an OMRON PLC Channel

To configure an OMRON PLC Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the OMRON PLC Subsystem from the list of available subsystems.

  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.
  5. Set the Address column.

Configuration File Structure

Channel configuration information (e.g. hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import an OPLC channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import an OPLC channel, see Tab-Delimited File Structure.

The OMRON PLC channel configuration contains both the generic channel definition columns and the specific OMRON PLC input parameters defined in the tab-delimited configuration file.

"Yes" indicates that more than one column value may be specified for a single channel. In this case, additional lines follow the first line of a channel definition.

An additional line is identified by a "*" in its first column (the generic NAME column). Only fields that cause a channel to be defined on more than one line (like LIMIT_TYPE when a channel has more than one limit defined) are qualified as multi-line. Fields that appear multiple times as attributes of a multi-line field (like VALUE), are not qualified as multi-line.

"No" means that the channel is restricted to a single line of the spread sheet.

"Yes" indicates that the field must contain data (it cannot be left blank or filled with an asterisk).

"No" indicates that the field is always optional.

Subsystem Editor for OMRON PLC Channels

To edit the OMRON PLC Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.

To add a new subsystem:

  1. From the Edit menu, select Append Data Record or, in the left pane of the splitter window and then click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select OPLC in the dialogue box to select the type of subsystem.
    A new line in Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to OPLC. The user must then assign a name and description to the subsystem.

Setting the Maximum Addresses

The maximum Float and Boolean Addresses required for the Subsystem can be edited in the right pane of the splitter window of the Subsystem Editor screen.

Since these addresses are defined using textbox dialogues, the EditSpecific menu item is greyed out and the textboxes are edited directly.

RTE Initialisation File

The OMRON PLC Subsystem is configured by the RTE, along with all the other subsystems, as a request from the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file would include the following OMRON PLC (OPLC) Subsystem information:

#--------------------------------------------------------------------------
# OMRON PLC Subsystem
#--------------------------------------------------------------------------

SS OPLC TRUE
CI oplc_dev_name /dev/ttyM1
#The time out of the device in seconds
CI oplc_time_out 5
#RS-232-C device baud rate
CI oplc_baud_rate 9600
#RS-232-C device stop bits
CI oplc_stop_bits 2
#RS-232-C device data bits
CI oplc_data_bits 7
#RS-232-C device parity
CI oplc_parity 2
#Name of the Interrupt triggering channel
CI oplc_trig_chan PLC_trig
#Name of the Acknowledgement channel
CI oplc_ack_chan PLC_ack
