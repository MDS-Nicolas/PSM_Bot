ProDAQ VXI Subsystem (Option*)

Overview

The ProDAQ VXI subsystem is a data acquisition front end system that uses PCI Express VXIbus to communicate to the host computer. The hardware is used for analogue input and output, frequency input, and discrete input and output signals, based on user installed plug-in function cards. Analogue input channels are further enhanced via external signal conditioning units for special signals such as RTD's or strain gauges. The signal conditioning units are fully integrated with the analogue input channels for configuration, calibration, and operation.

The ProDAQ VXI hardware consists of a 13 slot Smart VXIbus chassis, which can hold 12 motherboard modules and one slot-0 interface. Each motherboard can hold up to eight function cards (with each function card having 8, 16, or 48 channels). The function cards can be installed in the motherboard independently of the other installed cards.

Bustec's ProDAQ VXI hardware should only be used with other Bustec ProDAQ hardware and not interchanged with VXI hardware from other manufacturers.

Configuring a ProDAQ VXI Subsystem

To configure a ProDAQ VXI subsystem:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit Menu as described in Configuration Editors.
  2. Select the name of the ProDAQ Subsystem from the list of available subsystems.
  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the channel specific columns as described in Channel Editor.
  5. Fill the ProDAQ specific columns as required.

Configuration File Structure

Channel configuration information (e.g. hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import a ProDAQ channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import a ProDAQ channel, see Tab-Delimited File Structure.

"Yes" indicates that more than one column value may be specified for a single channel. In this case, additional lines follow the first line of a channel definition.

Only fields that cause a channel to be defined on more than one line (like LIMIT_TYPE when a channel has more than one limit defined) are qualified as multi-line. Fields that appear multiple times as attributes of a multi-line field (like VALUE), are not qualified as multi-line.

"Yes" indicates that the field must contain data (it cannot be left blank or filled with an asterisk).

"No" indicates that the field is always optional.

Subsystem Editor for ProDAQ Channels

To edit the ProDAQ Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.

To add a new subsystem:

  1. From the Edit menu, select Append Data Record or, in the left pane of the splitter window and then click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select PRODAQ in the dialogue box to select the type of subsystem.
    A new line in Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to PRODAQ. The user must then assign a name and description to the subsystem.

RTE Initialisation File

The ProDAQ VXI subsystem is configured by the RTE, along with all the other subsystems, as a request from the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file would include the following ProDAQ VXI subsystem information:

# ========================================
# PRODAQ subsystem
# ----------------------------------------
SS  PRODAQ                TRUE

And

#--------------------------------------------------------------------------
# Init and Configuration Module
#--------------------------------------------------------------------------
CI    ss_extrap_value        PRODAQ   0.1
