Subsystem Editor

Overview

The Subsystem Editor is the graphical user interface used to create and edit the subsystem data in the Configuration Database. The Subsystem Editor allows creating and editing of subsystem configuration data.

The Subsystem Editor uses the Configuration Database to retrieve and save the configuration information for a large portfolio of subsystems used for data acquisition when performing an Engine Test. The Subsystem Editor allows the user to define and to modify hardware configuration data required for the individual subsystems.

The Subsystem Editor reads the available Generic Subsystem Types from the Configuration Server initialisation file and displays the available generic subsystem types in the list of supported subsystems.

Subsystem Editor Menu Bar

The Subsystem Editor menu bar (below) assists the user in creating and modifying hardware configurations through a series of menus and menu options.

The Subsystem Editor provides the following menus:

  • File
  • Edit
  • View
  • Configuration
  • Subsystem (e.g.: GASS)
  • Help

File

The File menu of the Subsystem Editor (below) provides a number of options for handling files within the editor.

The File menu options are:

  • Save
  • Reload Config - reloads the subsystem configuration to include recent modifications
  • Import ...
  • Export ...
  • Print ...
  • Print Preview
  • Print Setup ...
  • Exit

Edit

The Edit menu of the Subsystem Editor (below) provides a number of options for editing files within the editor.

For generic Edit menu functionalities, see Configuration Editors.

The Enable Channel(s) / Disable Channel(s) option of the Edit menu allows the user to enable or disable all channels associated with the subsystem. This can be used when the user does not want to configure any channels associated with this subsystem.

The Enable flag in the subsystem definition should not be used if the user does not want to configure the subsystem. Channel definitions may be lost if this is done.

The 'Append Data Record' option of the Edit menu allows any of the subsystems included in the portfolio of subsystems to be selected and appended to the current subsystem configuration.

A new subsystem is added to the collection of subsystems and the name and description of the subsystem has to be defined. The type is set to the type of subsystem selected in the append process.

Subsystem

The subsystem menu (e.g.: TDM) of the Subsystem Editor assists the user when editing subsystem specific parameters.

If the subsystem does not have any subsystem specific parameters, the subsystem menu is greyed out and does not support any edit functions. The text no subsystem specific data is displayed in the right splitter window of the Subsystem Editor Screen.

Similarly, if the subsystem can be configured with one or more textboxes in the right splitter window, where an address, a number or simply a path is required, the Subsystem menu item is also greyed out.

If however a subsystem comprises a number of hardware modules which in turn accommodate a number of hardware channels, the right side of the splitter window of the Subsystem Editor Screen will offer a configuration possibility for the subsystem in the form of one or more spreadsheets.

In this case, the subsystem type is automatically appended to the menu bar item (i.e.: 'Subsystem' becomes 'Subsystem PBS' for a PBS Subsystem and a number of edit items are offered depending on the subsystem being edited.

On leaving the current row of the collection of subsystems in the left hand pane of the splitter window, a consistency check is performed on the new subsystem and if necessary, errors in the configuration are displayed in the message list at the bottom of the screen.

A consistency check is initiated when the 'Edit/Check' is selected in the menu bar. The same consistency check is also automatically initiated when trying to save any modifications in the Subsystem Editor. Information concerning the status of the current Subsystem configuration is displayed in the message list at the bottom of the screen.

Basic Subsystem Identification

A subsystem is uniquely identified by its name. The following rules apply for subsystem names:

  • names can consist of alpha-numeric characters and the characters "_#@"
  • the first character of the name must be an alphabetical character or the character "_" (i.e. numbers and the characters "#@" are not allowed)
  • the maximum name length is 60 characters

The generic subsystem names are defined in the configserver.ini file, where the subsystem is labelled as "NSS".

XML File Structure

The subsystems are stored in the Configuration Database locally in an XML file.

The subsystem information consists of a list of attributes. A Subsystem comprises a list of the generic subsystem attributes and if necessary the subsystem specific attributes.

The structure of a Subsystem is shown in the following example which shows the simplest subsystem type with no further subsystem specific attributes:

<Subsystems LastModificationDate="2003-07-22 09:02:25">
<Subsystem Name="myCalculated" 
Description="Calculated Subsystem" 
Enabled="true" 
LastModificationDate="2003-07-22 09:02:25" 
Id= "0"
Type= "Calculated"< br> CreationDate="2003-07-22 09:01:58"/>
</Subsystems>
