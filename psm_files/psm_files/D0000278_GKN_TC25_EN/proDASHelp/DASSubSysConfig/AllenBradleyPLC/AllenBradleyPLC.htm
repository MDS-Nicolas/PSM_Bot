Allen Bradley PLC Subsystem (Option*)

Overview

The Programmable Logic Controller (PLC), Allen Bradley PLC-5/40E Series, with discrete and analogue input/output modules, controls and monitors the engine test facility, engine support systems and some engine controls. The PLC is a stand alone system used to run the ladder logic code to control the Engine Control System (ECS) and Facility Control System (FCS). The PLC utilises an Ethernet interface module allowing TCP/IP communication. Integrating the PLC subsystem, via TCP/IP, into proDAS allows all engine controls to be managed from the main console.

Configuring a PLC Channel

To configure an Allen Bradley PLC Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the Allen Bradley PLC Subsystem from the list of available subsystems.
  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.
  5. Set the Address column.
  6. And if the channel data type is Boolean, set the Bit column.

Configuration File Structure

Channel configuration information (e.g.: hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import a PLCAB channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import a PLCAB channel, see Tab-Delimited File Structure.

XML File Structure

Channels

The channels are stored in files Channels.xml in the configuration directories.

A PLCAB Channel contains generic information, described in Channel Editor - Generic Channel Parameters - XML File Structure, and the specific attributes Address and Bit. These attributes are explained in the table below.

Column Name XML Attribute Name Tab-Delimited Attribute Name Description Type Acceptable Values
Address Address ADDRESS The physical memory address of the data item on the PLC Integer An Integer between 0 and the maximum address defined in the PLCAB subsystem
Bit Bit BIT The bit position of the data in the 16 bit word. Only applicable for Boolean channels. Integer An Integer between 0 and 15

An Allen Bradley PLC Subsystem contains generic information, described in Subsystem Editor - Generic Subsystem Parameters - XML File Structure, and the specific attributes:

Column Name XML Attribute Name Tab-Delimited Attribute Name Description Type Acceptable Values
Max Float Input Address MaxFloatInAddress FLOAT_IN_ADD Maximum address available for the float data travelling from the PLC to the RTE Integer positive
Max Float Output Address MaxFloatOutAddress FLOAT_OUT_ADD Maximum address available for the float data travelling from the RTE to the PLC Integer positive

Tab-Delimited File Structure

The PLC (Allan Bradley) subsystem refers only to those channels that are to be transferred to the RTE for display, monitoring, logging, etc. The PLC ladder logic and displays are downloaded directly to the PLC and PLC PC.

The PLC (Allan Bradley) channel configuration contains both the generic channel definition columns and the specific PLC (Allan Bradley) input parameters defined in the tab-delimited configuration file.

The rules, syntax and guidelines for creating PLC Allan Bradley channels are described in the following table:

Column Name Description Syntax (data type expected in the column) Multiple Values Allowed (See Note*) Data Required (See Note**)
ADDRESS Specifies the physical memory address of the data item on the PLC. Number 0 - 999 No Yes
BIT Required for DISCRETE channels. Specifies the bit position of the data in the 16-bit word at the specified ADDRESS.

Ignored for FLOAT type channels.		 Number 0 - 15 No See Description.

"Yes" indicates that more than one column value may be specified for a single channel. In this case, additional lines follow the first line of a channel definition.

An additional line is identified by a "*" in its first column (the generic NAME column). Only fields that cause a channel to be defined on more than one line (like LIMIT_TYPE when a channel has more than one limit defined) are qualified as multi-line. Fields that appear multiple times as attributes of a multi-line field (like VALUE), are not qualified as multi-line.

"No" means that the channel is restricted to a single line of the spread sheet.

"Yes" indicates that the field must contain data (it cannot be left blank or filled with an asterisk).

"No" indicates that the field is always optional.

Subsystem Editor for PLC Channels

To edit the Allen Bradley PLC Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration.

To add a new subsystem:

  1. From the Edit menu, select Append Data Record or, in the left pane of the splitter window, click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select PLCAB in the dialogue box to select the type of subsystem.
    A new line in the Subsystem Editor is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to PLCAB. The user must then assign a name and description to the subsystem.

Setting the Maximum Addresses

The maximum addresses for the Subsystem can be edited in the right pane of the splitter window of the Subsystem Editor screen.

Since these addresses are defined using textbox dialogues, the EditSpecific menu item is greyed out and the text boxes are edited directly.

RTE Initialisation File

The PLC (Allan Bradley) subsystem is configured by the RTE, along with all the other subsystems, as a request from the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file would include the following PLC subsystem information:

#--------------------------------------------------------------------------
# PLC Subsystem
#--------------------------------------------------------------------------
SS PLC TRUE
CI host_name plc1
CI service plc_tcp
CI float_in F36
CI float_out F26
CI disc_in B35
CI disc_out B25
