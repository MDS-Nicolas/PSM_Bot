DTS3250 Subsystem (Option*)

Overview

The Digital Thermocouple Scanner DTS3250 Subsystem is used to communicate with the DTS3250 and DTS4050 families of scanners manufactured by Scanivalve Corporation. The supported scanner models include:

  • DTS3250/16Tx
  • DTS3250/32Tx
  • DTS3250/64Tx
  • DTS4050/16Tx
  • DTS4050/32Tx
  • DTS405064/Tx

Though the subsystem is referred to as the DTS3250 subsystem, both DTS3250 and DTS4050 scanners are supported.

The scanners offer 16, 32, or 64 thermocouple measurement channels with the last two digits of the model name indicating the number of dedicated thermocouple channels available on the scanner. The scanners also return the internal Uniform Temperature Reference (UTR) temperatures, which are measured by built-in RTDs.

There is one internal UTR for each group of eight thermocouple channels.

Communication between the scanner and proDAS is done via 10 Base T Ethernet. proDAS supports the following scan rates:

  • up to 40 samples/channel/second for 16 channel scanners
  • up to 20 samples/channel/second for 32 channel scanners
  • up to 10 samples/channel/second for 64 channel scanners

Configuration information for the DTS3250 Subsystem is stored in and read from the configuration file on the Host Computer (.config) and the configuration information (xml configuration data) on the Management Interface Computer.

The DTS3250 driver querries each scanner at the start of the initialisation phase to determine that the model number (channel count) matches the configuration specified in the Subsystem Editor. A configuration mismatch results in:

  • an error message displayed in the UEL
  • a descriptive entry of the error in the Trace proDAS file
  • a conclusion to any further configuration of the scanner

Configuring a DTS3250 Channel

To configure a DTS3250 Channel:

If the desired subsystem is missing from the subsystem list or multiple subsystems (of the same type) are required, the operator must first create the subsystem(s). To create a subsystem, see Subsystem Editor.

  1. Append a channel by selecting Append from the Edit menu as described in Configuration Editors.
  2. Select the name of the DTS3250 Subsystem from the list of available subsystems.

  3. Fill the category independent columns as described in Configuration Editors.
  4. Fill the Channel Specific Columns as described in the Channel Editor.
  5. Set the Host Name column.
  6. Set the Channel Type.
  7. Set the Channel Number.
  8. Set the Output Eu column.
  9. If required, manually modify the Low Limit and High Limit columns.

    These columns are automatically populated by the Channel Editor.

  10. Set the Sheild SW column.

Configuration File Structure

Channel configuration information (e.g.: hardware connection, calculation, limits or actions) can be defined and manipulated through the Channel Editor or imported directly from either an XML file or a tab-delimited text file.

To import configuration information from an XML file, see Selecting the menu item File - Import in the Configuration Editors module. For detailed XML file structure information required to import a DTS3250 channel, see XML File Structure.

To import configuration information from a tab-delimited file, see Extras -- Import TAB-delimited in the Channel Editor module. For detailed tab-delimited file structure information required to import a DTS3250 channel, see Tab-Delimited File Structure.

"Yes" indicates that more than one column value may be specified for a single channel. In this case, additional lines follow the first line of a channel definition.

An additional line is identified by a "*" in its first column (the generic NAME column). Only fields that cause a channel to be defined on more than one line (like LIMIT_TYPE when a channel has more than one limit defined) are qualified as multi-line. Fields that appear multiple times as attributes of a multi-line field (like VALUE), are not qualified as multi-line.

"No" means that the channel is restricted to a single line of the spread sheet.

"Yes" indicates that the field must contain data (it cannot be left blank or filled with an asterisk).

"No" indicates that the field is always optional.

Subsystem Editor for DTS3250 Channels

To edit a DTS3250 Subsystem, the Subsystem Editor must be selected from within the Configuration Administrator.

The tool bar in the Configuration Administrator allows the various configuration editors available for proDAS to be selected.

The Subsystem Editor allows new subsystems to be added to the current configuration Subsystem Editor - Menu Edit.


To add a new subsystem:

  1. Select the menu item Edit - Append Data Record in the menu bar or click in the left pane of the splitter window and then click the right mouse button and select Append Data Record from the context sensitive menu.
  2. Select DTS3250 Subsystem in the dialogue box to select the type of subsystem.
    A new line is appended to the current subsystem configuration in the left pane of the splitter window and the subsystem type is set to the type just selected (as shown in the typical Subsystem Editor screenshot above).
    The user must then assign a name and description to the subsystem.

RTE Initialisation File

The DTS3250 Subsystem is configured by the RTE, along with all the other subsystems, as a request form the Management Interface GUI. For legacy purposes, the configuration request can also be specified in the command line of the RTE.

A typical RTE initialisation file would include the following DTS3250 Subsystem information:

# ==============================================
# DTS (Digital Thermocouple Scanners) subsystem
# ----------------------------------------------
SS    DTS                TRUE
# DTS service name (default telnet)
CI     port              telnet
# software or hardware trigger; SW or HW
CI     trigger           HW
# external trigger rate (Hz)
CI     hw_trigger_rate   100
# number of averages during calibration
CI     calavg            1
# maximum allowable difference between RTD values
CI     maxdelta          10.0
# scan rate for accelerated transfer
CI     accel_xfer        20
# reply time-out in seconds
CI     reply_timeout     3
#------------------------------------------------
