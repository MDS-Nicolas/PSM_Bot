Instrument Controls (Option*)

Overview

Accessed from the Management GUI Tools menu, the proDAS Instrument Controls have been divided into the following main categories:

  • Pre Test Functions
  • Post Test Functions
  • Configuration Editors
  • Custom Tools

The Management GUI Tools menu is configurable. The contents of each main category is customised for every customer to display only the functions applicable to that customer.

Pre Test Functions

proDAS gives the operator the ability to run various Pre Test Functions prior to acquiring data. These tests may allow the user to perform functions such as confirming channel readings are within the appropriate ranges, and zeroing temperature and pressure channels. The following Pre Test Functions, as applicable, can be performed:

The Management GUI associates each function with a configurable user level such that only authorised users can perform these functions.

  • AVM Configuration and Diagnostics
  • DC Strain Control
  • DDS AVM Calibration
  • DDS Calibration
  • DTCI System Control
  • DTS3250 System Control
  • Engine Management System (EMS)
  • FBO GUI
  • HAMS Calibration
  • Hyscan Control
  • IEEE Controls
  • Open Transducer Detection (OTD) Zero / Transient TC Zero
  • ProDAQ Calibration
  • Sensor Calibration Application (SCA)

  • Thrust Calibration:
    • MDS Thrust Calibration Utility
    • Thrust Calibration Application (TCA)
  • TruTemp Control

The above list of pre-run checks are not applicable to every customer. The execution of the pre-run checks depends on customer hardware and procedures. Please refer to the "Home" section of the Docstation for details of the pre-run checks applicable for your system.

AVM Configuration and Diagnostics

The AVM Configuration and Diagnostics screen is displayed at the startup of the AVM Configuration and Diagnostic Utility and is the main navigation screen for all functions within the utility.

From the AVM Configuration and Diagnostics screen, the operator can:

  • access the AVM Analogue Channel Configure screen
  • access the Calibration screen
  • exit the AVM Configuration and Diagnostics Utility
  • minimize the utility
  • access an online help file

While the AVM Configuration and Diagnostics screen is displayed, a communications check verifies that the ENDEVCO equipment is responding to the AVM Configuration and Diagnostics Utility. Flashing LEDs and an event log allow the operator to monitor the progress of the communications check. During the communications check, all buttons are greyed out and non-accessible by the operator.

Accessing the AVM Configuration and Diagnostics Window

To access the AVM Configuration and Diagnostics window:

  1. From the Management GUI Control Panel Tools menu, select AVM.
    The AVM Configuration and Diagnostics window is displayed and a communications check is performed.
    On completion of the communications check, the buttons on the AVM Configuration and Diagnostics screen become enabled.

AVM Analog Channel Configure

The AVM Analog Channel Settings screen can be displayed, upon successful completion of the communications check, by clicking the Analog CHANNEL CONFIGURE button on the AVM Configuration and Diagnostics screen.

The screen automatically determines which features are applicable based on the configuration that has been read from the AVM. Features that are not applicable will be greyed out (see Velocity in the screen below).

The AVM Analog Channel Settings screen can display the current configurations and status of the AVM analog channels. From the AVM Analog Channel Settings screen, the operator can:

  • load (receive) the current channel settings directly from the ENDEVCO unit
  • save (transmit) the currently selected channel settings directly to the ENDEVCO unit
  • access the Self Test screen for analogue channels
  • access an on-line help file
  • close the AVM Analog Channel Settings screen and return to the previous screen

The AVM Analog Channel Settings screen is divided into four panels. The upper (first) control panel allows the operator to select the:

  • Channel Number - based on the number of Analogue cards in the Endevco chassis
  • Filter Type - low pass, high pass or band pass filter
  • Mode (output mode) - acceleration in g's, velocity in ips, displacement in mils
  • Transducer Type - Differential Piezoelectric (DPE) transducer, DC/g; Single Ended Piezoelectric (SEPE) transducer, pC/g; Velocity transducer, mV/ips; Isotron transducer or RCC (mv/pC)
  • Time Constant - DC voltage update rate
  • Filter Select - Bypass Internal Filter (BIF), Enable Internal Filter (EIF), Enable External Filter (EEF)
  • Filter Order - chooses 4 pole or 6 pole Butterworth Internal Filter

The second control panel allows the operator to set the:

  • Lower Cutoff - sets corner frequency of high-pass internal filter
  • Upper Cutoff - sets corner frequency of low-pass internal filter
  • Acceleration - sets the transducer sensitivity in pC/g
  • Velocity - sets the transducer sensitivity in mV/ips
  • AC Volts (full-scale) - sets the fullscale AC output voltage
  • DC volts (full-scale) - sets the fullscale DC output voltage
  • Continuous Range (full-scale EU) - sets the fullscale EU range

The third panel displays various messages relating to the use of the AVM Analogue Channel Settings screen and the communication between the AVM Diagnostic Utility and the ENDEVCO chassis.

The bottom panel provides buttons that allow the operator to:

  • RECEIVE setting from UNIT - in order to check current configuration
  • trANSMIT setting to UNIT - in order to update the configuration
  • run a SELF TEST - used to check Tracking Filter Amplifier System (TFAS) hardware integrity
  • Return to the AVM Configuration and Diagnostics window
  • access the online Help

AVM Analog Channel Settings

The AVM Analog Channel Settings are selected from the AVM Analog Channel Settings screen. To configure the analog channels:

  1. Click-and-drag the controls, in the control panels, to the desired settings.
  2. Click on the up and down arrows, on the display windows beneath the corresponding controls, to increase or decrease (fine tune) the settings for the control. Selecting the field allows the user to input values manually.
  3. If the Transducer Type is set to Velocity, Isotron, or Noise Test, the Acceleration control in the second control panel is disabled (greyed out).
  4. If the Transducer Type is set to DPE or SEPE, the Velocity control in the second control panel is disabled (greyed out).
  5. If the Filter Select is set to BIF (Bypass Internal Filter), the Filter Type control in the second control panel is disabled (greyed out).
  6. To send the new channel settings to the ENDEVCO unit, select trANSMIT setting to Unit.
  7. To load the channels settings from the ENDEVCO unit, select RECEIVE setting from Unit.
  8. Ensure that the new channel settings that were sent are the same as the received settings.

Analog Channel Self Test

The Analog Channel Self Test function is used to determine the card, transducer and cable integrity as well as to determine if the card is performing within specifications. The channel internally processes a signal and compares the output with predefined limits.

The analog channel self-test is initiated from the AVM Analog Channel Settings screen.

  1. To initiate a self-test of the AVM Analog Channels, select SELF TEST.
    The AVM Analog Self Test screen is displayed:
  2. Select the channels to be included in the self-test by clicking in the corresponding box.
    A check mark indicates that the channel has been selected. To deselect the channel, click in the corresponding box.
  3. To start the AVM Analog Channels self-test, select SELF TEST.
    If no channels have been selected when the SELF TEST button is selected, an Information Window dialogue box is displayed with the message "You must select at least one channel to self test" Select OK to return to the AVM Analog Self Test screen.
  4. When the self-test is completed, a confirmation dialogue box is displayed.
  5. Select OK.
  6. To return to the AVM Analog Channel Settings screen, select Return on the AVM Analog Self Test screen.

DDS Calibration

DDS Calibration is used to automatically generate the calibration stimuli, measure the ADC output from the DDS and compute the appropriate calibration coefficients.

Performing a DDS Calibration

To perform a DDS Calibration:

  1. From the Management GUI Control Panel Tools menu, select DDS Calibration.
    The Dynamic Calibration Type Selection window is displayed.

    The DAS or Telemetry button will be disabled if not available in the configuration file.

    If DAS is selected, go to step 2.
    If Telemetry is selected, go to step 14.
    If Both is selected, go to step 15
    If Abort is selected, the DDS Calibration application is closed.
  2. Select DAS.
    The required sensor and channel data is retrieved from the database. Once retrieved, the Calibration Types panel is enabled.

    Speed channels do not have to be calibrated.

    For all calibration types selected (other than General Purpose Live Calibration), the information and results are displayed in the main window.

  3. From the Calibration Types panel, highlight GenPurposeLiveCal and click Select. The general purpose live calibration screen is displayed.

    The Skip button advances the channel. The Return button stops the General Purpose Live Calibration.

    The user can select AC Live Calibration or DC Live Calibration for all the configured General Purpose channels.

    If AC Live Calibration is required go to step 4.
    If DC Live Calibration is required go to step 8.
  4. Select AC Live Calibration. The AC Live Calibration screen is displayed.
  5. Enter the required information and select Go.
  6. Select Accept Point.
    The results are entered into the table. The Cancel Point button allows the user to only abort the current calibration point. The Accept Point and Remove Point buttons only become enabled once the system has ensured that the Measured Frequency is within the set range. If more than one point is required, repeat steps 5 and 6.

    There must be at least one "Normalised" entry in the results table before the Cal Complete button is enabled. To normalise the entry, double click on the entry in the Normalised column. The resulting "Yes" indicates normalisation.

  7. Select Cal Complete.

    Cal Complete - takes the user to the General Purpose Live Calibration screen and advances to the next channel selection.
    Abort Calibration - takes the user to the General Purpose Live Calibration screen but keeps the same channel selection.

  8. Select DC Live Calibration. The DC Live Calibration screen is displayed.
  9. Enter required information and select Go.
  10. Select Accept Point.
    The results are entered into the table. The Accept Point and Remove Point buttons only become enabled once the system has acquired the data twice. Repeat steps 9 and 10 to enter a second point.

    There must be at least two entries in the results table before the Cal Complete button is enabled.

  11. Select Cal Complete. The DC Live exit confirmation dialogue box is displayed.

    Perform AC Calibration - takes the user to the AC Live Calibration screen.
    Cal Complete - takes the user to the General Purpose Live Calibration screen and advances to the next channel selection.
    Cancel - takes the user back to the DC Live Calibration screen.

  12. For any other calibration type other than General Purpose Live Calibration, highlight the calibration type and click Select.
    The calibration is performed. If One Channel is selected, a Channels selection panel is displayed.
  13. Select an individual channel for calibration and click Select.
    The calibration is performed and completed.
  14. Select Telemetry.
    As part of the data retrieval a reading of the STINCA Gain settings is performed.
    When finished a report of any errors in retrieving telemetry data from the database is displayed.
    The Calibration Types panel is then enabled, showing the list of available telemetry selections for calibration.
    To start the calibration process, highlight a Calibration Type and click Select. The calibration is performed and results are displayed on the panel.
  15. Select Both. All DAS and Telemetry calibration types are displayed.
  16. To print the report, select Print.
  17. To reconfigure the DDS Subsystem, select Return.
  18. To view the help file, select Help.

DTCI System Control

The user controls the DTCI system from the Management GUI control panel interface to zero, set purge mode on, and set purge mode off, with a specific panel for the latter two. The following characteristics apply to the operation of the DTCI Subsystem:

  • if Zero All is selected by the operator, the RTE initiates a zero operation on all scanners connected to all configured DTCI systems; the new zero coefficients are stored back to the EEPROM of the scanners
  • if Purge Mode On is selected, the RTE commands all scanners connected to all configured DTCI systems into purge mode; the purge operation switches the calibration valves of all affected scanners from the RUN position to the CAL position
  • if Purge Mode Off is selected, the RTE commands all scanners connected to all configured DTCI systems out of purge mode into run mode; the affected scanners have their calibration valves switched from CAL to RUN

The FBO GUI, also known as FBO CWINCA-TINCA application, is used to verify the continuity for the CWINCA Telemetry unit and the connectivity for the TINCA Telemetry unit. The raw data is retrieved from the Cal Server.

The FBO GUI cannot be used at the same time as the DDS Calibration.

Depending on configuration the Open Transducer Detection (OTD) Zero or Transient TC Zero screen is displayed.

The channels using this feature are often thermocouples.

Static Check

The Static Check feature performs a check of all measured channel values, compares the values to a known condition, such as zero and flags all channels which are out of the expected limits.

The Static Check INI file supports groupings of static checks to be performed. The INI file is structured as follows:

[StaticCheck]

This general section contains information on the verbosity for logging the results, the number of decimal places to use when displaying channel values, the number of static check subsystem groups (optional), and the number of subsystems that will be checked.

Key Description
Verbosity If set to HIGH, information on all channels that have passed or failed the checks are displayed to the log. If set to LOW only information on channels that have failed the checks are displayed.
ChannelValueDecimalPlaces Number of decimal places for displaying channel values.
NumberOfSSGroupsToCheck Number of Static Check subsystem groups to be processed. This field is optional, and if not defined, only one static check group is assumed.
NumberOfSSToCheck Total number of subsystems from all groups that need to be checked.

[StaticCheckGroups]

This optional section consists of the assignment of the names to the static check groups. These are user defined and for visual purposes only in the Static Check Panel. If this section is not defined, only one static check group is assumed.

Key Description
SSGroup01 The user specified name for the first static check group of subsystems. The last two digits of the field increments sequentially for subsequent static check groups (e.g.SSGroup01, SSGroup02, SSGroup03).

[StaticCheck01] (section header without grouping)
[StaticCheck01-01] (section header with grouping)

This is a section header for the first static check. If there is no grouping of static checks, the last two digits of the section header increments sequentially for subsequent section headers (e.g. StaticCheck01, StaticCheck02, StaticCheck03, etc.).

If grouping of static checks is defined, the first two digits at the end of the section header name indicate the sequential number of a static check subsystem group, whereas the last two digits at the end of the section header indicate the sequential number of the static check within the current group. Both indicator digit pairs increment sequentially for subsequent section headers (e.g. StaticCheck01-01, StaticCheck01-02, StaticCheck02-01, StaticCheck02-02, StaticCheck02-03, StaticCheck03-01, etc.).

The keys within these two section headers are identical.

Key Description
SSTypeName This is the name of the subsystem whose channels are checked (e.g. GASSAI, GASSAO, GASSDIO, GASSVXI, PBS).
Group This is the name of the group whose channels are checked. At least one of the SSTypeName of Group must be defined.

This field can be used as a filter for any subsystem or on its own.
VXIGroup This is the name of the calibration group whose channels are checked (e.g. Capacitance Probe, Vibration).

This field is only used for the GASS related subsystems (GASSAI, GASSAO, GASSDIO, GASSVXI, GASSTC).
NameBeginsWith Filter the list of channels with the defined letters at the beginning of the name.
NameContains Filter the list of channels that contain the defined letters in the name.
NameEndsWith Filter the list of channels with the defined letters at the end of the name.
UseRefChannel This is a Y/N switch that indicates whether the channels are checked against a reference channel.

Either the UseRefChannel or UseRefValue key should be present, but not both at the same time. If neither field is defined, a reference value of 0 will be used.
RefChannelName If UseRefChannel=Y, this field determines the channel name of the reference channel.

This field is not required if there is no reference channel.
UseRefValue This is a Y/N switch that defines whether the selected channels are compared against a reference value.

Either the UseRefChannel or UseRefValue key should be present, but not both at the same time. If neither field is defined, a reference value of 0 is used.
RefValue If UseRefValue=Y, this field specifies the reference value that the channels are checked against.
Tolerance This is the +/- tolerance for the reference value, which is obtained from the defined reference channel or from the defined reference value. The value of the channel being checked must be within the reference value of +/- the tolerance value in order to pass the check. If not defined, a tolerance value of 0 is used.

Temperature Scanning Module (TSM) Offset Correction

The TSM panel gives the user the ability to select one or more temperature scanners, any number of channels, and to specify the temperature/resistance that will be applied during an offset adjustment.

To perform an offset correction:

  1. From the Management GUI Control Panel Tools menu, select TSM. The Temperature Scanner Select Scanner window is displayed.
  2. Select one or more scanners from the Select Scanner window and click Next or click the Specify Channel Type tab. The Specify Channel Type tab is displayed with all channels associated with the previously selected scanner.
  3. Select a channel of the appropriate type from the Specify Channel Type window and click Next or click the Pick Channels tab. The Pick Channels tab is displayed.
  4. Select one or more channels from the Select channels window (or click Select All) for Offset Correction and click Next or Offset Correction tab. The Offset Correction tab is displayed.
  5. Enter a target value in the Target Value field. Click Offset Correction to calibrate the channel(s).

    The target value should have the same units as the "Default EU" as defined in the scanners subsystem (see Subsystem Editor for TSM Channels for details).

    All scanners being zeroed at the same time must have the same "Default EU".

  6. Click Next or the Update tab. The Update tab is displayed.
  7. To complete the offset correction and store the data back to the scanner(s), click Update.

TruTemp Control

TruTemp measures thermocouple (T/C) input voltage and resistance temperature detector (RTD) resistance and transmits these values to proDAS for storage in the current value table (CVT). The TruTemp System Control panel provides diagnostic procedures to:

  • disable or enable the thermocouple open circuit test
  • measure the loop resistance of temperature thermocouples
  • measure the thermocouple conductance to earth

Disable/Enable OC Test

The RTE must be at idle state to Disable/Enable OC Test.

To Disable or Enable OC Test:

  1. From the Management GUI Control Panel Tools menu, select TruTemp. The TruTemp System Control panel is displayed.
  2. To toggle the OC Test, select Disable OC Test or Enable OC Test.
    If the RTE is at IDLE state, a message window is displayed asking the user to confirm the operation.
  3. Select Yes. The Management GUI sets the scanner configuration. A UEL message is then displayed and the user is returned to the TruTemp System Control Page.
  4. Select Close to close the TruTemp System Control Panel.

Loop Resistance

During the loop resistance test, the resistance of each thermocouple channel is measured to ensure that the thermocouple is connected to the TruTemp System and that there are no breaks in the thermocouple wire.

The RTE must be in Scan mode to perform a Loop Resistance test.

To perform a loop resistance test:

  1. From the Management GUI Control Panel Tools menu, select TruTemp. The TruTemp System Control panel is displayed.
  2. Select Loop Resistance.
    A message window is displayed asking the user to confirm the operation and a message is displayed in the UEL.
  3. Select OK to start the test.
  4. When the test is complete a message window is displayed. Any defective channels are listed in the Event Handler Window and recorded in the UEL. The results of the test are stored in the file /users/RTE/bin/tmp/trUTEMP2.log on the Host Computer.
  5. Select OK. The message window is closed and the user is returned to the TruTemp System Control panel.
  6. Select Close to close the TruTemp System Control Panel.

Conductance To Earth

The conductance to earth test checks for electrical shorts, between the thermocouple wire and the metal components of the engine, caused by damaged insulation on the thermocouple wire.

The RTE must be in Scan mode to perform a Conductance To Earth test.

To perform a conductance to earth test:

  1. From the Management GUI Control Panel Tools menu, select TruTemp. The TruTemp System Control panel is displayed.
  2. Select Conductance To Earth.
    A message window is displayed asking the user to confirm the operation and a message is displayed in the UEL.
  3. Select OK to start the test.
  4. When the test is complete a message window is displayed. Any defective channels are listed in the Event Handler Window and recorded in the UEL. The results of the test are stored in the file /users/RTE/bin/tmp/trUTEMP2.log on the Host Computer.
  5. Select OK. The message window is closed and the user is returned to the TruTemp System Control panel.
  6. Select Close to close the TruTemp System Control Panel.

Zeroing or Purging Pressure Scanner Channels (PBS)

The following terminology has the same meaning:
  •  Pressure Brick
  •  Pressure Scanner

Scanners specified in the Management GUI ini file parameter PurgeIgnoreList do not appear in the list of purge channels in the Pressure Scanner Functions Panel.

Purging can be performed on all configured pressure scanner channels in the system before each test.

The enable purge channel is defined in the MgtGUI.ini file as:
  •  [PBS]
  •  Purge Time = 30
  •  Enable Purge Chan = Purge Enable
  •  Purge Ignore List = PB131487

If the channel listed as the enable purge channel (Purge Enable) is False, then the purge functions are disabled.

The Pressure Scanner Functions panel provides procedures to:

  • purge all scanners
  • purge one scanner
  • zero all scanners

Accessing the Pressure Scanner Functions Panel

To access the Pressure Scanner Functions panel:

  1. From the Management GUI Control Panel Tools menu, select PBS.
    The Pressure Scanner Functions panel is displayed.

Purge All Scanners

The Purge All function allows the user to purge the pressure lines between the scanners and engine. The purge is performed on all configured pressure scanner channels.

Purging the pressure scanner isolates the transducers from the measurement ports and connects the ports to an external purge connection (pressure source). Pressure is forced out of the measurement ports, removing any blockages or contaminants between the scanner and the pressure tapping on the engine.

The pressure source is typically a conditioned air or nitrogen supply in the test cell.

Do not purge scanners with the dust caps installed or with the measurement port sealed, as damage to the scanners will result.

To purge all scanners:

  1. Enter the duration of the purge, in seconds, in the Purge Time field of the Pressure Scanner Functions panel.

    The default purge time is 30 seconds. This time can be modified in the config.ini file.

  2. Select Purge All Scanners.
    A dialogue box is displayed, indicating the scanners are being purged.
  3. When all scanners have been purged, the dialogue box is closed and the operator is returned to the Pressure Scanner Functions panel. The pressure channels are returned to their normal measurement mode.

Purge One Scanner

The Purge One Scanner function allows the user to purge the pressure lines between a single pressure scanner and the engine.

Do not purge scanners with the dust caps installed or with the measurement port sealed, as damage to the scanners will result.

To purge one scanner:

  1. Enter the duration of the purge in the Purge Time field.
  2. Select Purge One Scanner.
  3. Select the name of a channel on the scanner to be purged from the Pressure Scanner Functions panel.
  4. Select Start or select Cancel to exit the purge one function.
    A dialogue box is displayed, indicating the scanner is being purged.
  5. When the scanner has been purged, the dialogue box is closed and the operator is returned to the Pressure Scanner Functions panel. The pressure channels are returned to their normal measurement mode.

Zero Scanners

The zero calibration (Zero) command causes the pressure scanners to shift the internal calibration valves into the CAL position. The pressures that are present at the CAL and the CAL REF inputs are used for the minimum or zero calibration pressure. On completion of the zero calibration, the calibration valves are shifted into the RUN position and the offset coefficients are updated in pneumatic pressure scanner memory.

The Zero All function allows the user to zero all configured pressure scanners simultaneously, resulting in a recalculated offset coefficient for each of the channels being calibrated and automatically compensating for any drift errors in the transducer offset.

Since the coefficients do not get permanently stored in the scanner it is recommended that the zero command be performed prior to each engine test.

To zero all scanners:

  1. Select Zero All from the Pressure Scanner Functions panel.
  2. When the scanners have been zeroed, the UEL states that the task has been completed.

The Zero Multiple function allows the user to zero multiple (at least one) configured pressure scanners simultaneously, resulting in a recalculated offset coefficient for each of the channels being calibrated and automatically compensating for any drift errors in the transducer offset.

Selecting one scanner will select all channels that are tied to that scanner.

To zero multiple scanners:

  1. Select Zero Multiple from the Pressure Scanner Functions panel.
  2. Select the scanners to be zeroed from the list in the Pressure Scanner Functions panel. Hold CTRL while selecting to select multiple scanners.
  3. Select Start or select Cancel to exit the purge one function.
    A dialogue box is displayed, indicating the scanner is being purged.
  4. When the scanners have been zeroed, the dialogue box is closed and the operator is returned to the Pressure Scanner Functions panel, and the UEL states that the task has been completed.
