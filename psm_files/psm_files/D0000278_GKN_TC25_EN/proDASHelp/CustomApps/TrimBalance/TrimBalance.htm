Engine Trim Balance (Option*)

Overview

The Trim Balance system is used in conjunction with the CEC Vibration Monitoring System that addresses engine vibration problems during test. The system is configured to collect data at predetermined engine speeds from vibration monitoring hardware. The data is evaluated and displayed in table and plot formats in the form of a balancing solution. The solution directs an operator to place various Original Equipment Manufacturer (OEM) balance weights at specific locations on an engine fan Low Pressure Turbine (LPT) or high pressure compressor (Core). Similar to the balance weights on the wheel rims of a car, the weights and their distribution pattern trim or lower vibration that occurs at specific rotational speeds.

The system uses two types of data (vibration phase angle and amplitude), measured on up to 3 vibration transducers, to produce a balancing solution for the fan, LPT or core. The balance weight part numbers and effective weights are presented to the user to allow for easy engine plane configuration. The solution is expressed with phase lag and sensitivity coefficients that indicate the response of the fan, LPT or core to the installed balance weights. The sensitivity coefficient is a measure of the amount of weight per unit of vibration change. It is used to calculate the magnitude of the weight to install. The phase lag is used to calculate the weight location. Vibration severity varies at different rotational speeds, thus several sets of coefficients are generated to calculate an overall balance solution. A record of coefficients is created after each successful balance. These coefficients apply to a family of engines, and are used to calculate the first weight (one-shot) solution during the next engine balance session.

The system allows the operator to view the cumulative effect of combining (summing) a balancing solution with another solution. Data from previous tests may also be retrieved.

Residual vibration is also predicted by Trim Balance after the software calculates the solution at the Residual Vibration Levels screen. At this screen, the operator may elect to provide a "User" weight solution. The user solution can, for example, be "tuned" for specific speeds, instead of using the "MDS" solution that is "averaged" for all speeds.

The system provides real time display of vibration amplitudes and phase angles. The Trim Balance System is configured to collect data at predetermined engine speeds from vibration monitoring hardware. The data is evaluated and displayed in table and plot formats in the form of a balancing solution. The vibration measurement system receives signals for the velocity transducers (voltage output), accelerometers with charge output (pC/g) and accelerometers with integral charge amplifiers (ICP type voltage outputs).

The CEC Vibration System tracks against either N1, N2, N3 vibration signals to produce relevant tracked orders. Additional vibration/trim balance displays are available to the test cell operator. The system reconfigures the CEC amplifiers to produce tracked order vibration and phase angle outputs. Tracking filters allow vibration levels that change with rotational speed to be analysed.

If an Endevco Vibration System is being used, the Trim Balance software can reconfigure the Endevco TFAS Vibration Signal Conditioner to Trim Balance mode allowing it to output the phase angle needed for engine balancing. The system restores the Endevco to initial values after balancing is complete.

During test, the system:

  • starts and stops test data logs at predetermined test speeds
  • indicates data quality (good, suspect or bad) and automatically selects good data while excluding bad data
  • expresses data in graphical and tabular form
  • generates and retrieves reports
  • updates sensitivity and phase lag coefficients to reflect test results for a specific family of engines
  • records imbalance data at multiple speeds

The system is configured with a unique tb log definition using the Transient Log Editor. A tb.def log file definition stores the channel identifiers that are used by the Trim Balance. The 'new' and 'specifics.ini' files, within the Trim Balance directory, identifies channels that proDAS uses to receive data for the test, and other engine-specific balance configuration settings.

The operator selects up to three vibration transducers to be used for balancing. Each transducer requires an amplitude and phase angle channel assignment. The speed data channel is common to all sensors, thus, up to seven data channels may be required by proDAS per test. An auto-report configuration parameter is included within tb.def to convert data into a tab-delimited text file.

After the Record Data button is selected on the Vibration Measurement GUI, Trim Balance sends the command to proDAS to start and stop data acquisition in the tb log. proDAS creates a report from the data using tb.def that is sent to Trim Balance for analysis. The analysis results are displayed on the Imbalance Vector Data GUI. A balance weight solution (weights and their distribution pattern on the engine fan, LPT or core) is displayed on the Balance Weight Vector GUI.

The system resides on the Management Interface Computer and communicates with the Endevco AVM System via an IEEE 488 interface (i.e. GPIB connection), if applicable, and communicates with proDAS through an Ethernet link.

The Trim Balance System handles the following:

  • Fan or Low Pressure Turbine (LPT) Balancing
  • Core Balancing

Starting the Trim Balance System

The Trim Balance System can be started in one of two ways:

  • via the Management GUI Control Panel where the username and password are passed automatically
  • as an independent application where the username and password are not passed automatically

To start Trim Balance from the Management GUI Control Panel:

  1. From the Tools menu, select Trim Balance.

    The operator is automatically logged into the editor with the same user name and privileges as entered for the Management GUI.

    Trim Balance is started and the Welcome to the Trim Balance Application screen is displayed.

To start Trim Balance as an independent application:

  1. Double-click the Trim Balance executable file, from your directory structure.
    The Login window is displayed.
  2. Enter a valid Username and Password.

    The user name and password must be a minimum of three characters and five characters long, respectively, to enable the Login button.

  3. Select Login.
    Trim Balance is started and the Welcome to the Trim Balance Application screen (below) is displayed.
  4. To close the Login window, select Cancel.

    The Welcome to the Trim Balance Application screen provides access to the Balance Plane Configuration screen and the Retrieve Report screen. The Welcome to the Trim Balance Application screen displays the Trim Balance Session Log. The operator can scroll through the session log by using the scroll bar on the right-hand side of the session log window.

Fan or Low Pressure Turbine (LPT) Balancing

To begin a Fan or LPT balance session:

  1. From the Welcome to the Trim Balance Application screen, select Balance Engine. The Balance Plane Configuration screen is displayed.
  2. To retrieve, generate, and print reports, select Get Report. The Retrieve Report screen is displayed.
  3. To close the Trim Balance Application and return to the Test Ending Utility screen, select Exit.
  4. For on-line help, select Help. The On-line Help screen is displayed.
  5. For information on the Trim Balance Application, select About Trim Balance. The About Trim Balance screen is displayed.
  6. To minimise the Trim Balance Application, select Minimize.

Balance Plane Configuration Screen

The Balance Plane Configuration screen allows the operator to review and change the current balance weight configuration for the engine being balanced.

To initialise the Balance Plane Configuration screen:

  1. From the Welcome to the Trim Balance Application screen, select Balance Engine.
    A "Please select..." window is displayed.

    Depending on how the system is configured for the given engine model the following possibilities are available:
    •       Fan only balance - the screen below is omitted
    •       Fan or LPT balance - the user chooses between provided options
    •       Core balance - is displayed only if the Core balance, in the "new file", is set to True. For Core Balance details, see Core Balancing.

  2. From the "Please select..." window, select Fan/LPT to perform FAN/LPT balancing.
  3. Select OK. The Balance Plane Configuration screen is displayed.
  4. If the weight values are unknown, select OK. The Limits and Tolerances screen is displayed. For further details, see Limits and Tolerances screen.

    The user can continue to acquire a first data set of imbalance data with the Initial Weight Configuration "unknown". The current balance weights can be surveyed and entered into the Trim Balance Application after the engine is shut down. For details, see Balance Weight Positions procedure, step 5.

  5. If the weight values are known, select Update. A new Balance Plane Configuration screen is displayed.
  6. To add the current weight configuration, select Add Weights. A new Balance Plane Configuration screen is displayed.
  7. To clear the information in the Initial Configuration panel, select Clear.
    The information in the Initial Configuration panel is cleared and the Clear button changes to a Revert button.
  8. To restore the information in the Initial Configuration panel, select Revert.
    The information is restored to the Initial Configuration panel and the Revert button changes to a Clear button.
  9. To add an item to the Initial Configuration panel, select the location and part number from the Location and Part Number lists, respectively, and select .
    If a location is unavailable (see the Balance Plane Configuration screen below), select the location and select .
    If the part number is not known, select the location and then select Unknown. The blade location will be added to the Initial Configuration panel and the part number will be listed as Unknown.
  10. To remove an item from the Initial Configuration panel, select the corresponding blade number from the Location list and select .
  11. When the Initial Configuration list has been completed, select Continue. The following Balance Plane Configuration screen is displayed.
  12. To make other changes to the Initial Configuration, select Update.

    This step can be performed until the user is satisfied with the results.

  13. If the Initial Configuration is acceptable, select OK.
    The test parameters are retrieved.
  14. When the test parameters have been retrieved, the Limits and Tolerance screen is displayed.
  15. If required, to close the Balance Plane Configuration screen and return to the Welcome to the Trim Balance Application screen, select Exit. A confirmation window is displayed.

Limits and Tolerances Screen

The Limits and Tolerances screen is displayed when the test parameters have been retrieved (see Balance Plane Configuration procedure, Step 13) or when Previous Screen is selected on the Vibration Measurement screen.

To set the limits and tolerances for the selected engine configuration:

  1. Modify the target tolerance (in RPM or Percent) in the Target Tolerance field, if required.
    Use the up and down arrows to make incremental adjustments to the target tolerance value.
  2. Modify the selection of the transducer(s) from the Select Transducer panel, if required.
    Any combination of up to three transducers can be selected.
  3. For each transducer selected, the user has the option to enter, or modify the High Limit and the High High Limit.
    Use the up and down arrows to make incremental adjustments to the limits.
  4. To return to the Balance Plane Configuration screen, select Previous.
  5. If all parameter settings are acceptable, select Continue.
    When the system has been prepared for data acquisition, the Vibration Measurement screen is displayed.

Vibration Measurement Screen

The acquisition of vibration measurement data is initiated from the Vibration Measurement screen. The appearance of the Vibration Measurement screen depends on the number of transducers selected from the Limits and Tolerances screen (any combination of up to three transducers can be selected). If only one transducer is selected, only one set of gauges is displayed. Each additional transducer adds a set of gauges to the Vibration Measurement screen. Magnitude, Phase, and Quality columns are added to the table for each additional transducer.

To return to the Limits and Tolerances screen at any time:

1. Select Previous Screen.
The Previous Screen Selected panel is displayed.

2. Select Yes.
The Previous Screen Selected panel and the Vibration Measurement screen are closed and the Limits and Tolerances screen is opened.

To resume vibration measurement:

1. Select No from the Previous Screen Selected panel.
The Previous Screen Selected panel is closed and the vibration measurement is resumed at the Vibration Measurement screen.

To start the acquisition of vibration measurement data:

  1. Wait until the N1 speed is within the target speed range.
    The At Target and Captured LEDs are red while N1 is out of the tolerance range.
    The Vibration and Phase gauges are updated continuously.
  2. When N1 is within tolerance range of the target speed, the At Target LED changes from red to green and the Acquire button is displayed and enabled.
  3. To record the data for the current target speed, select Acquire.
    The Acquire button is removed from the Vibration Measurement screen (below) and the Skip and Previous Screen buttons are disabled. While recording data the Captured LED changes from red to yellow. After approximately five seconds, the data is recorded and the Captured and At Target LEDs turn red and the Skip button is enabled.
  4. To advance to the next Target Speed without recording the data for the current target, select Skip.

    Skip is normally only selected if the target speed cannot be safely obtained (i.e. if the vibration level is too high or target speed is above take-off speed).

  5. Repeat Steps 1 through 4 until all target speeds have been processed (i.e.: Acquire or Skip).
  6. When all target speeds have been acquired, the Restart Acquisition, Engine Balanced, and Continue buttons are enabled and the At Target and Captured LEDs and the Vibration and Phase gauges are removed from the Vibration Measurement screen (below).
  7. If required, to restart data acquisition, select Restart Acquisition and go to Step 1 of the Vibration Measurement Screen.
  8. If the engine balance is acceptable, select Engine Balanced.
    The Vibration Measurement screen is closed and the Engine Balanced Screen is opened. An Update Coefficients confirmation box is displayed following a One Shot Solution. To update the coefficients file, select OK.
  9. To view the imbalance vector data and progress towards calculating a new balance solution, select Continue.
    The Vibration Measurement screen is closed and the Imbalance Vector Data Screen is opened.

Imbalance Vector Data Screen

The Imbalance Vector Data screen is displayed by selecting:

  • the Continue button on the Vibration Measurement screen
  • the Previous Screen button on the Weight Vector Data screen
  1. By default, all imbalance vector data is selected for viewing on the weight vector plot.
  2. Select Previous or Next to display different transducer data.
  3. To remove data from the plot view or to affect weight determination, click on the corresponding buttons in the Select column.

    Green quality vectors are always included regardless of the selection; red quality are always excluded regardless of the selection; yellow quality inclusion is dependant on the selection.

  4. When the vector data selections have been completed, a plot of the selected imbalance vector data can be viewed by selecting Plot Selections.
  5. To view a plot of the selected imbalance vector data, select Plot Selections.
    The plot is displayed on the Imbalance Vector Data screen (below) and the Plot Selections button changes to Update Plot.
  6. To change or update the imbalance vector data selections, enable or disable the buttons in the Select column and select Update Plot (repeat as required).
  7. If the imbalance vector data is acceptable, but the engine requires further balancing, select Continue to proceed with the balance weight calculations. A dialogue box (below) is displayed, prompting the operator to either cancel or proceed with the solution determination.

    The "red" data is automatically excluded.

  8. Select Cancel to abort the solution determination and return to Step 1.
  9. To continue with the solution determination, select OK.
    The weight vectors are calculated and the Weight Vector Data screen is displayed.
  10. If the vibration is within acceptable limits, select Engine Balanced. The Engine Balanced screen is displayed.
  11. To re-run the test, select Re-Run Test and go to Step 1 of the Vibration Measurement screen.
  12. To close the Imbalance Vector Data screen, select Exit.
    When the report has been printed or the balance session has been terminated, the Welcome to the Trim Balance Application screen is displayed.

Weight Vector Data Screen

If the imbalance vector data was acceptable on the Imbalance Vector Data screen and the Continue button is selected, the Weight Vector Data screen is displayed.

  1. By default, all weight vector data is selected for weight ("Outlier") determination (i.e.: all buttons in the Select column are enabled).
  2. To delete data from the Select Data for Outlier Determination table, make sure the corresponding buttons in the Select column are disabled.

    Green quality vectors are always included regardless of the selection; red quality are always excluded regardless of the selection; yellow quality inclusion is dependant on the selection.

    If the Previous or Next buttons are enabled, check these pages and make sure only the required data entries are selected.
  3. When the vector data selections have been completed, go to Step 6.
  4. To view a plot of the selected weight vector data, select Plot Selections.
    The plot is displayed on the Weight Vector Data screen and the Plot Selections button changes to Update Plot.
  5. To change or update the weight vector data selections, enable or disable the buttons in the Select column and select Update Plot (repeat as required).
  6. To proceed with weight solution calculations, select Continue.
    A dialogue box (below) is displayed, prompting the operator to either cancel or proceed with the solution determination.
  7. Select Cancel to abort the solution determination and return to the Weight Vector Data screen.
  8. To continue with the solution determination, select OK.
    When the solutions and residual values have been calculated, the Weight Vector Data screen is closed and the Residual Vibration Levels screen is displayed.
  9. To return to the Imbalance Vector Data screen, select Previous Screen.
  10. To close Weight Vector Data screen, select Exit.
    When the report has been printed or the balance session terminated, the Welcome to the Trim Balance Application screen is displayed.

Residual Vibration Levels Screen

The Residual Vibration Levels Screen (below) allows the operator to:

  • adjust the Fan / LPT weight solutions
  • display plots of Fan / LPT vectors along with MDS and User Solution weight vectors
  • display plots of predicted residual vibration levels for Fan / LPT for MDS and User Solutions
  • print the residual vibration level screens

The Residual Vibration Levels screen displays the initial MDS Trim Balance solution in the MDS Solution panel.

  1. To try another weight solution and see its effect on the residual vibration levels, select Adjust Values.
    A User Solution panel is opened on the Residual Vibration Levels screen (below).
  2. Enter the weight magnitude in the User Solution panel and press the Tab key or the Enter key on the keyboard.
  3. Enter the angle (in degrees) in the User Solution Angle field and press the Tab key or the Enter key on the keyboard.
  4. When the magnitude and angle have been entered, a User Solution box is added to the Residual Vibration Levels screen (below) and User Residual column is added to the Plane Values table.
  5. To modify the User Solution and recalculate and view the associated residual vibration levels, select Adjust Values and go to Step 2.
  6. To view the plots of the residual vibration levels, select See Plots. The Residual Vibration Levels screen is closed and the View Plots Residual Vibration Levels screen is displayed.
  7. To get the weight locations, select Get Weight Locations. The Residual Vibration Levels screen is closed and the Balance Weights Position screen is displayed.
  8. To print the screen, select Print Screen.
  9. To close Residual Vibration Levels screen, select Exit.
    When the report has been printed or the balance session terminated, the Welcome to the Trim Balance Application screen is displayed.

Viewing Plots of the Residual Vibration Levels

Balance Weight Positions Screen

The Residual Vibration Levels screen allows the operator to choose between proceeding to calculate an MDS weight solution or a user weight solution.

The Residual Vibration Levels screen (below) is displayed when the solutions and residual values have been calculated (see Weight Vector Data procedure, Step 8).

  1. From the Residual Vibration Levels screen (above), select Get Weight Locations.
    The Residual Vibration Levels screen (below), with a Select Solution slider switch and Reconsider and Continue buttons, is displayed. The Adjust Values, See Plots, and Print Screen buttons are disabled.
  2. To change the User Solution, select Reconsider. The initial Residual Vibration Levels screen is displayed.
  3. Select the solution (Operator or MDS) by clicking in the Select Solution slider bar field.
  4. Select Continue. If the initial weight is unknown the Balance Plane Configuration screen is displayed.
  5. Select Update. A new Balance Plane Configuration screen is displayed.
  6. Add weights to the Initial Configuration. When the Initial Configuration list has been completed, select Continue. The following Balance Plane Configuration screen is displayed.
  7. If the Initial Configuration is acceptable, select OK.
  8. When the new weight locations have been determined, the Balance Weight Positions screen (below) is displayed.
  9. Select the sum values (YES or NO) by clicking in the Sum Values slider bar field (e.g.: YES). If Yes is selected, the new weights are summed with the current weight and a new final weight is displayed. If No is selected, the new weight is installed and the current weight remains installed.
  10. Select Final Solution.
    The Balance Weight Positions screen (below) is displayed.
  11. To close Balance Weight Positions screen, select Exit.
    When the report has been printed or the balance session terminated, the Welcome to the Trim Balance Application screen is displayed.

Balance Plane Re-Configuration Screen

The Balance Plane Re-Configuration screen (below) can be accessed by selecting:

  • the Continue button on the Balance Weight Positions screen
  • the Update button on the Balance Plane Configuration screen
  • the Continue button on the Balance Plane Re-Configuration screen

To reconfigure the balance plane:

  1. To clear the weights for the Current Configuration panel, select Clear.
    The weights in the current configuration box are cleared and the Clear button changes to a Revert button.
  2. To restore the weights in the current configuration box, select Revert.
    The information is restored to the Current Configuration panel and the Revert button changes to a Clear button.
  3. To add a weight to the Current Configuration panel, select the location and part number from the Location and Part Number lists and select .
    Or select the location and select if the part number is not available to install a weight. For example, if the weight position is occupied by a shop weight that cannot be removed during balancing.
  4. To remove an item from the Current Configuration panel, select the corresponding blade number from the Location panel and select .
  5. When the current configuration list has been completed, select Continue.
    The Re-configuration Completed dialogue box is displayed.
  6. To close the Re-configuration Completed dialogue box, without applying the part list to the new configuration, select No and go to the Balance Plane Re-Configuration screen.
  7. To apply the part list to the new configuration, select Yes.
    The Re-configuration Completed dialogue box is closed and the Balance Plane Re-Configuration screen (below), showing the current (new) configuration, is displayed.
  8. To update the configuration and begin a new test, select Update and go to the initial Balance Plane Re-Configuration screen.
  9. To continue, select Continue.
    The Engine Configuration dialogue box is displayed.
  10. Select No to close the Engine Configuration dialogue box and return to the Balance Plane Re-Configuration screen.
  11. To update the configuration and begin the next engine test, select Yes.
    A series of summed solutions are calculated. When the calculations are completed, the Balance Plane Re-Configuration screen is closed and the Vibration Measurement screen is displayed.
  12. Begin collecting data. If the vibration levels are excessive, that is approaching or exceeding the safe operating limit, abort the data collecting process by selecting Previous Screen. The Balance Plane Re-Configuration screen is displayed.

    Using the Previous Screen button from the Vibration Measurement screen can only be used to reposition the first weight solution. Once subsequent weight solutions are applied using the Balance Plane Re-Configuration screen, they cannot be repositioned. A new solution will have to be calculated and applied based on a newly acquired data set.

  13. Move the weights the equivalent of 90 or 180 degrees in order to decrease vibration levels to the point where a second set of vibration data can be safely obtained thereby allowing the Trim Balance System to calculate a new solution. Select Continue. The Vibration Measurement screen is displayed.
    If vibration levels allow for data to be safely recorded, see Start Acquisition of Vibration Measurement Data to record new data and generate a new balance solution. If the vibration levels are still excessive, select Previous Screen and reposition the weight to another location. Repeat the process until the vibration levels are adequate.

Core Balancing

To begin a Core balance session:

  1. Start the Trim Balance System. For details, see Starting the Trim Balance System.
  2. From the Welcome to the Trim Balance Application screen, select Balance Engine. A "Please select..." window is displayed.
  3. From the "Please select FAN Balance or Core Balance" panel, select Core.
  4. From the "Do you want to continue previous Core Balance?" panel, select No.

    This message is only displayed if Core Trim Balance for the current engine serial number had previously been initiated.

    The Balance Plane Configuration screen is displayed.
  5. If the location of the core weights are Known, select Update and proceed to Step 26.
    If the location of the core weights is Unknown, leave the Initial Configuration as Unknown and select OK.

    Any RESTRICTED parts are used to make the locations permanently unavailable since they are adjacent to the locking blade.

    For details on updating the Initial Configuration with known weights, see Balance Plane Configuration Screen before proceeding to the next step.

    The Limits and Tolerances screen is displayed.
  6. Set the desired limits and tolerances and select Continue.

    For details on setting limits and tolerances, see Limits and Tolerances Screen.

    The Vibration Measurement screen is displayed.
  7. Acquire all required target speeds.

    For details on acquiring target speeds, see Vibration Measurement Screen.

    When all target speeds have been acquired, Restart Acquisition, Engine Balanced and Continue buttons are displayed.
  8. To restart data acquisition, select Restart Acquisition. Repeat Step 7 and Continue acquiring data until sufficient data is available before proceeding to the next step.
  9. To accept the engine balance, select Engine Balanced. Go to Step 46.
  10. To view the imbalance vector data and progress towards calculating a new balance solution, select Continue.
    The Imbalance Vector Data screen is displayed.

    For details on Imbalance Vector Data, see Imbalance Vector Data Screen.

    If the engine is balanced go to step Step 46.
  11. If the imbalance vector data is acceptable, select Continue. A dialogue box is displayed, prompting the operator to either cancel or proceed with the solution determination.
  12. To continue with the solution determination, select OK. The Weight Vector Data Screen is displayed.

    For details on Weight Vector Data, see Weight Vector Data Screen.

  13. To proceed with the weight solution calculations, select Continue. A dialogue box is displayed, prompting the operator to either cancel or proceed with the solution determination.
  14. To continue with the solution determination, select OK. The Residual Vibration Levels Screen is displayed.

    For details on Residual Vibration Levels, see Residual Vibration Levels Screen.

  15. Select Get Weight Locations. The Balance Weight Positions screen is displayed.
    1. If the initial weight has not been input, proceed to the next step.
    2. If the initial weight has been input, proceed to Step 30. The Balance Weight Positions screen is displayed.
  16. Select Exit. A confirmation dialogue box is displayed.
  17. Select Yes. The Welcome to the Trim Balance Application screen is displayed.
  18. Close the Trim Balance Application, select Exit.
  19. Determine the current weight configuration on the engine.
  20. Restart the Trim Balance Application.
  21. From the Welcome to the Trim Balance Application screen, select Balance Engine.
  22. From the "Please select FAN Balance or Core Balance" panel, select Core.
  23. From the "Do you want to continue previous Core Balance?" panel, select Yes. The Please select file to load window is displayed.

    The load window is only displayed if Core Trim Balance for the current engine serial number has been initiated more than once.

  24. From the list of available data, choose the desired file and select OK. The Balance Weight Positions screen is displayed.
  25. Select Update Configuration. The Balance Plane Configuration screen is displayed.
  26. Select Update. A new Balance Plane Configuration screen is displayed.
  27. To add the current weight configurations, select Add Weights. A new Balance Plane Configuration screen is displayed.

    Use the X button to indicate positions that are unavailable.

Main TBS Configuration File (tbconfig.ini)

File Path: proDAS\ini

The tbconfig.ini file contains the generic configuration information required to start the trim balance application. The text file is read at start-up and contains details pertinent to the functioning of the program.

Click here for a typical configuration file (tbconfig.ini).

Lines beginning with an asterisk (*) are the remarks added to assist in file creation and verification.

The above file is demarcated by the heading keys within square brackets. The file information identifies:

[Mode]

proDAS: the mode of operation (0 is for demonstration purposes, 1 for proDAS with Endevco and 2 for proDAS with CEC).

[APP]

RTEHost: the RTEHost computer name.
RTEPort: the parameter in the UI_SERVER section of the RTE initialisation file.
DSN: the Database Source name.
DSNConStr: the DSN Connection String (User Name and Password).

[Trace]

FileName: the file name and location.

SQLTrace: type of tracing, with 0 indicating tracing to a trace file using the COM utility and 1 indicating to trace to the VisualStudio output pane.
SQLTraceFile: the tracefile name and location.

[Supported Eng]

Number: the number of supported engines should equal the number of types specified.
Typen: engine names must be identical with those specified in the proDAS database.

[Setup]

Limits: a Boolean setting that permits or prohibits setting the HiHi and Hi limits of the vibration transducer(s) in the Test Configuration Panel. Limits can only be reset if this switch is set to True and if the security level is greater than Operator level.

[Logs]

The log file that should be recorded during and after a balancing session.

The logs that should be recorded after a balancing session, whether real-time report generation is enabled (normally set to TRUE), and an overwrite engine information option (i.e.: update balance weight components and the location of each balance weight component in the engine configuration file for that serial number - normally set to TRUE).

Event: <yyyy-mm-dd-hh-mm-ss>.log trace file for the TB Application.
Data: <yyyy-mm-dd-hh-mm-ss>.log.datn file used to track imbalance data, calculation results and installed weights.
Imbal: <yyyy-mm-dd-hh-mm-ss>.log.imbaln file logs imbalance vector data and total installed effective weight.
Datlog: <yyyy-mm-dd-hh-mm-ss>.log.datlogn file contains all proDAS log files used to acquire TB data.
Report: real-time report generation is enabled (normally set to TRUE).
Overwrite_Config: update the update balance weight components and the location of each balance weight component in the engine configuration file for that serial number - normally set to TRUE.

[VISA]

Addn: GPIB address of Endevco chassis.

[GPIB]

Speed: the speed of the GPIB/E-NET switch. Use 100 only for the 100 Mbps switch, and either 10 or 100 for the 10Mbps switch.
Wait: the delay in milliseconds given before the GPIB status byte is read following a GPIB send command (default setting is 500).

Engine Specifics File (specifics.ini)

File Path: proDAS\Data\Trimbal\engines\engine name (see the figure below)

The specifics.ini file contains engine specific information required to configure the Vibration System, the names of the channels to be collected, and the statistical quality limits for data sample inclusion/exclusion.

Click here for a typical engine specific configuration file for Fan/LPT Balancing.

Click here for a typical engine specific configuration file for Fan/Core Balancing.

The above files are demarcated by the Heading keys within square brackets. The file information provides:

[AVM]

command sequences used to configure the Endevco.

CmdNum: number of configuration commands to be sent to the Endevco chassis.

Cmdn: the command sets follow in sequence and are preceded by either a 0 or 1 in order to differentiate which GPIB device is being addressed.

Statn: commands to check the status of the tracking filters

Respn: expected response to each Statn command.

A period (.) designates a wildcard match for comparing the status and expected strings. The command sequence should be carefully examined to ensure that the correct tracking filter channel is being checked against an appropriate response.

[Channels] and [ChannelsCore]

Number: the number of channels to be processed by TB Application.

Ch0 to Ch6: the names of the channels providing the speed, vibration, and phase information from the channel registry. Channel names must be an exact match with those in the registry and must be presented in order: tach input, vibration channel 1, phase channel 1, vibration channel 2, phase channel 2, vibration channel 3, and phase channel 3.

[Stats] and [StatsCore]

the red and yellow data quality thresholds used for data acquisition quality indices. For speed, vibration, and phase factors, a maximum level is specified (FxMax=), followed by the red (FxRLim=) and yellow (FxYLim=) limits. P1P2SumAvg and ValAvgWts are measures of how closely weight solution vectors are grouped. P1P2SumAvg and ValAvgWts are used for first and second level outlier determination respectively. Only red and yellow limits are required for each of these parameters.

[Threshold] and [ThresholdCore]

VibLimn: the threshold limit under which the vibration level is considered too low for the AVM to accurately provide phase data. If vibration is less than this limit, and data quality is bad, vibration and phase are set to 0 for the subsequent analyses.

[Coefficients] and [CoefficientsCore]

CPFKeepn: the keep factor for coefficients computation. This value is used, as threshold, to remove some phase lag and sensitivity values form computation.

The specifics.ini file is an engine dependent file. If the AVM tracking filter mappings and/or the reference tachometer input is altered, this file must be changed accordingly.

New Engine Template File

File Path: proDAS\Data\Trimbal\engines\engine name

The New Engine Template file is used as a template to establish the basic engine information when a new engine is encountered. The Trim Balance System copies the template data into the appropriate engine directory and renames the file according to the serial number.

Click here for a sample file for a new engine (Fan/LPT Balancing).

Click here for a sample file for a new engine (Fan/Core Balancing).

The new file initially contains known configuration information that will not vary with engine balancing. Fields that change as a result of a balancing attempt (i.e.: those left blank in the above Sample new Files) will be added by the software at a later stage. The required fields are:

[Engine Details]

Name and Planes: the engine name and the number of balance planes.
RotationFLA: the direction of rotation
CoreBalance: True = Fan or Core, False = Fan or LPT
NoPosn: the number of balance weight locations. For this and the following 3 commands, use (n=0) for the Fan and(n=1)for the LPT.
Anglen: not currently used, set to 0.
CurrWtn: not currently used, set to 0.
MaxWtn: maximum allowable total installed weight.
100%RPM: the speed that corresponds to 100% rpm (left blank if speed is measured and displayed in rpm).

[Transducer Info]

Number, TnName, TnHiHi and TnHi: the number of vibration transducers and the respective names of the transducers, as well as the Hi Hi and Hi Limits (where n is the index of the transducer).

[Tracking Speeds] and [Core Tracking Speeds]

the number and values of speeds used for balancing.

[SpdFact] and [Core SpdFact]

weighting factor assigned to each balance speed. The sum of these factors must be 1.

[PUFact]

weighting factor assigned to each pickup (transducer). Values are specified for Fan and LPT balancing. Factors are grouped according to the transducers used for balancing: Factors 0 & 1 are used when balancing using TXs (transducers) 1 & 2, factors 2 & 3 are used for TXs 1 & 3, factors 3 & 4 are used for TXs 2 & 3, and factors 6, 7 & 8 are used for TXs 1, 2 and 3. The sum of these factors in each group must equal 1.

[Sensitivity]

x<plane>Sensy: sensitivity is the vibration response of the engine to one unit of applied weight. Sensitivities are of this form where plane is Fan or LPT, x (0 to 2) is the index of the transducer, and y is the index of the balance speed. For example, 0FanSens0 would represent the sensitivity of the fan at the first balancing speed for the first transducer.
x<plane>Plagy: Phase Lag is the difference between the required angular position of the balance weight and the Phase of the measured vibration. Plane and index representations are as above for Sens.

[Fan Weights]

the number of Fan weights, their part numbers, pan and effective weights.

[LPT Weights]

the number of LPT weights, their part numbers, pan and effective weights.

[Fan Config]

the number and part numbers and location of installed Fan weights to be used for the start of the current balance session.

[LPT Config]

the number and part numbers and location of installed LPT weights to be used for the start of the current balance session.

[Units & Tables] and [Core Units & Tables]

Weights & Vibs: engineering units of the effective weight and vibration.
Tables: a Boolean setting indicating if there are weight solution lookup tables available.

The TB Application assumes weight locations are evenly distributed around the balance plane. Some engines use some of these weight locations for bolts used to retain the Fan spinner. If this is the case, or if some of the holes are permanently unavailable by design, these locations are annotated in the Fan plane configuration section as: Part#n="Spinner Hole" at the appropriate Locn.

The new file is an engine dependent file. Improper modifications to this file will impair software functioning. Be careful when editing and saving changes to this file.

Recent Template File (template.rct.txt)

File Path: proDAS\Data\Trimbal\coefficients.

When a recent file is not found in the proDAS\Data\Trimbal\coefficients directory for the current engine type and the user level is greater than that of operator, this file (template.rct.txt) is used as a template for generating the engine name.rct.txt information. The software copies the template to a properly named file in the requisite folder, opens the new file with Notepad, allowing the user to enter the data in pre-specified fields. After the data has been entered, the user may then save the data (the file length is checked against the starting value) or exit Notepad without saving. If no additional information is added to the file or the user exits Notepad without saving, then the trim balance session will be aborted.

Demonstration/Test Data Sets

File Path: proDAS\Data\Trimbal\demo data\name (see below).

The folder names under demo data folder are the engine names that are available for demo mode. When Trim Balance is started, a demo panel is displayed, allowing the user to select the engine type. The selected engine name should match the engine specified in the Demo section of the tbconfig.ini file.

The naming convention for the files is speedxxxx.y, where xxxx represents a specific balance speed in revolutions-per-minute (RPM) and y represents the current iteration of the balance session.

Each file contains tab-delineated speed, amplitude, and phase information for each tracking speed. The format for each line of an iteration file is: time, actual speed, and the vibration amplitudes and phase angles for each transducer.

During execution of trim balance in the demo mode, if a speed file does not exist, then the operator must select skip during data acquisition.

A dialogue box is displayed and the operator can select which data-set to use during the balancing session. The operator can update the coefficient database and recent files while in the demonstration mode.
