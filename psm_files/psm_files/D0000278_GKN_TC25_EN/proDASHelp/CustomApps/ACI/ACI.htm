Automated CITS Import into proDAS (Option*)

Overview

The Automated CITS Import into proDAS (ACI) utility converts CITS configuration files into the format compatible for CIE input. The ACI utility directly updates the proDAS subsystem configuration via the Configuration Server interface. The ACI utility has an option to launch the CIE allowing configuration files to be imported into proDAS.

ACI is typically used as a stand-alone application prior to configuring proDAS.

The ACI utility operations are separated into the following four windows:

  • Select Source
  • Search and Replace
  • Update proDAS Configuration
  • Run CIE

Starting the ACI

The ACI can be started either as a stand-alone application or from the Management GUI Control Panel.

To start the ACI utility as a stand-alone application:

  1. Double-click the ACI executable file from the Management Interface Computer directory structure.
    The Import CITS into proDAS Utility window is displayed.

After a successful login, the ACI tries to lock the Config Server for the writing operation. If the lock operation is not possible, ACI displays a "Retry/Abort" message. The user must exit from any applications that has acquired the write lock (i.e. Channel Editor or Subsystem Editor).

To start the ACI utility from the Management GUI Control Panel:

  1. From the Management GUI Control Panel (below), select Tools / Configuration Editors / Automated Configuration Import.

Importing CITS Configuration Files

To import CITS configuration files:

  1. Start the ACI utility. The Import CITS into proDAS Utility window is displayed.
  2. Select between unzipping a source zip file or browsing to an existing configuration folder that needs to be converted.
  3. To select a Source Zip file, select Browse.... and then Unzip. The ACI unzips the contents of the file into a newly created folder named "\ENG_[engine_number]\[DateTime stamp]".
  4. To select an existing CITS folder, select Browse....
  5. Once a configuration has been selected, select Next >. The Search and Replace window is displayed.

    The sub-steps performed on the Search and Replace window are marked OK (complete) or ERR (indicates problems). The results are reported on the final page of the ACI.

    EU tab:
    The EU tab displays new unit conversions (ACI has not previously remembered). If the default selection is incorrect, select a different unit conversion. After selecting the unit conversion, select Remember. The next time ACI encounters the same CITS unit, it automatically makes the replacement.
    Select Process Data.

    CEC / AVM tab:
    The CEC / AVM tab provides the operator the option to Use Merged AVM / CEC subsystem data or Use Default CEC File.

    If the AVM and CEC subsystems are already merged (for the selected source) then the option "No Processing" is available instead of "Use Merged AVM / CED subsystem data".

    If for the selected source the AVM and CEC Subsystems are already merged then No Processing option is available.
    If the user has had to perform selections on the panels (i.e. EU processing) the ACI utility will not automatically continue until the user selects Process Data.

    The Search and Replace Steps are processed and an OK is placed beside each successful operation.

    The processing results table displays the outcome of each processed step.
    The Data integrity specifies whether the subsystems in the selected source match the schema.
    The EU processing specifies whether the units are proDAS known units or need to be specified.
    The CTLog.def processing specifies whether the CTLog channels defined in the ini file have been successfully added to the CTLog.def file under the LOGDEF folder.
    The AVM / CEC Processing indicates the status of the AVM and CEC subsystems merge (provided it happened).
    The Overall processing displays a summary of all the above processing steps.

  6. When all of the steps have been processed, select Next >. The Update proDAS Configuration window is displayed.

    The sub-steps performed on the Update proDAS Configuration page are marked complete with an OK or ERR (indicates problems). The results are reported on the final page of the ACI.

    SCUTR tab:
    The SCUTR tab displays new SCUTR serial numbers that ACI has not remembered. The serial numbers are automatically updated. Once updated, the Result column displays Created.

    ARINC tab:
    The ARINC tab displays new ARINC pinouts that ACI has not remembered. The pinouts are automatically updated. Once updated, the Result column displays Created.

    PBS tab:
    The PBS tab displays the new Sensor Name that does not exist in the configuration. The Result column displays Input Required. To update the configuration, select Run. The Result column changes to Created and the PBS Sensors/Scanners status is changed to OK.
    TBS tab:
    The TBS tab displays the new Serial Number that does not exist in the configuration. The Result column displays Input Required. To update the configuration, select Run. The Result column changes to Created and the TBS Serial Numbers status is changed to OK.
  7. When all the rules are applied, select Next >. The Run CIE window is displayed.

    Prior to running CIE, the ACI provides a status log which records successfully processed subsystem files as well as failed files.

    Log tab:
    The Log tab displays a list of items that have been modified.

    Filter tab:
    The Filter tab provides the operator the ability to filter the logs by Subsystem, Area, Operation, Success, and Level.
    Reports tab:
    The Reports tab generates reports of the processed subsystem files. The report types include All Log Entries, Modified and Deleted data, and Invalid Units.


  8. To run the Configuration Import/Export utility, select Finish. A ‘Starting ConfImpEx’ dos prompt is displayed.

    The Finish button is enabled only if all the previous steps have been completed successfully.

    To navigate back to the previous step to make any additional changes, select "< Back". CIE will not be launched.

    To start over, select "<< Start". CIE will not be launched.

    The CIE Import window is displayed indicating Import successful.
  9. Select OK to exit the ACI.
