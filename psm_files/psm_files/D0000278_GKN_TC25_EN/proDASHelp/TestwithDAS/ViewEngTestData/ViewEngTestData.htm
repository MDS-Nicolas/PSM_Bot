Viewing Engine Test Data

Overview

proDAS enables engine test data to be displayed, printed, and played back through the use of the Reporting and Viewing Environment (RAVE) application, the Time On Condition and Engine Usage Monitoring (TOCEUM) Subsystem interface, and the Replay Mode. The RAVE and TOCEUM applications can be accessed from the proDAS Management GUI Control Panel. The Replay Control GUI is launched automatically from the Management GUI Control Panel when proDAS is configured in Replay Mode.

Reporting and Viewing Environment Application (RAVE)

The Reporting and Viewing Environment (RAVE) is an application used to view data recorded during a test and to create reports.

The engine test data, collected by proDAS, consists of:

  • fullsets and event comments (saved in the proDAS database)
  • Critical, Transient, and Continuous Logs (stored in the log directories on the Host Computer)

The Critical, Transient, and Continuous logs require conversion from binary to ASCII text for report viewing, analysis or printing purposes.

The RAVE application consists of three main screens:

  • the Test Selection screen
  • the Reports screen
  • the Data screen

Test Selection Screen

The Test Selection screen (displayed below) is launched after login. The user can retrieve any combination of engine tests using the query builder tool. The range of retrieved data is refined by execution of a single query defined as a set of selected search parameters. Typically, data is retrieved by selecting predefined (already saved) queries from a drop-down list. A query cannot locate a historical test or group of tests that have been archived.

Test Selection Screen
Buttons, Panels and Fields Function
Predefined Queries The Predefined Queries panel retrieves engine test data based on queries that have already been defined and saved. Queries retrieve specific test data by using a combination of search criteria such as test cell, engine type, engine serial number, test name and date.
Query controls The buttons in the Query controls panel execute, save and display query data.
  • Execute - retrieves engine tests based upon parameters in the QUERY BUILDER panels
  • Save series - saves QUERY BUILDER data in an existing Category folder. A new Category folder may be created if desired
  • Clear all - clears data within the QUERY BUILDER panels
  • Reports - displays the Reports screen for the engine test highlighted in the Query Results panel
  • SHOW/HIDE QUERY BUILDER - displays or closes the QUERY BUILDER area
    (for detailed information on the QUERY BUILDER, see Creating New Queries with the Query Builder Panel in Building Reports with RAVE)
  • Replace - overwrites engine tests in the Query Results panel with data that is retrieved from executing a new query
  • Add - accumulates engine test in the field
  • scroll buttons (<< < > >>) are used to display a number of queries that are currently in use (i.e.: a query series)
  • Data - displays the associate Fullsets for the engine test that is highlighted in the Query Results panel
QUERY BUILDER The QUERY BUILDER panel is subdivided into Standard Test Criteria and Custom Test Criteria sections. There are five fields and five buttons within the Standard Test Criteria section for selecting data for test cell, engine type, engine serial number, test name and test date. The Custom Test Criteria section has twelve data fields and six buttons for retrieving customer specific data.
Query Results The Query Results panel displays the retrieved data under the header bars:
  • Test Cell
  • Test Date
  • Test Name
  • Test Description
  • Engine
  • Serial Number
  • Engine Standard
  • Customer
The header bars within this field have similar functions to standard spreadsheet headers. Columns may be dragged laterally and placed in any order from left to right, and selecting a header bar will sort the corresponding data in ascending or descending order.
Session history The Session History panel displays user related messages during the query session. All messages are stored in (and may be retrieved from) an RTE trace file. The Session history panel may be configured at the administrator level to display only operator messages or to include error and debugging messages to suit user requirements.

The Clear button removes the entry displayed in the Session History panel.
Exit The Exit button closes the Test Selection screen and exits RAVE.

Reports Screen

The REPORTS button in the QUERY CONTROLS panel of the the Test Selection screen displays the Reports screen (see below). The specific engine test that was highlighted in the QUERY RESULTS panel of the Test Selection screen is included in the Reports Screen. Report templates are selected by the user and populated by RAVE. The reports are produced in Microsoft Excel. Typical reports are:

  • Logsheets (containing fullsets (steady state data) and comments)
  • Transient Data Reports (depicting transient logs)
  • Fullset Reports (depicting steady state data)
  • Combination Reports (depicting both steady state and transient data)
Reports Screen
Buttons, Panels and Fields Function
Currently selected tests The Currently selected tests panel displays the engine test that was highlighted in the Query Results panel of the Test Selection screen.
Single engine reports templates. The Single engine reports templates. panel contains names of previously created reports (in .xls spreadsheet form).
REPORT TEMPLATE CONTENTS: The REPORT TEMPLATE CONTENTS: panel displays the .ini configuration file of the selected Single engine reports template. The file defines the report contents, such as channels and test step names, limit lines and comments.
Report controls The Report controls buttons and functions are:
  • Select All - highlight the report templates within the Single Engine Report Templates panel
  • Unselect All - deselect the report templates within the Single Engine Report Templates panel
  • Report Builder - launches the Report Builder wizard used to create new report templates
  • Peek - displays the .xls (Excel workbook) file based on the selected report template populated with the last test data that was deposited into the template

    Peek is only available when logged in as an Administrator level user.

  • View - generates and displays an .xls report based on the selected report template and data extracted from the TRSCDB for the given engine test
  • Print - prints the selected report in the form of an Excel workbook
Session history The Session history panel displays user related messages during the query session. All messages are stored in (and may be retrieved from) the RAVE trace file. The Session history panel may be configured by the administrator to display only operator messages or to include error and troubleshooting messages to suit the user's requirements.

The Clear button removes the entry displayed in the Session history panel.
Back The Back button closes the Reports screen and displays the Test Selection screen.

Data Screen

The Data button on the Test Selection screen displays the Data screen. The Data screen allows the user to view data from the selected test.

Starting the Reporting and Viewing Environment (RAVE)

To start RAVE:

  1. From the Management GUI Control Panel (below) Tools menu, select Post Test Functions / Real-Time Report or RAVE.

    If RAVE is selected the RTReports screen is not displayed.

    The RTReports screen is displayed.
  2. Select from the RTReports screen.
    The Test Selection screen is displayed.

Retrieving Engine Tests using Saved or Predefined Queries

The user can retrieve any combination of engine tests with the screen query tools. The tools may execute a single query with a set of selected search parameters to refine the range of retrieved data. Most of the time, data is retrieved by selecting predefined (already saved) queries from a drop-down list. A query cannot locate a historical test or group of tests that have been archived to tape.

  1. Select a previously saved category from the Query and Report Categories drop-down list in the Predefined Queries field.
  2. Select the desired query from the Current queries drop-down list in the Predefined Queries field.
  3. Select Execute in the Query controls field to execute the query and retrieve the engine tests based on the search criteria defined in the query.

    The Clear all button in the Query controls field deletes both unsaved queries and query data.

    Engine test data from executed queries is displayed in the Query Results field under the header bars:
    • Test Cell
    • Test Date
    • Test Name
    • Test Description
    • Engine
    • S/N(serial number)
    • Engine Standard
    • Customer
    The header bars in the Query Results field have functions similar to standard spreadsheet headers (i.e.: columns may be dragged laterally and placed in any order from left to right and clicking on a header bar sorts the corresponding data in ascending or descending order).
  4. To add additional engine tests from another query or to replace the current engine tests already queried:
    1. Select another previously saved query from the Query and Report Categories drop-down list in the Predefined Queries field.
    2. Select Replace to overwrite the present engine test in the Query Results field or select Add to retain the present data and to include the additional engine tests reprieved by the new query.
    3. Select Execute.

    The scroll buttons display queries that are temporarily stored in local memory (i.e.: a query series) but have not yet been saved to disk.
  5. Select the desired engine test from the Query Results field.
    For viewing the Fullsets and Events associated with the highlighted engine test, see Viewing Fullsets and Events using RAVE.
    For generating reports from the highlighted engine test, see Generating Reports using RAVE.

Retrieving Engine Tests using Query Controls

From the Predefined Queries panel on the Test Selection screen:

  1. Select a previously saved query (e.g.: ATP_RAVE) from the drop-down list in the Query and Report Categories field.
  2. Select the desired query (e.g.: atp_rave1) from the drop-down list within the Current queries field.
  3. Select Execute from the Query controls panel to retrieve engine tests that are associated with the desired query.
  4. Select the desired engine test (highlighted) listed in the Query Results panel.
  5. Select Reports from the Query controls panel to open the Reports screen.
    The Reports screen is displayed.

Viewing Fullsets and Events Using RAVE

  1. Select a previously saved query from the Query and Report Categories drop-down list in the Predefined Queries field.
  2. Select the desired query from the Current queries drop-down list in the Predefined Queries field.
  3. Select the Execute button in the Query controls field to retrieve the engine tests associated with the desired query.
  4. Highlight the engine test of interest within the Query Results field and select the Data button in the Query controls field.
    The Data screen, populated with all the fullsets and events for the selected engine test, is displayed.

    The fullsets from the highlighted engine test are displayed in the FULLSETS field under the header bars:
    • Test Cell
    • Test Date
    • Test Name
    • Test Step
    • Fullset Date
    • Fullset Comment
    • Fullset Number

    The events from the highlighted engine test are displayed in the EVENTS field under the header bars:
    • Test Cell
    • Test Date
    • Test Name
    • Test Step
    • Event Date
    • Event Comment
    The header bars in the FULLSETS and EVENTS fields have functions similar to standard spreadsheet headers (i.e.: columns may be dragged laterally and placed in any order from left to right and clicking on a header bar sorts the corresponding data in ascending or descending order).

    There is no PEEK feature for the data in the EVENTS field.

  5. To highlight the last Fullset within the test, select LAST FULLSET.
    To highlight all the Fullsets within the test, select SELECT ALL.
    To highlight specific Fullsets within the test, push the Shift-key or the Ctrl-key and use the mouse to click-on selected Fullsets.
  6. Select the PEEK button to display the selected Fullset data on the Peek Exporting screen.
    The Peek Exporting screen allows the user to preview all of the highlighted Fullsets.

    The Fullset table stores the minimum and maximum channel values, the standard deviation and the number of samples for any channels flagged for fullset stability processing.

  7. Select Export to transfer the previewed Fullsets to a text file that can be saved to the hard disk of the Management Interface Computer. The Save As window is displayed.
  8. From the Save in: drop-down list, select the location to which the text file is to be saved.
  9. Enter the filename for the text file in the File name field.
  10. Select Save to save the file.
    The Save As window is closed and the Peek Exporting screen is displayed.
  11. Select BACK on the Peek Exporting screen to return to the Data screen.

    When SHOW LOGS is selected, the button toggles to SHOW EVENTS.

    The SHOW EVENTS screen allows the user to enter comments to the database using non-English characters.


    The logs are displayed in the LOGS field under the header bars:
    • Test Step
    • Engine Name
    • Test Name
    • Log Name
    • File Path
    • Log Size
    • Log Date
  12. Select BACK on the Data screen to return to the Test Selection screen.
  13. Select Exit on the Test Selection screen to close the RAVE application and return to the Management GUI Control Panel.

Editing and Inserting Test Diary Events

Test Diary events can be added (inserted) or edited within the Events Panel of the Data screen.

To edit or add events in the Test Diary:

  1. From the QUERY RESULTS panel, highlight the required test data (i.e.: ATP_ENGINE).

  2. From the Query Controls panel select Data. The Data screen is displayed with a FULLSETS and EVENTS panel.

Editing Events

The Event Comment text, test step identifier and event date (timestamp) of an operator generated event message can be edited. These event messages are displayed in a blue font. Events that have been edited via RAVE are identified by a blue font and an ***EDITED*** prefix in the Event Comment column. Events that have been added via RAVE are identified by a blue font and an ***ADDED*** prefix in the Event Comment column.

To edit an event:

  1. Right-click on an event in the EVENTS panel. An edit context menu is displayed.
  2. Select the required action from the edit menu:
    1. Select Edit Test Step from the edit menu to update the Test Step text of the event. The EDIT TEST STEP window is displayed.
      Enter the test step text in the field and select OK. The Test Step of the event is updated.
    2. Select Edit Timestamp from the edit menu to update the Test Date text of the event. The EDIT TIMESTAMP window is displayed.
      Enter the timestamp in the field and select OK. The Event Date of the event is updated.
    3. Select Edit User Comment from the edit menu to update the Event Comment text of the event.
      Enter the user comment in the field and select OK. The Event Comment of the event is updated.

Inserting a New Event

The insertion point of a new event relative to existing events is determined by the Event Date information. An inserted event is identified by a blue font and an ***ADDED*** prefix in the Event Comment column.

To insert a a new event:

  1. Right-click on an event in the in the EVENTS panel. An Insert New Event context menu item is displayed ().

    The EVENTS section can display comments containing non-English characters.

  2. Select Insert New Event. An INSERT NEW EVENT window for the event timestamp is displayed.
    Enter the timestamp in the field.

    This field supports non-English characters.

    Select OK to create a timestamp for the new event. An INSERT NEW EVENT window for the event user comment is displayed.
  3. Enter the user comment in the field.

    This field supports non-English characters.

    Select OK to create a user comment for the new event. An INSERT NEW EVENT window for the event test step is displayed.
  4. Enter the test step in the field.

    Do not exceed 20 characters in the field. Comments that are longer than 80 characters can be added or edited, however multiple event messages are generated due to the 80 character limit per Event comment.

    This field supports non-English characters.

    Select OK to create a test step for the new event.

Viewing Log Data

To view log data:

  1. From the QUERY RESULTS panel, highlight the required test data (i.e.: ATP_ENGINE).
  2. From the Query Controls panel select Data. The Data screen is displayed.

  3. Highlight one log from the panel and select PREVIEW.
    The Log Preview screen is displayed.
  4. From the Tools tab, select up to ten channels from the Available Channels panel and move them to the Selected Channels panel by selecting .
  5. Move the Log Data slider to retrieve up to 32,000 scans of log data.

    The Log Data slider is enabled only when there are more than 32,000 scans of data within the log.

  6. Select (Fetch) to retrieve the selected channel data from the log. The data is loaded into Rave.
  7. Select the required check boxes in the Selected Channels field.
  8. Select the Relative or Absolute scale radio button to modify the amplitude (y-axis) and time scale (x-axis) of the graph.
  9. Move the Start Time and End Time sliders to select a specific time frame within the log.
  10. Select (Graph) to display data from the selected channels in graphical form.
  11. Select the test from the Events drop-down list to center the time range of the data around the selected event.

    This feature is only enabled when previewing a Continuous Log.

  12. Select (Quality) to generate a Log Quality Summary Report.
    The Log Quality Summary Report is displayed.

    The Log Quality Summary Report may also be displayed with the LOG QUALITY button on the Data screen.

  13. Select (Fullset).
    The Generate Fullset window is displayed.
  14. Select a test step from the TEST STEP drop-down list or enter a new test step title within the TEST STEP field.
  15. If desired, enter a comment within the COMMENT field to overwrite the default comment.
  16. Select (Fullset) to generate the fullset of data, and transmit the fullset to the database.
    A confirmation window is displayed. Select Yes, and a second confirmation box is displayed. Select OK.
  17. Select the Data tab on the Log Preview screen to view all the data that has been retrieved and graphed by RAVE.
    The Chart Data and Log Data fields are displayed.
  18. To export the graphical data into a tab-delimited file (which can be viewed in Excel), select the Export tab. The Available Channels, Selected Channels and Export fields are displayed.
  19. Select the desired channels from the Available Channels field with the arrow buttons .
  20. Select (Fetch) to retrieve data for the Selected Channels. The data that is displayed within the Export field is defined by the chart time frame.

    There is a limit on the amount of data that can be fetched. This limit is based on the number of channels selected and scans per channel that were recorded.

    There is a RAVE ini parameter that controls how much memory/disk space is allocated to fetch channel data. The following parameter value would need to be increased in order to allow the fetching of more channel data:

    [GRAPH]
    FetchExportLimit=100000000

  21. To save the Export field data to disk, select (Save).
  22. To print the Export field data, select (Print).

Generating ASCII Reports

To generate an ASCII report:

  1. From the Data Screen Logs Panel, highlight a log and select ASCII Report.
    The Select Channels screen is displayed.
  2. Select the required channel(s) from the All Channels list and use the right arrow button to move the desired channel(s) to the Selected Channels list. Use the left arrow button to remove a channel from the Selected Channels list.
  3. Select Browse to specify the location of the saved report.
  4. Select OK to save the ASCII report.
  5. To return to the Data Screen without saving, select Cancel.

Generating Reports using RAVE

A new report may be generated using the REPORT CONTROLS buttons on the Reports screen or an existing report can be populated with the selected engine test data and viewed from the Reports screen.

From the Test Selection screen:

  1. Select a previously saved query from the Query and Report Categories drop-down list in the Predefined Queries field.
  2. Select the desired query from the Current queries drop-down list in the Predefined Queries field.
  3. Select Execute in the Query controls field to retrieve data.
  4. Select the desired engine test from the Query Results field.
  5. Select REPORTS in the Query controls field.
    The Reports screen is displayed.
  6. Highlight the desired template from the Single Engine Report Templates field.
  7. Select View to generate and display the .xls (Excel workbook) file.
    The .xls (Excel workbook) file is based on the Report Templates and is populated with engine test data that has been extracted from the proDAS database.
  8. When View is selected, the Duplicate Comments dialogue box is displayed if the engine test data contains more than one comment with the same TEST_STEP name.
  9. Highlight (select) the comment or comments to be included in the report.

    The button Select All highlights all comments to be included in the report and displays a Select Comments prompt.
    Select OK to generate the report in Excel without further prompting for duplicate comments.

  10. When all the applicable comments have been highlighted, select Continue.
    The Single Engine Report - Duplicate Test Steps Found! dialogue box is displayed for each occurrence of a duplicate TEST_STEP name.

    This can be repeated for all TEST_STEP's that have been defined for the report.

    This occurs for each engine test data.
  11. If duplicate test steps are found, select the test step that is to be included for the currently selected report to be generated.
  12. When the test step has been selected, select Continue.
    The Single Engine Report - Duplicate Test Steps Found! dialogue box is closed and the Report is generated in Excel.

    This is repeated for all instances of Duplicate test steps being found.

  13. To print the generated report in Excel, select File>Print from the Excel toolbar.

    The Logsheets and reports are saved in a default Populated Reports folder within the corresponding Reports directory.

  14. To close the report, select File>Exit from the Excel toolbar. The Reports screen in RAVE is visible.
  15. Select BACK on the Reports screen to return to the Test Selection screen.
  16. Select EXIT on the Test Selection screen to close the RAVE application.

Generating a Log Quality Summary Report

To generate a log quality summary report:

  1. From the Query Results panel, highlight the required test data (e.g. ATP_ENGINE).
  2. From the Query controls panel, select Data.
    The Data (Fullsets) screen is displayed.
  3. Highlight one log (e.g.: Pulse 5) from the panel and select LOG QUALITY.
    The Log Quality Summary Report is displayed.

    The Log Quality Summary Report may also be displayed with the LOG QUALITY button when graphically previewing the transient log data.

Configuring proDAS in Replay Mode

The engine test data, collected and saved by proDAS, consists of:

  • fullsets and comments (saved into the proDAS database on the Host Computer)
  • Critical, Transient, and Event Logs (that are stored in the log directories on the Host Computer)

The Replay Control GUI allows the operator to replay the saved Critical and Transient logs.

While in the Replay mode, the operator can perform the following functions:

  • change operator pages
  • use the Spotlight feature
  • create a report from the loaded log
  • print an operator screen
  • review messages in the Event Handler window

The Replay Control GUI is launched automatically from the Management GUI Control Panel when proDAS is configured in Replay mode.

To launch the Replay Control GUI:

  1. From the Management GUI Control Panel configure proDAS in Replay mode, see Configuring proDAS for an Engine Test.
    The Replay Control GUI is launched automatically when proDAS is configured.

Playing Back Saved Critical or Transient Logs

To select a log for playback:

  1. Select from the Replay Control GUI (below).
    The Open dialogue box (below) is displayed.
  2. From the Look in: drop-down list, select the folder (e.g.: hist_logs) where the log file is located.
  3. Select the log file (e.g.: ATP_LOG.1) from the list and select Open.
    The Open dialogue box is closed.
    The selected log file is sent to the RTE and the location of the selected log file is displayed in the Log File: field of the Replay Control GUI.

    Since the selected log file is not a continuous log, the Skip buttons and Events drop-box are disabled.

  4. For information on using the Replay Control GUI to play back the selected log file, see the Replay Control GUI table.

Creating a Report from the Log Being Played Back

While proDAS is in the Playback mode, the operator can convert the Transient or Critical log, that is being played back into a text file (report), allowing the engine parameters to be analysed using a standard test editor.

To create a report from the log being played back, see Creating a New Report.

To view the the report, see Real Time Reports.

Recalculating Channel Data while in Replay Mode

While in replay mode, the calculated subsystem functions the same as in normal scan mode.

For information and procedures on recalculating channel data, see External Hook Framework Recalculation.
