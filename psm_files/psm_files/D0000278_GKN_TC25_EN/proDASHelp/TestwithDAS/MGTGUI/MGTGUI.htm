Management GUI

Overview

The Management GUI is the main interface between the user and the proDAS system, providing a user-friendly interface to control all aspects of the testing process. From the Management GUI, the Real-Time Engine (RTE) can be configured in either acquisition mode or replay mode. The Management GUI also controls access to all other components of proDAS, such as the configuration and calibration tools.

The Management GUI automatically resets to acquisition mode when it is closed while in replay mode or when proDAS restarts.

When started, the Management GUI connects to the RTE. Anytime a connection to the RTE is established, a configurable start-up script is launched. This script starts the Unified Event Log (UEL) Display. The Management GUI Control Panel is also displayed. It is from this panel that the user is able to configure proDAS.

When the user first logs into the system, prior to the configuration of the RTE, the Management GUI allows the user to:

  • configure proDAS
  • edit the configuration data
  • edit the User Administration settings
  • exit proDAS
  • run post processing software

When the RTE is configured but not scanning, the Management GUI allows the user to:

  • edit the configuration data
  • start the RTE Scanning
  • run post processing software
  • edit the User Administration settings
  • exit proDAS

The RTE will not fully initialise if there is less than the defined amount of disk space available (the defined amount is 250 MB). If less than 250 MB of space is available when the RTE is initialised, a UEL warning message is displayed and the RTE does not fully initialise. The system also checks every five minutes (default) whether there is enough disc space. If there is not, a warning is displayed in the UEL. This automatic checking feature can be disabled in the proDAS initialisation file.

After the RTE is scanning, the Management GUI allows the user to:

  • edit the configuration data
  • open the Test Page
  • run post processing software
  • run other proDAS Tools and Utilities
  • run pre-run checks and calibration functions
  • stop the RTE from scanning
  • edit the User Administration settings
  • access the Test Information Page
  • exit proDAS

The RTE can support a maximum of 50 hardware subsystems and a total of 64 subsystems.

Starting the Management GUI

To start the Management GUI:

  1. Double-click the Management GUI icon.
    The Login window is displayed.

  2. Enter a valid Username and Password.

    The password is case sensitive.

  3. Select Login.
    The Management GUI is started and the Management GUI Control Panel is displayed.

Test Information Page

The Test Information Page (below) is used to store test specific information such as the engine serial number, build number, engine type, engine standard, test cell and test name to the Test Results and Sensor Calibration Database (TRSCDB). It also has the ability to write values to settable channels which can be used for defining the engine model being tested. The Test Information Page is automatically displayed under the following two conditions:

  • if the Management GUI is started after the RTE has already been configured
  • when the Management GUI Control Panel Apply button is selected

From the Management GUI Control Panel, the Test Information Page can be manually accessed by the Tools pull-down menu.

All editable fields in the Test Information Page support non-English characters. The length of the comments in each field is limited to the settings defined in the database.

Default Test Information Page

If a customer specific Test Information Page is not available the Management GUI displays a default Test Information Page (above). The default Test Information Page:

  • when displayed, is populated with any existing data for the current test
  • allows the user to directly update the test information contained in the Test Header table of the Test Results and Sensor Calibration Database (TRSCDB), except for customer, engine serial number, build number, engine type, engine standard, test cell, configuration and test name
  • will not update the Customer Specific Table
  • will not set channels in the CVT (maintained only by customer specific Test Information Pages)
  • allows the user to set/reset the Test Completed flag
  • allows the user to reset all fields to the original data which existed for the current test
  • is displayed if a customer specific Test Information Page returns an error
  • allows the user to apply the information entered by the user

When the Abort button on the default Test Information Page is pressed, the user cannot close the Test Information Page until the RTE configuration is fully aborted.

Customer Specific Test Information Page

If a customer specific Test Information Page is available, the Management GUI ensures that it is displayed and passes to it any required information. The customer specific Test Information Page:

  • (in the case of a new test) is populated with the data from the last existing test with the same engine serial number
  • stores any changes to the test information to the Test Results and Sensor Calibration Database (TRSCDB) in the test header and customer specific tables
  • logs any changes to the test information to the Event Table in the Test Results and Sensor Calibration Database (TRSCDB)
  • allows the user to set channels in the CVT
  • allows the user to apply the information entered by the user

Please refer to the "Home" section of the PSM for details on your current Test Information Page.

Critical Log

The critical log is a rotating log which is started automatically when Start Scan is selected on the Management GUI Control Panel. The critical log records data continuously from all of the channels defined as critical in the current configuration.

The critical log records a rotating data buffer in seconds. The length of the buffer is determined by adding the crit_pre-event and the crit_post_event values, where:

  • crit_pre_event is the maximum length of time that data is recorded before the critical event which triggers the log to be saved. The value for crit_pre_event (in seconds) is defined in the RTE initialisation file
  • crit_post_event is the length of time that data is recorded after the critical event which triggers the log to be saved. The value for crit_post_event (in seconds) is defined in the RTE initialisation file

If proDAS has been monitoring an engine test for less than the crit_pre_event seconds, then only the number of seconds recorded since the start of scanning plus crit_post_event seconds will be saved in the critical log.

Maximum total time in seconds of a critical log is 300 seconds (5 minutes).

A critical log can be saved whenever a record of the activity on all the critical channels is required. When the critical log is saved by the user, the buffered data is saved to the critical log file.

For detailed information on configuring a critical channel, see the Channel Editor.

Critical logs can be saved:

  • by selecting the Save Critical Log button on the Test Page displayed on the Management Interface Computer
  • automatically as an action to an alarm or warning. The alarm conditions and the actions associated with each limit are specified in the individual channel's configuration (for detailed information on adding an action to an alarm, see the Channel Editor)

The log is saved to the hard disk, of the Host Computer, in the file:

/users/RTE/logs/<enginename><testname>/hist_logs/crYYMMDDHHmmss

where:

  • enginename is the name of the engine tested
  • testname is the name of the test on the Test Information Page
  • cr identifies the file as a critical log
  • YYMMDD is the current year, month, and day
  • HHmmss is the hour (0-23), minute, and second at which the log was saved

When a critical log has been saved a message will be written to the UEL.

Manoeuvre

A manoeuvre is defined as a single recording, of specified duration, of dynamic data. During a test, the system operator can opt to record chunks of data (i.e.: manoeuvres) rather than record all of the data. For example, during a two hour period, six separate, eight minute manoeuvres could be recorded and saved to disk or two consecutive manoeuvres could be recorded and saved to disk. Manoeuvres are available for manual post-processing immediately after writing to disk. Manoeuvres are processed automatically, if the system is configured for on-line analysis, immediately after writing to disk.

The system operator can specify the Elapsed Time of an unscheduled manoeuvre (i.e.: in minutes) by entering the time in the Elapsed Time field. The time, in minutes, can be incremented or decremented by using the up and down arrows at the right of the Elapsed Time field. The manoeuvre stops automatically, when the available time is reached, and the system switches to the Monitor mode from the Recording mode. When monitoring resumes, a new background file is created.

The analyst can perform post-processing on the data either during the test or immediately after the test as the manoeuvre data is accessible immediately after the recording of the manoeuvre has been completed and for a set time after the test (until the data has been archived). Post-processing can be carried out on a separate PC at the same time the system is monitoring and recording new data.

Archived and restored tests cannot be resumed.

Save Sentry

The Save Critical Log function shall trigger the saving of the DDS Sentry.

From the Test Page, selecting the Save Critical Log/Save Sentry button starts the save sentry operation. This results in the collection of the background files from all pertinent disk sets as well as the disk set switching. Confirmation of the Save Sentry is displayed in the Test Diary window.

Test Diary Window

The Test Diary Window is used to record the test history information including all data recordings, operator comments and the test header information. The Test Page screen contains a Test Diary tab which lists the most recent 200 Test Diary entries. The entries are listed in chronological order, with the most recent events at the bottom of the list. The Test DiaryWindow refreshes each time the Management GUI writes data to the Test Diary. A Pause button above the Test Diary Window, when selected, pauses the updating of the Test Diary Window enabling the operator to scroll back to review earlier entries. Selecting the Play button, above the Test Diary Window, refreshes the Test Diary Window.

The Management GUI checks to determine if the Test Diary Window needs to be refreshed with the latest database entries every 10 seconds. The Management GUI records the following events to the Test Diary:

  • fullset recorded, with Scan Identifier, Comment, and Extract Id, whether started manually from the Management GUI or via a Test Procedure
  • HAMS fullset recorded, with Comment and Extract Id
  • Critical log saved
  • Logs recorded, with LogName.Instance# and Extract Id, whether started manually from the Management GUI or via a Test Procedure
  • Dynamic Manoeuvres recorded with dynamic data file identifier (both start and stop events)
  • Quality Ceiling changes
  • Set Limit changes
  • Set Channel Values
  • Shift Change information (logged to the Test Dairy with a unique Test Step of TEST_INFO)
  • Result statements from Test Procedures
  • Test procedure execution result (retry, abort, continue)
  • Comments entered by the user

The user is able to add comments to the Test Diary as free format text or from a list of pre-defined comments using the Comments Panel. The comments field is limited up to 255 characters. The user is able to cut and paste one entry at a time (using Ctrl-C and Ctrl-V keys only) from the Test Diary Window to the comment entry field.

The Test Diary header information is updated through the Test Information Page or Shift Change panel. Changes to any field in the Test Information Page is logged to the Test Diary with a unique Test Step of TEST_INFO.

If a Test Step is not specified with an event, it is written to the database as N/A.

Non-English comments entered in fields that allow non-English characters are displayed in the Test Diary Window the way they are entered in the field.

To add a new comment:

  1. From the Comment Editor, right-click a comment and select Add New Comment. The Comment Information Panel is displayed.
  2. Enter a comment in the Comment Text field.

    The comment can be no longer than 255 characters.

  3. Select either Fullset, Log or User to determine the type of comment. The Update button is enabled.

    If the User comment check box is not selected, the Category filter drop-down list is greyed out because the filter has no effect on the Fullset comments.

  5. Select Add as New to add a new comment or Update to change an existing comment.

RT Channel Configuration Change

The RT Channel Configuration Change (3C Panel) allows users to make on-line changes to the RTE during runtime for the following:

  • Engineering Units
  • Equation of a configured calculated channel (including the modification of the look-up table definition within a calculated expression)
  • Add/remove configured channels to/from the critical log
  • Sensors assigned to a configured channel
  • Add, modify or remove a limit value associated with a configured channel
  • Update the corresponding configuration data, provided the configuration is not already locked by another user

The Channel Config Change feature cannot be accessed while proDAS is scanning. Only the Administrator has the authorization to perform the task.

To access the RT Channel Configuration Change panel:

  1. From the Management GUI Tools menu, select Channel Config Change. The Management GUI - Channels to display window is displayed.

  2. Set the criteria for the channels to display on the Change Panel and select Continue. The configuration data is loaded and the RT Channel Configuration Change panel is displayed.
  3. Enter the desired changes, select Update Current or Update All to apply the changes.

    4. When the changes are applied and the configuration control is enabled, all changes are automatically committed to the SVN repository.

  4. To exit the RT Channel Configuration Change panel, select Close.
