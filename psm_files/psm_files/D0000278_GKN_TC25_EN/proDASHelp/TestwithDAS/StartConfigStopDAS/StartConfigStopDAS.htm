Starting, Configuring and Stopping proDAS

Overview

The proDAS system must be started in a specific order. The proDAS Real-Time Engine (RTE), executing on the Host Computer, must be started first and is normally always running on the Real-Time monitors. The proDAS Management GUI is then started and establishes a connection to the RTE once the user has logged in.

The RTE starts automatically when the user logs in. During the Host Computer login process a program on the Host Computer called prodassession is started in the background. This process is transparent to the user. The prodassession program detects whether the RTE is running or not. If the user selects the Exit proDAS button on the Management GUI Control Panel and exits the RTE, the prodassession program detects the exit event and restarts the RTE within 15-20 seconds.

A complete power-up of the proDAS hardware equipment may be required upon completion of maintenance activities. For a complete hardware power-up and proDAS startup sequence, from a complete power off condition, refer to the instructions in the Docstation.

The hardware must be powered up in a specific order prior to starting and configuring proDAS.

The proDAS hardware equipment should be left on at all times. A complete powerdown of proDAS should only be implemented for maintenance activities, anticipated power outages, or if a power failure occurs. For a complete powerdown sequence of proDAS, refer to the instructions in the Docstation.

For information regarding proDAS usernames and passwords, see User Accounts and Passwords.

Starting proDAS

As mentioned in the overview, the proDAS system must be started in a specific order. The RTE, which normally runs on the Real-Time monitors, is started first. Once an engine test has been completed and the user exits the RTE (by selecting the Exit proDAS button on the Management GUI Control Panel), the RTE application will restart automatically within 15-20 seconds.

Before the RTE can enter the scan mode and monitor engine parameters, the RTE must be configured by:

  • Starting proDAS on the Management Interface Computer
  • Configuring proDAS
    • Configuring proDAS for an Engine Test

Starting proDAS on the Management Interface Computer

Check that the Management Interface Computer is connected to a power source and that the keyboard, mouse and monitor cables are properly connected.

  1. Power-up the Management Interface Computer and monitor.
  2. Double-click the Management GUI icon.
    The proDAS login window is displayed.
  3. Enter the user name in the Username: field.
  4. Enter the user password in the Password: field and select Login.

    The password is case sensitive.

    The Management GUI is started and the Management GUI Control Panel is displayed. The Unified Event Log (UEL) is also started on the Real-Time Display Computer.
    For detailed information on the Management GUI Control Panel, see Management GUI Control Panel.

Configuring proDAS

The user must download a master configuration to the RTE before the RTE can enter the scan mode and monitor the engine parameters. The master configuration defines the data that the RTE receives or transmits during an engine test. The master configuration consists of a collection of the following:

  • break point tables
  • channels
  • engineering units
  • macros
  • polynomials
  • subsystems
  • text output pages
  • transient logs
  • user functions

Each master configuration is assigned a unique number when it is created. This number does not change, although the user may update the master configuration by adding or deleting configuration information which belong to the master configuration. In addition, the comment describing the master configuration can be edited.

The elements that comprise the master configuration (i.e.: channels and logs) can be modified using an editor or utility designed for this specific purpose. The following editors and utilities are available from the Tools menu of the Management GUI Control Panel:

The Tools menu list is based on customer requirements and may not contain all of the editors and utilities listed below.

  • Config Admin (Configuration Administration)
  • Channel Editor
  • HW Subsys Editor (Hardware Subsystem Editor)
  • EU Editor (Engineering Unit Editor)
  • BPT Editor (Breakpoint Table Editor)
  • Polynomial Editor
  • Transient Log Editor
  • Macro Editor
  • TOP Editor (Text Output Pages)
  • User Functions
  • RTD Editor (Real-Time Display Editor)
  • Recalculation
  • Script Editor
  • Sensor Editor
  • TAU (Test Archive Utility)

Configuring proDAS for an Engine Test

proDAS can be configured in the following two ways:

  • starting with the Test Name
  • starting with the Configuration

To configure proDAS for an engine test:

  1. Start the Management GUI. The Management GUI Control Panel is displayed.

    When started, the Management GUI Control Panel defaults to displaying the most recent test and configuration data.

  2. Select either the Replay or Acquisition operating mode.
  3. Select either:
    • the pin (toggle button) next to the Test Name drop-down list and select the desired test name or New Test
    • the pin (toggle button) next to the Configuration drop-down list and select the desired configuration
  4. Fill in the remaining fields.

    As each field is selected, the fields below will be filtered based on the previous selection (i.e.: when a test cell is selected in the Test Cell drop-down list, only the engine types associated with that test cell are available in the Engine Type drop-down list).

    If there is only one valid option for the field, the Management GUI will automatically fill it in.

  5. Click Apply.
    If New Test was selected the New Test box is displayed.

    Only letters a to z (lower and upper case), numbers 0 to 9, underscores, and periods can be used in these fields (e.g.: Tb_G.5).

  6. Fill in the Test Name, Engine Serial Number and Engine Build Number.

    A unique test name must be entered. It cannot be the same as an existing test name.

  7. Click Apply. The Test Information Page is displayed and proDAS begins the configuration process.

    This is a default Test Information Page. Customer specific pages may appear different.

    To de-configure the RTE and return back to the Management GUI Control Panel, select Abort.

  8. Fill out any required fields and do one of the following:
    • To apply any changes, click Apply.
    • To undo any changes, click Reset.
  9. If Apply is selected, the Test Information Page is closed, the engine configuration is downloaded and the Management GUI Control Panel (below) is displayed.

    If Acquisition was selected, the RTE will automatically begin scanning and if Replay was selected, the Replay Control GUI will be displayed.

  10. Check the messages displayed in the UEL Display to ensure that the download was successful.
  11. Assuming a successful download, proDAS is now ready for engine testing.
    If the Acquisition operating mode was selected, see Running an Engine Test.
    If the Replay operating mode was selected, see Replay Control.

Stopping proDAS

proDAS can be stopped in the following three different ways:

  • de-configuring proDAS
  • exiting the Management GUI
  • exiting proDAS

De-configuring proDAS

De-configuring proDAS will stop the current test and allow the user to re-configure by exiting the RTE. To de-configure proDAS:

  1. From the Management GUI Control Panel File menu, select De-configure proDAS. A confirmation window is displayed.
  2. Select Yes to de-configure or No to cancel and return to the Management GUI Control Panel.
    If Yes is selected and proDAS is scanning, a stop scan script will be launched shutting down utilities such as the Alarm Summary Display and Real-Time Displays. Once stop scan is complete, the RTE will exit and a stop script will be launched shutting down the Unified Event Log (UEL). The RTE is restarted automatically (see Overview) and the Management GUI is reset so a new configuration may be selected.

Exiting the Management GUI

Exiting the Management GUI will exit the Management GUI Control Panel but it will not affect the state of the RTE (i.e. if the RTE was scanning data, it will continue to scan data).

To exit the Management GUI:

  1. From the Management GUI Control Panel File menu, select Close Management GUI. The Management GUI Control Panel is closed.

    The RTE will continue to gather data.

Exiting proDAS

Exiting proDAS will perform the function of de-configuring proDAS and stopping the Management GUI. To exit proDAS:

  1. From the Management GUI Control Panel, select Exit proDAS or from the File menu, select Exit proDAS.
    A confirmation window is displayed.
  2. Select Yes to exit proDAS or select No to cancel and return to the Management GUI Control Panel.
