Break Point Table Editor

Overview

The Break Point Table Editor is used to create and edit break point tables.

Only break point tables used for calculations (e.g.: in calculated channels), the User Functions, or an External Hook Program are created and edited using this editor. Break point tables used in calibration are covered in the Sensor Editor.

Break Point Table Editor supports the import/export of CSV files representing the Break Point Table data.


The Break Point Table Editor provides the following functionality:

  • File
  • Edit
  • View
  • Configuration
  • Help
  • Index, Level, Enabled, Active, LastModDate, Name and Description
  • Specific Properties of a Break Point Table
  • The List of Break Points
  • Modification of the Break Points
  • Graphical Display of a Break Point Table

Layout

It is possible to split the main window horizontally or vertically with an entry in the BreakPointTableEditor.ini file. When split vertically (see sample screen above) the "Selected Data Record" pane and the "Data Records" pane are parallel. When split horizontally (see sample screen below), the "Selected Data Record" pane and the "Data Records" pane are stacked.


The layout can be selected by setting the following ini properties:

[General]

# for horizontal layout:
WindowLayout=stacked

# for vertical layout:
WindowLayout=parallel

Specific Properties of a Break Point Table

If the user appends a new data record, a dialogue is displayed where the user can select the type of the new break point table. Currently two and three dimensional break point tables are supported.


The following specific properties are displayed in the spreadsheet:

Column Name XML Attribute Name Description Type Acceptable Values
Type Type Type of the break point table String 2d or 3d (read only)
#BP n/a Number of break points Integer read-only
X Unit XEngineeringUnit Engineering unit for the x values String A defined engineering unit
Y Unit YEngineeringUnit Engineering unit for the y values String A defined engineering unit
Z Unit ZEngineeringUnit Engineering unit for the z values String A defined engineering unit (only 3d)

List of Break Points

The list of break points from the selected break point table is displayed in the right-hand pane of the window. The pane is called the List of Points View. For each break point, the list contains the following properties:

Column Name XML Attribute Name Description Type Acceptable Values
X X x value Float arbitrary
Y Y y value Float arbitrary
Z Z z value Float arbitrary (only 3d)

Matrix View (3D only)

All 3D tables are displayed in a matrix-like structure called Matrix View.

The Operator can switch between the Matrix View and the Set of Points View providing the existing values of the point coordinates are supported. The Operator can switch from the Matrix View to the Set of Points View (even after the matrix has been partially populated); however, switching from the Set of Points View to the Matrix View is not possible.


Modification of the Break Points

The break point list can be modified either using the Break Point menu or the context menu. The context menu is displayed by clicking the right mouse button.

  • Add Break Point adds a break point at the current cursor position
  • Delete Break Point deletes a break point at the current cursor position. Pushing the down arrow button appends a new break point
  • Cut Break Point copies the selected rows to the clipboard and deletes them
  • Copy Break Point copies the selected rows to the clipboard without deleting them
  • Paste Break Point (Insert) inserts the content of the clipboard at the current cursor position. The following break points are moved downward
  • Paste Break Point (Overwrite) inserts the content of the clipboard at the current cursor position and overwrites existing data

Graphical Display of a Break Point Table

The break point tables can be displayed graphically by the menu item Break Point - Display Break Point Table or using the last item of the context menu or pushing the button Display Graphics. Three dimensional break point tables are displayed as a set of curves with x as curve parameter and z as independent parameter.
