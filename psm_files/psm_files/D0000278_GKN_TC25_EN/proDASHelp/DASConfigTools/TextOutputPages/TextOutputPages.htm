Text Output Pages

Overview

The Text Output Pages is a specialised reporting system within proDAS consisting of pages of preformatted text which contain placeholders for channel values. They allow a client to:

  • print the pages and write them to file, with inserted values
  • display dialogues for user input of values

The Text Output Pages includes two components:

  • Text Output Page Editor (used to define the layout of the printed pages)
  • Text Output Page Server (used to populate the pages dynamically at run-time)

Text Output Page Editor

The Text Output Page Editor uses the Configuration Database to retrieve and save the configuration information required to perform an Engine Test. The Text Output Page Editor allows the user to define and manipulate Text Output Pages.

The Text Output Page Editor provides the following menu bar to help the user configure the Text Output Pages. In the page view, there is also a context menu available which provides commands for editing.

Simulation

The Simulation menu allows the user to verify the layout of the currently edited page using the Text Output Page Server.

Text Output

This operation uses the Text Output Page Server to generate a formatted string of the edited page. The resulting text is displayed by the Editor in a dialogue window using a fixed font.

Text Printout

This operation uses the Text Output Page Server to generate a formatted string of the edited page. The resulting text is sent to the Print Server. A Print Options dialogue box is displayed.

Text Dialog

This operation uses the Text Output Page Server to generate a dialogue window showing the edited page. Entries that are marked as editable are shown as controls which allow the user to modify the values.

Properties...

If the user right-clicks an existing entry and selects the Properties... item, then the specific properties dialogue for the entry type is opened where the user can edit the settings (see Text Output Page Entry Types).

New Entry

This command allows the creation of a new entry type. This can be any of the four types listed below. A properties dialogue is opened which has initial settings that the user can modify.

Fixed Text Entry...

This creates a new Fixed Text Entry at the specified position, see Fixed Text Entry.

Variable Text Entry...

This creates a new Variable Text Entry at the specified position, see Variable Text Entry.

Channel Entry...

This creates a new Channel Entry at the specified position, see Channel Entry.

Group of Channel Entries...

This allows the creation of multiple channel values at the same time. The dialogue has most of the controls that are also in the Channel Entry properties dialogue.

The Available Channels window shows the channels that have been defined for the configuration. The user may select and add them to the Selected Channels. The Selected Channels are the channels for which entries are created using the settings.

All settings for the group are the same, except the position.

The position of the entries is calculated automatically, either Vertical or Horizontal.

For Vertical positioning, the entries are ordered one below the other until the page border is reached. Then the next column is created.

For Horizontal positioning, the entries are placed in line one after the other until the page border is reached. Then the next line is used.

Delete Entry

This command deletes the selected entry.

Add Form Feed

This command creates a new Form Feed Entry, see Form Feed Entry.

Delete Form Feed

This deletes a Form Feed Entry.

Cut, Copy, and Paste

The Cut command copies the selected entry into the clipboard of the Text Output Page Editor and deletes it from the page.

The Copy command copies the selected entry into the clipboard of the text Output Page Editor without deleting it from the page.

The Paste command creates a copy of the entry from the clipboard at the current position on the page. The entry remains in the clipboard until the editor is closed or another entry overwrites it.

Add

The Add command allows rows and columns to be added to the Text Output Page in the following manner:

Column before

This command allows the user to add a new column to the left-hand side of the current selected cell. This command increases the overall size of the page and entries to the right-hand side of the new column are moved accordingly.

Column after

This command allows the user to add a new column to the right-hand side of the current selected cell. This command increases the overall size of the page and entries to the right-hand side of the new column are moved accordingly.

Row before

This command allows the user to add a new row above the selected cell. This command increases the overall size of the page. Entries below the new row are moved accordingly.

Row after

This command allows the user to add a new row below the selected cell. This command increases the overall size of the page. Entries below the new row are moved accordingly.

Remove

The Remove command allows rows and columns to be removed from Text Output Pages in the following manner:

Empty Columns

This command allows the user to delete a number of contiguous empty columns. This decreases the overall size of the page.

Rows

This command allows the user to delete a number of rows. This decreases the overall size of the page. Entries within the deleted columns are deleted.

Creating a Text Output Page

The Text Output Page Editor allows the user to predefine the elements of the printed page, by placing both text and specialised place holders onto the page. The specialised place holders are known as Text Output Page Entries.

To create a text output page:

  1. From the Configuration Administrator, select the desired configuration and click on the Text Output Page Editor icon.
  2. In the left hand pane, right click and select Append Data Record from the context menu.
  3. Enter a name and description of the new page.
  4. If desired, adjust the Cols and Rows to reflect the desired page dimensions.
  5. Add the desired entries to the Text Output Page by right-clicking on the right-hand pane and selecting Add Entry from the context menu.
    For a description of the types of entries that can be placed in a Text Output Page, see the Text Output Page Entry Types.
  6. When the page is laid out as desired, select File Save to save the formatted page.

Text Output Page Entry Types

A Text Output Page may contain four different types of entries:

  • Fixed Text Entry
  • Variable Text Entry
  • Channel Entry
  • Form Feed Entry

Fixed Text Entry

A Fixed Text Entry allows the user to enter an arbitrary fixed string of characters onto a Text Output Page. These entries are not editable in a dialogue nor updated dynamically. Such text appears on the page as it will appear in the output.

This entry type is defined by:

  • a column and row position in the page
  • simple text (i.e.: text which does not contain control characters)

To add a Fixed Text Entry field to a Text Output Page:

  1. Click on the cell in the right hand grid which represents the location where the field is to be placed.
  2. Right-click and select New Entry->Fixed Text Entry from the context menu that is displayed.
    The Fixed Text Entry Properties dialogue box is displayed.
  3. Enter the desired text in the Fixed Text field.
  4. If desired, the Column and Row position can be adjusted by entering a new number in the Column or Row fields.

    These fields default to the cell from which the dialogue box was opened.

Variable Text Entry

A Variable Text Entry field represents a placeholder for test information. At run-time, the Text Output Page Server reads the test information from the Test Info or Customer Specific tables in the Test Results and Sensor Calibration Database and populates the field.

The Variable Text Entry is defined by:

  • a column and row position on the page
  • a name corresponding to a field in the Test Results and Sensor Calibration Database
  • a format which can be either a string format or an enumeration format
  • a flag indicating whether the entry is editable in a dialogue
  • a flag indicating whether it is possible to enter variable text for enumeration formats in a dialogue

The Variable Text Entry appears on the page view of the editor as a contiguous set of grey cells in one row (i.e.: ). It shows the name of the database field within <>-brackets. The length of the entry is calculated from the format settings.

To add a Variable Text Entry field to a Text Output Page:

  1. Click on the cell in the right-hand grid which represents the location where the field is to be placed.
  2. Right click and select New Entry->Variable Text Entry from the context menu that is displayed.
    The Variable Text Entry Properties dialogue box is displayed.
  3. If the desired field is contained in the Test Info table, select the field name from the drop-down list provided.
    If the desired field is contained in the Customer Specific table, click on the Customer Specific radio-button and manually enter the desired field name.
  4. If desired, the Column and Row position can be adjusted by entering a new number in the Column or Row fields.

    These fields default to the cell from which the dialogue box was opened.

  5. Format the string as desired by selecting the Edit Field or Enumeration Format radio button.
    For more information on the string formats, see Format Controls.
  6. If desired, set the Read-Only flag to ensure that the entry cannot be modified by the user.
    If not set as Read-only, the value can be dynamically changed at run-time.
  7. If desired, select the Enter New flag to specify that the entry allows the user to enter variable values for enumeration formats which do not already exist.
    If Enter New flag is not selected, then only the values listed in the enumeration are allowed.

Channel Entry

A Channel Entry is a placeholder for values which can be read from the Current Value Table of the Real-Time Engine (RTE) or from the Test Results and Sensor Calibration Database (TRSCDB). A channel can represent a physical measured value at the device-under-test or a calculated value.

The Text Output Page Server populates a Channel Entry with the value read from the specified data source, either the RTE or the TRSCDB, using the specified format.

The Channel Entry is defined by:

  • a column and row position on the page
  • a name of the channel which is to be read
  • a value that specifies the space to be reserved for the channel name
  • a value for the space to be reserved for the alternate channel name
  • a value for the space to be reserved for the engineering unit
  • a value for the space to be reserved for the channel description
  • a format for the channel value which may be one of the following:
    • a floating point number format either scientific or fixed point
    • a boolean format which displays two different strings for the values TRUE and FALSE
    • an integer number format which can be either a number or an enumeration as well as a format for date and time values
  • a flag which indicates whether the entry is displayed in a dialogue output
  • a flag which indicates whether the entry is read-only
  • optionally minimum and maximum values which can be entered by the user for float and integer values in a dialogue output if the entry is editable

A channel entry is formatted in the following way:

[Channel Name] + [Alternate Name] + Channel Value + [Engineering Unit] + [Channel description]

where a parameter (specifier) in [] denotes that the parameter is optional.

For example:

To add a Channel Entry field to a Text Output Page:

  1. Click on the cell in the right hand grid which represents the location where the field is to be placed.
  2. Right-click and select New Entry->Channel Entry from the context menu that is displayed.
    The Channel Entry Properties dialogue box is displayed.
  3. Select the desired Channel Name from the list of channel names displayed.
  4. If desired, the Column and Row position can be adjusted by entering a new number in the Column or Row fields.

    These fields default to the cell from which the dialogue box was opened.

  5. Enter the desired length of the string for output of the Channel Name, Alternate Name, Engineering Unit, and Channel Description.
    A value of 0 (zero) suppresses the display of the specific field.
  6. Select the desired display format.
    It is possible to define seven kinds of formats for channel entries, but not all of them can be applied to a single channel. The format that is used, depends on the datatype of the channel which was defined in the Channel Editor. For more information on the formats, see Format Controls.
  7. If desired, select the Display Value in Dialogue flag to specify that the entry is to be displayed in a dialogue.
  8. If desired, select the Read-only flag to ensure that the value cannot be edited.
  9. Set the desired Input Range to specify the range of valid values.
    The entry must be within the Input Range to be editable.

Form Feed Entry

A Form Feed Entry allows the user to control the printout of a Text Output Page definition. After a form feed, the printer starts a new page. In the Text Output Page Editor, a Form Feed Entry is displayed as a red line which separates the pages to be printed.

The Form Feed Entry is defined by:

  • a row position

To add a Form Feed Entry field to a Text Output Page:

  1. Click on the row in the right-hand grid which represents the location where the Form Feed is to be placed.

    The column position has no influence on the behavior or appearance of a Form Feed Entry.

  2. Right-click and select Add Form Feed from the context menu that is displayed.
