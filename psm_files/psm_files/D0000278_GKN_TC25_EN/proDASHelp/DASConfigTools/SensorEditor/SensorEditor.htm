Sensor Editor

Overview

Expired sensors are reported in the Unified Event Log (UEL) during proDAS startup.

proDAS acquires data from thermal, pressure and humidity sensors which require periodic calibration. The Sensor Editor application organizes the calibration data for each sensor in a directory with search features that locate the data by association with a sensor name, type, serial number and component part. The Sensor Editor also:

  • creates, deletes, modifies, imports and exports sensor component and calibration data
  • generates sensor summary and calibration reports
  • displays calibration data in graphical form

Sensor component and calibration data is exported and imported from/to the Sensor Editor in the form of .xml and .txt files. The files include name, type, functional information and an auxiliary name to distinguish sensors of the same type (sensors of the same type are used by proDAS for different functions).

If a sensor is imported from an external system, having a negative sensor ID, most of the functions of the Sensor Editor's edit menu are deactivated.

Sensor Editor Components

The Sensor Editor screen is divided into panels and buttons for selecting and displaying the sensor data.

The screen consists of the following primary components:

  • function buttons and menus
  • Sensor Directory panel
  • Sensors panel
  • Calibrations panel
  • Calibration and Raw Table panels
  • Status Bar

Sensor Directory Panel

Sensor data is displayed by selecting the sensors from within the Sensor Directory panel on the left side of the Sensor Editor.

The directory organizes the sensors in a similar fashion as Windows folders and files. The primary headings (engine, facility and system) are expanded by selecting the plus (+) icon to reveal sensor groups. During a search, the headings automatically expand to display a sensor that is located by the search engine.

Each sensor within a primary heading (SCUTR in the preceding image) is identified by a part name (MG-001 in the preceding image) that identifies the sensor type (e.g., thermocouple, pressure transducer or accelerometer). Sensors that share the same part name are differentiated by a sensor name and auxiliary address (e.g., DG000001_xyzCJ_1 in the preceding image). The total number of sensors that share the same part name is indicated in brackets (SCUTR [1265/1301] and MG-001 [1121/1157] in the preceding image).

The check box beside the auxiliary address is selected to display associated sensor data in the Sensors, Calibrations, Calibration Table and Raw Table panels.

A right - click selection on a primary heading displays a menu with Select All and Select None options. The part names and the Select All / Select None apply to all sensors listed in the left panel.

Sensors Panel

After sensors are selected from within the Sensor Directory, the Sensor panel displays the data in rows (one row for each selected sensor part). Highlighting a row populates the Calibrations Panel, Calibration Table and Raw Table panels with associated data. Sensors with expired calibration dates are highlighted by the Sensor Editor in red. Sensors that are approaching the calibration expiry date are highlighted in yellow.

The data in each row is separated by the following column headings:

Calibration and Raw Table Panels

During calibration, signals are injected into sensors from calibration equipment and the resulting sensor output values are stored in a database. The Sensor Editor displays the database values in the Calibration Table and Raw Table panels. The Raw Table panel displays input and output values in engineering units that were measured during calibration. The Calibration Table panel displays data that has been derived from the raw table data by the Sensor Calibration Application (SCA).

The Calibration and Raw data is displayed in table form by selecting Data (below) or the Table Display button ( ) at the top of the Sensor Editor window.

Both Calibration and Raw tables have a Sequence column that indicates instances of sensor measurements. The number of rows in the Sequence column correspond to points that form a graphical curve. Data points in a Raw table are displayed in a simple two-dimensional curve with engineering units in the X and Y axes. The derived data points in a Calibration table contain slope and offset functions to form a calibration curve.

Calibration tables may be 2D (two-dimensional) breakpoint, 3D (three-dimensional) breakpoint or polynomial types. A 2D breakpoint table has X and Y columns that contain input and output values. A 3D breakpoint table has X and Y columns that contain input values and a Z column that contains output values. A polynomial table has only Sequence and Coefficient columns.

The numerical data is displayed in two-dimensional graph form by selecting Graph (below) or the Graph Display button ( ) at the top of the Sensor Editor window.

The test equipment that is used to obtain the numerical / graphical data is displayed by selecting Equipment (below) or the Calibration Equipment Display button ( ) at the top of the Sensor Editor window.

Status Bar

The Status Bar displays messages, alarms and warnings as Sensor Editor functions are executed. Select Clear to delete the Status Bar contents.

Starting the Sensor Editor

To start the Sensor Editor:

  1. From the Management GUI Control panel Tools menu, select Sensor Editor. The Sensor Editor screen is displayed.

Viewing Sensor Calibration Data

Sensor calibration data may be displayed in the Sensors panel using:

  • Sensor Directory
  • Filter
  • Find

Sensor Directory

To display sensor calibration data using the Sensor Directory:

  1. Locate the sensor names within the directory by system, facility or engine location (e.g., SCUTR), part name (e.g., MG-001), sensor name and auxiliary address (e.g., DG000001_xyz.CJ_1).
  2. Select the check boxes that are beside the auxiliary addresses of the required sensors in the directory. The Sensors, Calibrations, Calibration Table and Raw Table panels are populated with sensor data.

Locating Sensors in the Directory from the Sensors Panel

From the Sensors panel:

  1. Select (highlight) a sensor from the panel.
  2. Right click on the selected row. A menu is displayed.
  3. Select Select in browser from the menu. The Sensor Directory expands and highlights the sensor.

Filter

The Filter window locates and displays a range of sensor data in the Sensors, Calibrations, Calibration Table and Raw Table panels. The range of displayed data is defined by the search parameters in the Filter window. To display sensor calibration data using the Filter feature:

  1. From the Edit menu, select Filter or from the Function Buttons ( ). The Filter window is displayed.
  2. Toggle Less/More to hide or display the Calibration Date functions on the Filter window.
  3. Define the range of the filter by entering search data in the fields of the window. If necessary, narrow the filter range by applying the appropriate expressions to the data fields. Select the expressions from the drop-down lists that are displayed using the arrow buttons. Select Clear Condition to clear the filter (no filter is applied to the displayed data).
  4. If necessary, select additional filter parameters from the Wildcards/Regular Expressions, Table Type and Calibration Date fields.
    The Calibration Date range may be entered by selecting dates from the drop-down calendars in the Calibration Date fields.
  5. Select Apply Condition to activate the Filter. The defined sensor data is displayed in the Sensors, Calibrations, Calibration Table and Raw Table panels.
  6. Select Clear Condition to deactivate the filter.
  7. Select Close to close the window.

Find

The Find window locates and highlights an instance (row) of sensor data in the Sensors panel. The window has several parameters that define a data search. When the Find feature is selected, the total number of matching sensors is displayed in the Status Bar, the first matching sensor is highlighted in the Sensor Directory panel, and the Find window is displayed.

Subsequent matches of the sensor are located with the Find Next feature. The Find Previous feature displays preceding matches of the sensor. To display sensor calibration data using the Find feature:

  1. From the Edit menu, select Find or from the Functions Buttons ( ). The Find window is displayed.
  2. Toggle Less/More to hide or display the Calibration Date functions on the Filter window.
  3. Define the range of the search by entering search data in the fields of the window. If necessary, narrow the search range by applying the appropriate expressions to the data fields. Select the expressions from the drop-down lists that are displayed using the arrow buttons.
  4. If necessary, select additional search parameters from the Wildcards/Regular Expressions, Table Type and Calibration Date fields.
    The Calibration Date range may be entered by selecting dates from the drop-down calendars in the Calibration Date fields.
  5. Select Find Next or the corresponding button ( ) from the Functions Buttons to highlight the first instance of the defined sensor data in the Sensors panel. Select Find Next again to highlight subsequent instances of the data. Select Find Previous or the corresponding button ( ) from the Functions Buttons to highlight the previous instance of the data in the panel.
  6. Select Close to close the window.

Creating and Editing New Sensor Data

New sensor type and part data is created using the Edit Sensor Type and Edit Sensor Part functions of the Edit menu (there are no equivalent function buttons). Sensor data is edited using the Update Sensor window.

Create New Sensor Type

To create a new sensor type:

  1. From the Edit menu, select Edit Sensor Type. The Edit Sensor Types window is displayed.
  2. Enter the names of the new sensor types in the Type Name column of the window.
  3. Select OK to enter the data into the database.
    The names are displayed in the Sensor Type fields within the Update Sensor, Add Sensor, Clone Sensor, Add Calibration Record and Update Calibration Record windows.

Create New Sensor Part

To create a new sensor part:

  1. From the Edit menu, select Edit Sensor Part. The Edit Sensor Parts window is displayed.
  2. Enter the names and descriptions of the new sensor parts in the Part Name and Descriptions columns of the window.
  3. Select OK to enter the data into the database.
    The names are displayed in the Sensor Part Name fields within the Update Sensor, Add Sensor, Clone Sensor, Add Calibration Record and Update Calibration Record windows.

Edit Sensor Data

To edit sensor data:

  1. Select (highlight) the required sensor data on the Sensors panel.
  2. From the Edit menu, select Edit Sensor or from the Functions Buttons ( ). The Update Sensor window is displayed.
  3. Modify the required sensor data in the active fields.
  4. Select OK.

Creating Calibration Data for a New Sensor

Calibration data for a new sensor is created using the Add Sensor and Clone Sensor windows. The Add Sensor window creates calibration data for a new sensor. The Clone Sensor window copies existing data from an existing sensor and applies (pastes) the data to the new sensor.

Creating New Sensor Calibration Data using the Add Sensor Window

To create new sensor calibration data using the Add Sensor Window:

  1. From the Edit menu, select Add Sensor or from the Functions Buttons ( ) . The Add Sensor window is displayed first with the Sensor Data tab.
  2. Enter the required data into the Sensor Data tab fields (the Sensor ID field already contains data). The Calibration Data tab cannot be selected until data is entered into all required fields, which are identified by an indicator ( ).

    3. The Sensor Name cannot be the same as the name of an existing sensor. If the user attempts to use an existing sensor name, a message appears indicating that the Sensor Name must be unique and the sensor data cannot be saved.

  3. Select the Calibration Data tab. The Calibration Data panel is displayed.
  4. Select OK if no data is required for the Calibration Data panel fields. The Add Sensor window closes and the calibration data is activated for the sensor.
  5. Enter any required data into the Calibration Data fields. Identify the required calibration equipment by selecting the appropriate row in the V field.

    Available equipment with expired calibration dates are highlighted by the Sensor Editor in red. Available equipment that are approaching the calibration expiry date are highlighted in yellow.

  6. Select the Table Data/Raw Table Data tab to view the resulting calibration and raw data tables for the new sensor data. Select Graph to display the table data in graphical form. Data from an xls. file may be copied and pasted into either field, and data may be copied from the fields in the window.

    7. When creating a sensor with a 2D Break Point Table, if the table data and raw table data fields are left blank during creation, an error message will inform the user after creation that the fields are blank. The sensor will have to be deleted and recreated to insert data into these fields.

  7. Select the Graph tab to view the sensor data in a graph.
  8. Select OK to close the Add Sensor window and activate the calibration data for the sensor.

Creating New Sensor Calibration Data using the Clone Sensor Window

To create new sensor calibration data using the Clone Sensor Window:

  1. Select (highlight) the row in the Sensors panel that contains the required sensor calibration data.
  2. From the Edit menu, select Clone Sensor or from the Functions Buttons ( ). The Clone Sensor window is displayed first with the Sensor Data tab.
  3. Enter the required data into the Sensor Name and Auxiliary Address fields (the other fields contain copied data). The Calibration Data tab cannot be selected until data is entered into the two fields.
  4. Select the Calibration Data tab. The Calibration Data panel is displayed.
  5. Select OK if no data is required for the Calibration Data panel fields. The Clone Sensor window closes and the calibration data is activated for the sensor.
  6. Enter any required data into the Calibration Data fields. Identify the required calibration equipment by selecting the appropriate row in the V field.

    Available equipment with expired calibration dates are highlighted by the Sensor Editor in red. Available equipment that are approaching the calibration expiry date are highlighted in yellow.

  7. Select the Table Data/Raw Table Data tab to view the resulting calibration and raw data tables for the new sensor data. Select Graph to display the table data in graphical form. Data from an xls. file may be copied and pasted into either field, and data may be copied from the fields in the window.

    When creating a sensor with a 2D Break Point Table, if the table data and raw table data fields are left blank during creation, an error message will inform the user after creation that the fields are blank. The sensor will have to be deleted and recreated to insert data into these fields.

  1. Select OK to close the Clone Sensor window and activate the calibration data for the sensor.

Adding Sensor Calibration Data to a Sensor

The Add Calibration Record window creates a new calibration record for an existing sensor (i.e., highlighted in the Sensors panel). The new calibration data is automatically activated in the database and yellow-highlighted in the Calibrations panel. The older data is retained as part of the calibration history.

To add sensor calibration data to a sensor:

  1. Select (highlight) the required sensor in the Sensors panel.
  2. From the Edit menu, select Add Calibration or from the Functions Buttons ( ). The Add Calibration Record window is displayed with the preselected Calibration Data tab.
  3. Enter any required data into the Calibration Data fields. Identify the required calibration equipment by selecting the appropriate row in the V field.

    Available equipment with expired calibration dates are highlighted by the Sensor Editor in red. Available equipment that are approaching the calibration expiry date are highlighted in yellow.

  4. Select OK.

Editing Sensor Calibration Data

The Update Calibration Record window modifies existing sensor calibration data. To edit sensor calibration data:

  1. Select (highlight) the row in the Sensors panel that contains the desired sensor calibration data.
  2. From the Edit menu, select Edit Calibration or from the Functions Buttons ( ). The Update Calibration Record window is displayed with the preselected Calibration Data tab.
  3. Enter any required data into the Calibration Data fields. Identify the required calibration equipment by selecting the appropriate row in the V field.

    Available equipment with expired calibration dates are highlighted by the Sensor Editor in red. Available equipment that are approaching the calibration expiry date are highlighted in yellow.

  4. Select OK.

Deleting Sensor Calibration Data

To delete sensor calibration data:

  1. Select (highlight) the row in the Sensors panel that contains the sensor calibration data to be deleted.
  2. From the Edit menu, Select Delete Sensor or from the Functions Buttons ( ).

Activating Sensor Calibration Data

The Activate Calibration function applies (activates) data from a sensor calibration history to a sensor. The function also indicates the calibration data that is currently active for the sensor.

The current calibration data may be locked in the database by another user with a higher permission level.

  1. Select (highlight) the desired sensor from the Sensors panel.
  2. Select the required calibration data from the sensor calibration history in the Calibrations panel.
    The currently activated data which is normally highlighted in yellow is now highlighted in green.
    The normally disabled Activate button in the Functions Buttons ( ) is now enabled ( ).
  3. From the Edit menu, select Activate Calibration or from the Functions buttons ( ). A confirmation window is displayed after the activation is applied to the new sensor calibration data.
  4. Select OK.

Editing Sensor Calibration Equipment Data

The calibration equipment database is created and edited with the Edit Calibration Equipment function of the Edit menu (there is no equivalent function button).

To edit sensor calibration equipment data:

  1. From the Edit menu, select Edit Calibration Equipment at the top of the Sensor Editor screen. The Edit Calibration Equipment window is displayed.
  2. Enter the type, part name, serial number and calibration due date of the calibration equipment in the appropriate columns of the window.
  3. Select OK to enter the data into the database.

    The data is displayed in the Calibration Data fields of the Add Sensor, Clone Sensor, Add Calibration Record and Update Calibration Record windows.

Importing Files to the Sensor Editor from proDAS

Files may be imported into the Sensor Editor in .txt and .xml formats. For details, see XML Format Definition File and XML Schematic of a Sensor File.

To import files into the Sensor Editor:

  1. Select Import Sensor ( ) or Import Sensor from the File menu. A window is displayed with the last directory from which files were selected.
  2. Highlight the file(s) to be imported and select Open.
    Importing errors are displayed in the Status Bar.

The default import path is configured in the SensorEditor.ini file, and may be modified by the user:

[Application]
EXPORTPATH=c:\gsw5
IMPORTPATH=..\Data\SensorEditor\Exports
REPORTPATH=..\DATA\SENSOREDITOR\REPORT
SENSORREPORTTEMPLATE= d:\prodas\Data\SensorEditor\Sensor Report Template.xlt
SUMMARYREPORTTEMPLATE= d:\prodas\Data\SensorEditor\Summary Report Template.xlt
NUMBER OF POINTS=160
Expire Soon=350

Exporting Files from the Sensor Editor to proDAS

The Export Section allows the user to take any sensor calibration data from the database and export it to disk as either a .txt or .xml file. For details, see XML Format Definition File and XML Schematic of a Sensor File.

To export files from the Sensor Editor:

  1. Select Export Sensor ( ) or Export Sensor from the File menu. The Export Sensor window with a default export folder location is displayed.
  2. If necessary, select the folder from which to export files.
  3. Select the file types (XML or Tab Delimited Text) in the folder to export.
  4. Select Export.

The export path is configured in the SensorEditor.ini file, and may be modified by the user:

[Application]
EXPORTPATH=c:\gsw5
IMPORTPATH=..\Data\SensorEditor\Exports
REPORTPATH=..\DATA\SENSOREDITOR\REPORT
SENSORREPORTTEMPLATE= d:\prodas\Data\SensorEditor\Sensor Report Template.xlt
SUMMARYREPORTTEMPLATE= d:\prodas\Data\SensorEditor\Summary Report Template.xlt
NUMBER OF POINTS=160
Expire Soon=350

Exported file names are automatically created by the application as a combination of the sensor and calibration columns of the sensor list (example: BIN.2DBPT 01-04-2003 9:52:09PM.csv).

Sensor Summary Report

The Sensor Summary Report includes the following headings:

  • name
  • auxiliary address
  • type
  • part name
  • serial number
  • calibration date
  • calibration due date

The Sensor Calibration Report groups sensors into worksheets by expiry status:

  • expired
  • soon to expire
  • valid
  • never expires

Sensor Calibration Report

The Sensor Calibration Report includes the following headings on each worksheet:

  • expiry status
  • calibration date
  • due date (day, month and year)
  • user name
  • ambient temperature
  • ambient humidity
  • pass/fail status
  • comments
  • associated audits
  • associated equipment

Displayed calibration date and time information is formatted to the regional settings of the computer.
