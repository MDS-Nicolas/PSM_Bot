Creating and Modifying Display Pages

Overview

Display pages are created, administrated and modified by the RTD Editor and displayed in the RTD Driver.

The RTD Editor

The RTD Editor is an extension of the DV-Draw editor.

Since the RTD Editor (displayed below) is based on the DV-Draw editor, which exists only in an English version, there is only an English version of the editor.

The static text option and the alternate channel names support non-English characters.

For more information, see the DataViews documentation. The DataViews documentation can be accessed through the Properties context menu entry, which is accessed by right-clicking any active display object.

The Undo and Redo functions within the DataViews software do not apply to properties of the ActiveX Controls. The effects of these functions are inconsistent. If these functions are used, it is recommended that the user monitor their effects.

Display Objects

A display page is a rectangle containing a number of rectangular objects. The following types of objects are available:

  • drawings, including text
  • DataViews Graph objects
  • external images
  • RTD ActiveX Controls
  • other ActiveX Controls

Predefining Edit Areas

When a display page is opened in a view window, the Real-Time Display Driver automatically maximizes its size WITHOUT changing the Aspect Ratio (AR) of the display page (to prevent distortion of the display page).

AR = page width
page height

The available display space is optimized when the AR of the display page matches the AR of the view window. The AR of the display page is calculated from the Edit Area. Based on the particular system, a display template is available for each view window size and location.

When creating pages, care should be taken to use the appropriate template and ensure that all objects on the page are within the boundaries of the Edit Area

Preparation

Insert a section [EditArea] into the initialization file RTDEditor.ini of the RTD Editor and add one to ten paragraphs (groups of three rows each) containing the following entries:

  • Desc0=description of monitor with number 0

  • Height0=<height of this monitor in pixels>

Creating an Edit Area in the RTD Editor

To create an edit area in the RTD Editor:

  1. From the Edit menu, select Edit Area, Create. The Size of EditArea window is displayed.
  2. Select one of the preconfigured groups. The aspect ratio height/width is calculated automatically.
  3. Once selected, an Edit Area with the given size is created and indicated on the canvas. This object is named ‘EditArea’ automatically.

    Only a single Edit Area can be created.

Working with the Edit Area

The Edit Area cannot be selected using the mouse. To select the Edit Area:

  1. From the Edit menu, select Edit Area, Select.

The Edit Area can be modified (resized) or deleted.

Changing the aspect ratio of the Edit Area

If a given aspect ratio for the Edit Area is required, either change its width and keep the height or change the height and keep its width. To change the aspect ratio:

  1. From the Edit menu, select Edit Area, Adapt Width or Adapt Height.

Measuring the view size using the RTD Driver

The View window of the RTD Driver enables you to measure the actual size of the view. To measure the view size:

  1. From the View menu, select Size. A message window is displayed indicating the width and height of the view in pixels.
    These numbers can be entered into the RTDEditor.ini file to reach the required Edit Area size.

Data Variables

The data variables serve as a source of values to be displayed. The types of data variables are:

  • Channels
  • 2D Break Point Tables
  • 3D Break Point Tables
  • Polynomials
  • Test (data describing the test)
  • Fullset (channels)

To use the data variables, the data variables must be connected with either a property of an ActiveX control or of a DataViews Graph object.

A dialogue box, showing which data variables are available, can be displayed by selecting Data Variables from the View menu. The same dialogue box is displayed when a data variable is connected to a property of an object.

The type of data variable, that can be connected to a property, depends on the type of object.

If a Channel is connected to a property, the value to be displayed is taken from the Current Value Table at runtime.

If a 2D Break Point Table is connected to a property, either the x or the y coordinate must be chosen. The remaining coordinate must then be connected to a corresponding property.

If a 3D Break Point Table is connected to a property, the object displays a curve for each x value of the table formed by the (y,z) pairs of the respective x value.

If a Polynomial is connected to a property, either the x or the y coordinate must be chosen. The remaining coordinate must then be connected to a corresponding property.

Test data is only connected to string properties. If the test data is connected to such a property, then the corresponding string is displayed.

To connect a Fullset channel to a property, such that for each new fullset acquired during the test run the channel value is displayed (instead of the Current Value Table), select the Channels icon in the data variables dialogue and create a new data variable. The name of the data variable must then be defined by typing into the appropriate text field on the right side of the dialogue box.

If all fullset values of this channel are to be displayed regardless of the name of the respective test steps, name the new data variable

fullset/<channel name>

where <channel name> stands for the name of the affected channel.

If the channel values of a certain test step only are to be displayed, then name the new data variable

<test step name>/<channel name>

Where <test step name> stands for the name of the selected test step.

There is no support to select a correct channel name when using this method to connect a property with a fullset value

Finding Data Variables

The pages on which a channel, a 2D break point table, or a polynomial is used can be determined via the Find variable in display pages dialogue box (below). The dialogue box is invoked by selecting Find in Files from the File menu.

To find the pages on which a specific variable is used:

  1. Select the variable (e.g.: Channel) from the ProDas Variable drop list.
  2. Select the item to searched for (e.g.: fCalc2i) from the ProDas Variable drop list.
  3. Select Find.
    All display pages in the directory of the current engine type are searched and the pages containing the selected item are displayed in the list box.

Since the dialogue is non-modal, the display pages may be modified with the dialogue still visible. After such changes, the dialogue may be updated using the Refresh button. Selecting a file in the list box and selecting Open causes the selected page to be read and displayed in a new view.

The Hide button hides the dialogue box, allowing the same information to be displayed the next time the dialogue box is invoked. The Close button closes the dialogue box without saving the contained information.

Preferences

The User Preferences dialogue box, invoked by selecting Preferences from the File menu, contains a subset of the corresponding dialogue described in the DataViews documentation.

The default background and grid colors are displayed and may be edited via color selection dialogues. These colors are used when a new display page is created.

Dragging and rubberbanding let the user see the effect of reshaping an object while moving the control point. Dragging and rubberbanding can be turned on or off. When dragging and rubberbanding are off, the effect cannot be seen until the control point is released. The default setting is On.

Objects may be difficult to see while dragging if the object color is very close to the drawing background color, or if the colors are complementary. The available options set various color masks which result in different sets of colors being used for dragging. If the drag color is hard to see, try a different option. The default setting is From Object, which tries to maintain the color of the object during dragging.

The user can toggle between ASCII and binary format for saving files. Binary files load faster than ASCII files. Binary format is host-dependent so files in binary format may not be readable on another host. ASCII format is host-independent so if views are to be ported to another platform, save them as ASCII files. If a view is saved in ASCII, the view comment can be edited using the text editor. Do not edit any other part of the view file directly. Valid delimiters are ", ', $, !, and @.

Printing the Object List

The list of all objects of the active view can be printed by selecting Print Object List from the File menu. Printing is done via the Print Server.

Security

The RTD Editor requires at least security level 1 to save a display page. Users can either specify their username and password in the command line or be prompted for their username and password when the application starts.

Each user can write to any other directory, regardless of privileges. Therefore, a user without write privileges for the appropriate Engine Type level is able to configure display pages and save them elsewhere for later use in the Display Driver.

Rules

Rules specify the behaviour of an object or a view by describing the actions or changes which occur depending on specified events and conditions.

An action can be changing to another view, pause and resume reading the data and updating the display, redrawing the screen, turning off the rule engine or erasing views. Several actions that affect data are offered, such as:

  • setting a data variable to a specified value
  • setting the value of a destination data variable to the value of a source variable
  • increasing / decreasing the value of a data variable by a specified amount

This can be used to modify the value of a channel during runtime. However, only zero hertz scan rate channels may be modified during runtime.

Do Nothing is the default action selection and can be used to disable a rule for testing purposes or as a place holder when there is no action for the rule.

Each rule can be tied to a specific event such as a mouse click or a key push.

A condition can be which mouse button was clicked, which keyboard key was pushed, or whether or not data values meet specified criteria.

The rules are specified in the property dialogue which is displayed after a click with the right mouse button in the object or view, selecting the Properties entry and the Rules tab. In the following example, the Vector Text object named Equation (i.e.: VectorText:Equation) sets the value of the data variable fCalc1i to 1000 every time the user clicks with the mouse button.

The following action types are not supported by the RTD Driver and should not be used:

  • Faster
  • Slower
  • Execute Command
  • Execute Function in Script
  • Write Data Variable to Data Variable
  • Call Generated Handler
  • Overlay View
  • Erase Overlay View
  • Overlay Object
  • Erase Overlay Object
  • Erase Visible Overlays
  • Popup Object at Cursor
  • Erase Popup Object
  • Erase Visible Popups

For more information, see Using Rules in the DataView documentation.

Dynamics

Dynamic components may be added to an object so that changes occur to the object when the data changes.
The dynamic components, also called dynamic features, are listed in the following table:

Components Description
Attribute dynamics Changes properties such as color, line type, line width, curve type and arc direction.
Motion dynamics Changes position, rotation, and scale.
Subdrawing dynamics Displays different subdrawings depending on a data value.
Visibility dynamics Changes whether or not the object is visible.
Text dynamics Changes the content of a text object to display the data value.
Blinking dynamics Alternates the colors of an object between two sets of colors to make the object blink. An object can blink or not blink, or blink in different colors, depending on a data value.
Fill dynamics Changes fill effects of a filled graphical object.

Dynamic features that do not apply to the type of the currently selected object are disabled.

The dynamic features are specified in the property dialogue which is displayed after clicking with the right mouse button in the object and selecting the Properties entry and the Dynamic tab. In the example, the text of the Vector Text object named Equation is set to x = <data value of channel fCalc1i>. At the same time, the object rotates according to the value of this channel.

For more information, see Using Object Dynamics in the DataView documentation.

Error Window

The error window displays all errors which have occurred in the application since it was called and which have not yet been cleared. Errors are listed in chronological order with the time when they occurred. The latest error is at the bottom.

When an error occurs, the error window pops into the foreground. The Errors window can also be displayed by selecting Show Errors from the Window menu.

The selection in the window or the complete content can be cleared using the corresponding button. The window can be hidden by clicking the Hide button.

Page Administration

Page sequences serve to navigate quickly through sequences of related pages at runtime (i.e.: using the RTD Driver). A dialogue to administrate these sequences can be invoked by selecting Page Sequence from the File menu.

In the dialogue, all existing page sequences are displayed in a list box. A new page sequence can be created by clicking the New button in the dialogue. The selected page sequence can be renamed, copied or deleted by clicking the corresponding button.

The display pages of the selected page sequence are displayed in the second list box. The third list box shows the available pages of the current engine type which are not contained in the page sequence. If a page is selected in one of these list boxes, it may be moved to the other by clicking the appropriate arrow button.

Navigation Bar

The navigation bar displays the default configuration as the current configuration.

To change the current configuration, close all open Views/Pages/Subdrawings and use the Configuration menu.

The general properties dialogue is invoked via the Context menu. If an object is selected, the properties of the object are displayed. In the case of multiple selection, the dialogue of the first object is displayed. If no object is selected, the property dialogue of the page is displayed. Simulation

The behaviour of a display page during runtime can be simulated. Simulation mode is entered by selecting Run from the Test menu. If a graphical DataViews object or an RTD ActiveX Control is connected with a data variable, values for this data variable are simulated. In most cases, a sine-like function is used.

There is a control panel (i.e.: a small window), which is always on top of the Preview and Simulation window, that allows the following manipulations:

  • zoom in and out
  • zoom/pan reset (i.e.: restore the original size)
  • pan in a direction (i.e.: move the visible part in one of four directions)
  • drag rectangular area to zoom
  • choose point to pan (i.e.: the clicked point becomes the middle of the drawing area)
  • show a thumbnail of the page

If the control panel is hidden, it can be made visible by clicking the Show control panel button. The simulated page can be printed with the Print button. The frequency of the simulated data can be changed in the frequency spin control. The simulation can be paused by releasing the Run button. In pause mode, a step can be made by clicking the Step button. Finally, the window can be closed with the Close button.

The display simulation displays the changing values for all controls, whether the channel associated with this control has a scan rate above zero or not.

Object Groups

It is possible to combine several objects into a group. Groups can be used to move or resize objects while keeping the relationship to each other. The following is possible:

  • create a new group ("Group")
  • dissolve a group ("Ungroup")
  • recreate the most recently dissolved group ("Regroup")

A group can be defined by:

  • selecting objects and groups to be combined to a new group
  • selecting Group from the Object menu

A group can be dissolved by:

  • selecting the group that to be dissolved
  • selecting UnGroup from the Object menu

The most recently dissolved group can be restored by (if there elements are still available):

  • selecting ReGroup from the Object menu

A group can be moved selecting one element of the group so that the whole group is selected (the surrounding rectangle is displayed). Once selected, the surrounding rectangle can be moved.

It is also possible to insert other ActiveX Controls (aside from the RTD ActiveX Controls) into a display page within the RTD Editor and display this page in the RTD Driver during runtime. Map Channels to ActiveX Control

For any ActiveX Control, another dialogue can be launched to map the required channels to the properties of the ActiveX Control. This Channel Mapping Dialogue can be displayed by double-clicking the ActiveX Control or by selecting the menu item Map Channels… from the context menu or from the main Object menu.

Channels cannot be mapped to the bevel control. The Map Channels option in the bevel context menu is disabled.

To map a channel to an ActiveX Control:

The channel mapping dialogue supports filtering by subsystem type and subsystem name.

The General Properties Dialogue must be used to map a 3D breakpoint table to an ActiveX Control. Within the ActiveX tab of the dialogue, select the CurveArray property, select a 3D Break Point Table as a variable name, and click Close.

  1. From the Subsystem type, select the desired subsystem.
  2. From the Subsystem name, select the desired subsystem.
  3. From the Select a Channel panel, select the desired channel.
  4. Select the Map To button to map the selected channel to the respective property.
  5. If the control is to display fullset values only, check the Fullset box.
  6. If the fullset values are to be restricted to certain test steps only, enter the name of the test step in the Test Step Name field or select a test step from the drop-down list.

    The drop-down list contains all used names of test steps.

  7. If an ActiveX Control only has a single mappable property, double-clicking a channel in the Select a Channel panel selects the channel, maps it to the single property and closes the dialogue.
    If an ActiveX Control has more mappable properties than can be displayed simultaneously, the currently invisible properties can be displayed by using the vertical scroll bar on the right hand side.
  8. To remove any mapping, select Clear.
  9. To discard all modifications and close the Map Channels to ActiveX Controls window, select Cancel.
  10. To accept and process all modifications and close the Map Channels to ActiveX Controls window, select OK.

Mapping any data variable other than a channel can only be achieved using the data variables dialogue.

The RTD Value Display Control

A RTD Value Control displays the current value of a channel, either numerically, or as a bar graph, or both.

The numerical display consists of the name, the value and the engineering unit display. All these elements are optional. If there is no bar graph, they may be arranged horizontally or vertically. Fonts, sizes, colors and alignment for these elements can be selected separately. The quality may be displayed together with the value.

The bar graph may be horizontal, vertical or circular. It is filled or displays an indicator according to the current value relative to the given minimum and maximum values. A scale and a thin bar indicating the corresponding limit values may be displayed next to the bar.

The alarm state of the channel can be indicated in several ways. If there is a bar which is filled, the filling color indicates the alarm state. The numeric display can indicate the alarm state either by the background color or by the text color.

The RTD Value Control displays integer type channels in both binary and hexadecimal representation; provided the format is specified appropriately.

The RTD Time Display Control

The RTD Time Display Control displays the time and date numerically. Normally, its properties Time and Date should be connected to the channels Time and Date respectively, which are automatically generated channels. If this is done, the time and date of the online data are displayed.

A control can display either time, or date, or both. If both are displayed, they can be arranged horizontally or vertically. Optionally, the channel names are shown. But this is only recommended, if special channels are used for time and date information. For both date and time a variety of formats is available.

Internally, the date and time are represented as float values. The date value is the number of days since the 30th of December 1899. The time value is the fraction of the day (i.e.: it is number between 0 and 1).

Creating a Palette Item for an ActiveX Control

There are two types of sub-drawings (or palette items) available:

  • sub-drawings based on image, text control, other sub-drawings, lines, etc. (created using DV-Draw or the RTD Editor)
  • sub-drawings based on external ActiveX component (created using DV-Draw only)

The RTD Editor can be used to initially create a palette item with an ActiveX Control with the proper ini file setting.

DV-Draw must be used to create a palette item for an ActiveX Control. Each palette item is represented by a subdrawing, (i.e. there is a respective subdrawing file and a bitmap file with an image of the subdrawing). For further details, refer to the DV-Draw Help menu.

To begin creating a palette item for an ActiveX Control, start DV-Draw with an empty view.

Creating the Data Source

Although it is possible to create a sub-drawing that is not mapped to any data object (constant, variable, etc), an ActiveX Control looks better if it is bound to some data variables.

To create the data source:

  1. From the Edit menu (DV-Draw) or the View menu (RTD Editor), select Data Variables.... The Data Selection window is displayed.
  2. Remove all data sources. From the Data Source List, select all data sources and push the delete button.

    If this step is skipped, the sub-drawing file contains information about all data sources (i.e. channels, breakpoint tables, polynomials, etc.) which is useless for a generic palette item. Since this may sum up to thousands of records, it is recommended for RTD Editor users to use an empty configuration when creating or modifying a palette item.

  3. From the New Data Source drop-down list, select Constant.
  4. Create a new data source by selecting the Add New Data Source button.
  5. Enter the string "Channels" in the File Origin: textbox.
  6. To close the Data Selection window, select Close.

    The RTD Editor uses another data source named Channels for the proDAS channels; the type of this data source is Files as opposed to the Constant data source created herein.

Customizing DV-Draw: Option “Insert an ActiveX Control”

This option is not displayed by default when DV-Draw starts. To display the “Create an ActiveX Control” option: Add the button ActiveX to the Object Creation toolbar:

  1. From the Tools menu, select Customize. The Customize window is displayed.
  2. Select the Commands tab.
  3. From the Categories: list select the toolbar Object Creation.
  4. From the Commands: panel, click-and-drag (ActiveX button) onto the Object Creation toolbar.
  5. To close the Customize window, select Close.

Inserting an ActiveX Control into a View

To create an ActiveX control object:

  1. From the Create Object toolbar, select . The Insert Control window is displayed.
  2. From the Control: list, select the desired control type and select OK.

    The fastest way to reach the RTD ActiveX Controls is by entering the letter "s".

  3. Click and drag to define a rectangle for the location of the ActiveX Control.

Configuring the ActiveX Control

Properties and Variable Mapping

You can now configure the ActiveX Control as desired (i.e. create appropriate data variables and map them to the ActiveX Control and define its properties). Since configuring the ActiveX Control is best achieved using the RTD Editor, save the view to a file and open it again in the RTD Editor.

Whether using DV-Draw or the RTD Editor, make sure that there is no other data source than the Constant data source named Channels. If any variables are to be mapped to the ActiveX Control, create new variables in this Constant data source named Channels.

Since some ActiveX Controls may look quite differently depending on the configured properties, it may be worth while creating multiple instances of the ActiveX Control displaying different properties.

Position of the ActiveX Control

To ensure the center point of the ActiveX Control or graphical object is identical with the origin of the coordinates used in the view:

  1. From the View menu, select Grid to display the grid. The origin is indicated by a cross.
  2. Move the AcitveX Control so that its center is over the origin.

    Not positioning the control's center to the origin results in a wrong position of the control once dragged from the palette and placed onto a view.

Adding an Item to the Palette

Adding an item to the palette takes the currently selected object and creates a sub-drawing file containing a copy of the object in the palette directory structure.

Adding a Palette Item

To add a subdrawing item to the palette:

  1. Go to the appropriate tab in the palette.
    The new item is added to the current palette tab. If you have not yet created a tab for this item, see Adding Tabs to the Palette in the DV-Draw documentation.
  2. Select the ActiveX Control.
  3. Right click and select “Add to Palette” from the context menu.

    A sub-drawing might be added to the palette by saving it directly into the palette folder. However, in this case it appears on the palette window only after restarting DV-Draw or the RTD Editor. The subsequent step is not required in this case.

Renaming the Palette Item

The new item is given a default name of Group#.sd, where # is the next number not already used. The first item you add using this method is named Group0.sd, the second is Group1.sd, etc. If you change these default names, their numbers are used again. Therefore, the palette item should be renamed. To rename the palette item:

  1. Open the palette directory (defined in the RTDEditor.ini file in the section General as item DVPaletteDir) and its subdirectory named identically to the palette tab into which the new item has been inserted.
  2. Rename the file (e.g. rename Group#.sd to MultiSwitch.sd).

Creating a Bitmap

Create a bitmap using an appropriate tool (e.g. MS Paint, or by capturing a screen shot of the ActiveX Control) and store this bitmap in a file in the same palette subdirectory as the respective sub-drawing file under the same name as the sub-drawing file (the extension of the bitmap file must be “.gif”, “.bmp” or “.dib”). Using Windows Explorer, make the bitmap file hidden.

Testing the Palette Item

The palette displays the newly added ActiveX Control, indicated by the bitmap image and the name of the sub-drawing file.

If the preview window of the palette is activated, clicking on the palette item shows the ActiveX Control in the preview window as it is configured in the sub-drawing file. When the palette item is dragged and dropped onto the view, it is displayed at the dropping location; its visual appearance is defined by the configuration of the ActiveX Control in the sub-drawing file.

The appropriate proDAS channels, breakpoint tables or polynomials must be mapped to the ActiveX Control and the properties of the ActiveX Control can be adapted.
