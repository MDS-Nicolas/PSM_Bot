Macro Editor

Overview

The Macro Editor is used to edit macros that have a Visual Basic Script (VBS) file extension (.vbs). The macros can be executed from the command line. These macros are used by the Recalculation GUI for the recalculation of fullset data, and by the Management GUI as Test Procedures.

The Macro Editor can only edit macros with a .vbs file extension. It will check the format of the file and prevent the user from editing files that do not meet the requirements. The Script Editor should be used to edit test procedure script files (.tps).

The Macro Editor provides the following functionality:

  • File
  • Edit
  • View
  • Configuration
  • Help
  • columns: Index, Level, Enabled, Active, Last Mod Date, Name, Description, File Name
  • Specific Macro Properties
  • Editing Macro Text
  • Refresh Macro Text
  • List of Categories

Editing Macro Text

The Macro Editor provides the user with two methods of editing the text of a macro. The first method utilises a context menu, with the entries Undo, Cut, Copy, Paste, Delete, and Select All, in the edit field of the right pane of the Macro Editor.

The second method utilises an external editor to edit the text of a macro. The external editor is selected by selecting Select External Editor from the Macro menu. The external editor is then started by selecting Call External Editor from the Macro menu. The text of the currently selected macro is displayed when the external editor is started.

Refresh Macro Text

Once the macro text has been edited and the external editor has been closed, the macro text can be reloaded by selecting Refresh Macro Text or by selecting Refresh Macro Text from the Edit Specific menu.

List of Categories

The right pane displays two lists of categories. One is a list of available categories, as determined by the initialisation file and the categories of all macros. The other is a list of all the selected categories which belong to the currently selected macro.

A new category can be added by entering the name in the edit field next to the Add button and then selecting the Add button.

The Recalculation GUI selects all macros, with a category of Recalculation, as valid recalculation scripts. The Management GUI selects macros, with a configurable category name, as valid Test Procedures. This category name is defined using the MacroCategory keyword in the Management GUI initialisation file.
