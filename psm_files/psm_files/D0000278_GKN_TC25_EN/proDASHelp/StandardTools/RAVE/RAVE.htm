Reporting and Viewing Environment (RAVE)

Overview

The Reporting and Viewing Environment (RAVE) is an application used to view data recorded during a test and to create reports.

Starting RAVE

RAVE can be started in one of two ways:

  • via the Management GUI Control Panel where the username and password are passed automatically
  • as an independent application where the username and password are not passed automatically

To start RAVE from the Management GUI Control Panel:

  1. From the Management GUI Control Panel (below) Tools menu, select Post Test Functions / Real-Time Report or RAVE.

    If RAVE is selected the RTReports screen is not displayed.


    The RTReports screen is displayed.

  2. Select from the RTReports screen.
    The Test Selection screen is displayed. For details, see Test Selection Screen.

To start RAVE as an independent application:

  1. Double-click the RAVE executable file, from your directory structure.
    The RAVE Login window is displayed.

  2. Enter the user name and password in the corresponding fields.
    The user name and password must be a minimum of three characters and five characters long, respectively, to enable the Login button.
  3. Select a database from the Database drop-down list.
  4. Select Login.
  5. The Test Selection screen is displayed (click here to go to the Test Selection screen).

Test Selection Screen

The Test Selection screen (displayed below) is launched after login. The user can retrieve any combination of engine tests using the query builder tool. The range of retrieved data is refined by execution of a single query defined as a set of selected search parameters. Typically, data is retrieved by selecting predefined (already saved) queries from a drop-down list. A query cannot locate a historical test or group of tests that have been archived.

Test Selection Screen
Buttons, Panels and Fields Function
Predefined Queries The Predefined Queries panel retrieves engine test data based on queries that have already been defined and saved. Queries retrieve specific test data by using a combination of search criteria such as test cell, engine type, engine serial number, test name and date.
Query controls The buttons in the Query controls panel execute, save and display query data.
  • Execute - retrieves engine tests based upon parameters in the QUERY BUILDER panels
  • Save series - saves QUERY BUILDER data in an existing Category folder. A new Category folder may be created if desired
  • Clear all - clears data within the QUERY BUILDER panels
  • Reports - displays the Reports screen for the engine test highlighted in the Query Results panel
  • SHOW/HIDE QUERY BUILDER - displays or closes the QUERY BUILDER area (for detailed information on the QUERY BUILDER, see Creating New Queries with the Query Builder Panel in Building Reports with RAVE)
  • Replace - overwrites engine tests in the Query Results panel with data that is retrieved from executing a new query
  • Add - accumulates engine test in the field
  • scroll buttons (<< < > >>) are used to display a number of queries that are currently in use (i.e.: a query series)
  • Data - displays the associate Fullsets for the engine test that is highlighted in the Query Results panel
QUERY BUILDER The QUERY BUILDER panel is subdivided into Standard Test Criteria and Custom Test Criteria sections. There are five fields and five buttons within the Standard Test Criteria section for selecting data for test cell, engine type, engine serial number, test name and test date. The Custom Test Criteria section has twelve data fields and six buttons for retrieving customer specific data.
Query Results The Query Results panel displays the retrieved data under the header bars:
  • Test Cell
  • Test Date
  • Test Name
  • Test Description
  • Engine
  • Serial Number
  • Engine Standard
  • Customer
The header bars within this field have similar functions to standard spreadsheet headers. Columns may be dragged laterally and placed in any order from left to right, and selecting a header bar will sort the corresponding data in ascending or descending order.
Session history The Session History panel displays user related messages during the query session. All messages are stored in (and may be retrieved from) an RTE trace file. The Session history panel may be configured at the administrator level to display only operator messages or to include error and debugging messages to suit user requirements.

The Clear button removes the entry displayed in the Session History panel.
Exit The Exit button closes the Test Selection screen and exits RAVE.

Modifying Permission Levels

The default condition for each level may be customised through the Accounts and Permissions screens by an admitted user. To modify permission levels from the Test Selection Screen, the user must be logged in as the administrator.

From the Admin drop-down list:

  1. Select Permission Manager.
    The Permission Manager window is displayed.

  2. Select a function from the Available Functionality list.
  3. Select an Activiation Level (i.e.: 3) from the Activiation Level drop-down list and select Exit.
    The Permission Manager window is closed and the operator is returned to the Test Selection Screen.
    Select Reset to restore all permission levels to default values.

Reports Screen

The REPORTS button in the QUERY CONTROLS panel of the the Test Selection screen displays the Reports screen (see below). The specific engine test that was highlighted in the QUERY RESULTS panel of the Test Selection screen is included in the Reports Screen. Report templates are selected by the user and populated by RAVE. The reports are produced in Microsoft Excel. Typical reports are:

  • Logsheets (containing fullsets (steady state data) and comments)
  • Transient Data Reports (depicting transient logs)
  • Fullset Reports (depicting steady state data)
  • Combination Reports (depicting both steady state and transient data)
Reports Screen
Buttons, Panels and Fields Function
Currently selected tests The Currently selected tests panel displays the engine test that was highlighted in the Query Results panel of the Test Selection screen.
Single engine reports templates. The Single engine reports templates. panel contains names of previously created reports (in .xls spreadsheet form).
REPORT TEMPLATE CONTENTS: The REPORT TEMPLATE CONTENTS: panel displays the .ini configuration file of the selected Single engine reports template. The file defines the report contents, such as channels and test step names, limit lines and comments.
Report controls The Report controls buttons and functions are:
  • Select All - highlight the report templates within the Single Engine Report Templates panel
  • Unselect All - deselect the report templates within the Single Engine Report Templates panel
  • Report Builder - launches the Report Builder wizard used to create new report templates
  • Peek - displays the .xls (Excel workbook) file based on the selected report template populated with the last test data that was deposited into the template

    • Peek is only available when logged in as an Administrator level user.

    When using the peek function, ensure the peek window spans the full height of the screen. If the peek window is smaller than the size of the screen, some of the user buttons will not be visible.

  • View - generates and displays an .xls report based on the selected report template and data extracted from the TRSCDB for the given engine test
  • Print - prints the selected report in the form of an Excel workbook
Session history The Session history panel displays user related messages during the query session. All messages are stored in (and may be retrieved from) the RAVE trace file. The Session history panel may be configured by the administrator to display only operator messages or to include error and troubleshooting messages to suit the user's requirements.

The Clear button removes the entry displayed in the Session history panel.
Back The Back button closes the Reports screen and displays the Test Selection screen.

Creating and Retrieving Reports

Retrieving Saved Queries with the Predefined Queries Panel

From the PREDEFINED QUERIES panel on the Test Selection screen:

  1. Select a previously saved category (i.e.: ATP_Calib) from the drop-down list in the Query and Report Categories field.

  2. Select the desired query (i.e.: ATP_Calib) from the drop-down list within the Current Queries field.
  3. Select EXECUTE from the QUERY CONTROLS panel to execute the query and retrieve engine tests based on the search criteria defined in the query.
  4. Select the desired engine test (highlighted) listed in the QUERY RESULTS panel.
  5. Select REPORTS from the QUERY CONTROLS panel to open the Reports screen. The Reports screen is displayed.

Creating New Queries with the Query Builder Panel

From the QUERY CONTROLS panel on the Test Selection Screen:

  1. Select Show QUERY BUILDER to display the QUERY BUILDER panel. The QUERY BUILDER section contains Standard Test Criteria and Custom Test Criteria fields.
  2. Populate any of the five data fields in the Standard Test Criteria area (Test Cell, Engine, Engine S/N [serial number], Test Name and Test Date) with the desired query criteria. The Data may be entered manually or selected from drop-down lists. The Buttons beside each field [] activate windows which contain lists of test cells, engines, serial numbers, test names and test dates. Highlight the rows of desired data from the Master List (hold the Ctrl key and click with the mouse to select multiple items) and select to move the data to the Selected List. To return undesired data to the Master List, highlight the data (hold the Ctrl key and click with the mouse to select multiple items) and select .

  3. Select OK to accept the changes or Cancel to return to the QUERY BUILDER panel.
  4. To change the Operator, select (Operator). The Select an operator window is displayed.
  5. Select the desired operator from the Select an operator window, then select OK.

    The default condition is: = [Equal To]

    Manually entering data into any of the Standard Test Criteria fields changes the description of the operator button to CONTAINS.

    Exclusive to the Test Date, Engine S/N fields and Custom Test Criteria fields, a BETWEEN button may be selected for a range of dates or serial numbers. Both Test Date or Serial Number fields with an AND operator appear.

    Clicking inside the Test Date field automatically enters the present date and activates a drop-down button. Clicking the drop-down button activates a calendar; the field is automatically populated with any selection made. Clear Dates deletes all data within the Test Date field.
    Clicking on the month and year at the top of the calendar activates their respective selection features:

  6. If required, populate the Custom Test Criteria fields with the desired data. The Custom Test Criteria contains six fields with identical drop-down lists of Custom query data, from the Customer_Specific table in the proDAS database. An item that is selected from one list is automatically removed from the other lists. Similar to the Test Date and Engine S/N fields of the Standard Test Criteria, a BETWEEN operator may be selected by clicking on the operator buttons for a range of data. Buttons beside each field [] activate windows which list criteria that is unique to the item. Highlight the rows of desired data from the Master List (to highlight multiple items, hold the Ctrl key and click with the mouse) and select to move the data to the Selected List. To return undesired data to the Master List, highlight the data (to highlight multiple items, hold the Ctrl key and click with the mouse) and select .

  7. Select OK to accept the changes or Cancel to return to the QUERY BUILDER panel

    Similar to the Standard Test Criteria buttons, manually entering data into any of the Custom Test Criteria fields changes the description of the operator button to CONTAINS.

Executing Queries with the Query Controls Panel

The CLEAR ALL button in the QUERY CONTROLS panel deletes both unsaved queries and query results.

The REMOVE PRESENT QUERY button in the QUERY BUILDER panel deletes the displayed query.

To execute queries with the Query Controls panel:

  1. Select EXECUTE from the QUERY CONTROLS panel to retrieve data:

    The QUERY RESULTS panel displays the applicable tests:

  2. To add data from another query or replace with data from another query:
    • From the QUERY BUILDER panel, select Reset to clear the QUERY BUILDER fields. The present query is then stored in local memory.
    • Enter new query parameters in the QUERY BUILDER fields, either the Standard Test Criteria or the Custom Tests Criteria sections.
    • From the QUERY CONTROLS panel, select Replace to overwrite the present data within the QUERY RESULTS panel. Select Add to retain the present data.
    • Select EXECUTE.

    The selection buttons are used to display queries temporarily stored in local memory, not yet saved to disk (i.e. a query series).

Saving Queries with the Query Controls Panel

To save queries with the Query Controls panel:

  1. Select SAVE SERIES for the present query.

    A Saving the query... prompt window is displayed.

  2. If No is selected, enter a category name for the query and select OK.

  3. If Yes is selected, enter a query name and select OK.

  4. The query names are listed within the PREDEFINED QUERIES field.

Viewing Data with the Query Results Field

Engine test data from executed queries is displayed in the QUERY RESULTS panel.


Columns in the QUERY RESULTS are:

  • Test Cell
  • Test Date
  • Test Name
  • Test Description
  • Engine
  • S/N (Serial Number)
  • Engine Standard
  • Customer

The header bars in the QUERY RESULTS panel have functions similar to standard spreadsheet headers (i.e.: columns may be dragged laterally and placed in any order from left to right and clicking on a header bar will sort the corresponding data in ascending or descending order). Shift and Ctrl keys are used to select rows.

Viewing Queries with the Generated SQL Field

To view queries with the Generated SQL panel:

  1. The Generated SQL panel displays the Structured Query Language (SQL) code of the present query. To open the Generated SQL panel, select Admin/Show/Hide SQL from the drop-down list in the RAVE menu. The Generated SQL panel is displayed.

  2. The displayed code is assigned a Sub Query ID number ("1" in the example). Code sequences for other queries may be viewed with the query selection buttons . The code in the field may be edited in the Unlocked mode by toggling Lock SQL/Unlock SQL.

  3. While editing, the original code structure may be retrieved by selecting Restore SQL. After editing is completed, the new code is written to disk when Lock SQL/Unlock SQL is toggled to Lock SQL. If the updated code is incompatible with the QUERY BUILDER, a warning is displayed when Execute is selected.

Retrieving Existing Reports

To retrieve an existing report:

  1. Select a previously saved query from the Query & Report Categories drop-down list in the PREDEFINED QUERIES panel.
  2. Select the desired query from the Current Queries drop-down list in the PREDEFINED QUERIES panel.
  3. Select EXECUTE in the QUERY CONTROLS panel to retrieve data.
  4. Select the desired engine test from the QUERY RESULTS panel.
  5. Select REPORTS in the QUERY CONTROLS panel.
    The Reports screen is displayed.

  6. Highlight the desired template from the Single Engine Report Templates panel.
  7. Select VIEW to generate and display the .xls (Excel workbook) file.
    The .xls (Excel workbook) file is based on the Report Template and is populated with engine test data that has been extracted from the proDAS database.
  8. When VIEW is selected, the Duplicate Comments window is displayed, if the engine test data contains more than one comment with the same test step name.

  9. Highlight (select) the comments to be included in the report.
  10. When all the applicable comments have been highlighted, select Continue.
    The Single Engine Report - Duplicate Test Steps Found! window is displayed, if the engine test data contains more than one fullset with the same test step name.

  11. If duplicate test steps are found, select the test step that is to be included for the currently selected report to be generated.
  12. When the test step has been selected, select Continue.
    The Single Engine Report - Duplicate Test Steps Found! window is closed and the Report is generated in Excel.
  13. To print the generated report in Excel, select File>Print from the Excel toolbar.
  14. To close the report, select File>Exit form the Excel toolbar. The Reports screen in RAVE is visible.
  15. Select BACK on the Reports screen to return to the Test Selection screen.
  16. Select EXIT on the Test Selection screen to close the RAVE application and return to the User Interface Home screen.

    Select the PEEK button to display the last .xls (Excel workbook) file previously generated with the Report Template Contents configuration.

Creating a New Report

To create a new report (all four types of reports):

  1. From the Report screen, select REPORT BUILDER.
    The Report Builder Wizard - Start window is displayed.

  2. Select Next> to proceed to the Report Type window.
    The Report Type window is displayed.

  3. Select the Report or Logsheet radio button in the Report Type window.
    Four types of reports may be generated, depending upon selections in the Report Type and Data Source windows:
    • Steady-State (fullset)
    • Transient (Log)
    • Steady-State / Transient (Log) Combination
    • Logsheet
  4. Select Next> to proceed to the Data Source window for Steady-state or Transient reports.

    If Logsheet is selected, the Select Channels window is displayed.

  5. Select <Back to return to the Reports screen.

Report Configuration

The number and range of the worksheets in the report .xls file are dependent upon the selections that are made in the Report Builder Wizard. The contents of the report appear in the Report Template Contents field of the Reports screen. The sections in that file also correspond to selections that are made in the windows of the Wizard.
