User Security System (USS)

Overview

proDAS controls access to each of the proDAS applications via the User Security System (USS), ensuring that only an authorised user can login to a proDAS application. A common graphical user interface (GUI) login control is used to log in to the various proDAS applications.

The USS has a graphical interface that allows an administrator to manage the permissions and account information of proDAS users, resources, and resource users.

Launching a proDAS Application

When a proDAS application is launched, the USS login window is displayed.

The user must enter their username and password and then click Login. The login information (i.e.: Username and Password) and the identification name of the proDAS application is passed to the USS Server for validation. If the login information has been entered correctly, the USS Server validates the information, returns the login validation as true, and provides the security level (from 1 to 5) of the user. The security level of the user is extracted and the proDAS application is configured and started.

Launching Additional proDAS Applications

Once a user successfully logs into a proDAS application, they may be able to launch additional proDAS applications.

When an additional proDAS application is launched, the username and password are obtained from the login for the current proDAS application, and passed to the USS Server for validation.

The USS Server validates the information, returns the login validation as true, and provides the security level (from 1 to 5) of the user.

The second proDAS application extracts the security level of the user (the security level of the user determines the configuration of the application). The second proDAS application is configured and started.

If the USS Server is not available, any login, validation, or resource data attempt will fail.

proDAS applications can be launched from the command line. Launching from the command line accepts both encrypted passwords (/ep) and unencrypted passwords.

When an application is launched from an open application, the login information is encrypted and the .ini file reflects the encryption (/ep). When an application is launched from a shortcut or the command line, the login information is not automatically encrypted.

If the application does not start, contact the system administrator.

User Accounts and Passwords

The system administrator creates and modifies user accounts and passwords for the proDAS applications on the Management GUI Computer using the USS executable. Only personnel with the User Security System (USS) administrator password can add, edit or remove users, assign access levels, and perform system maintenance.

The system administrator assigns the appropriate access level to the different members of the engine test team. The password is case-sensitive.

There are six access levels, numbered from 0 to 5. The access levels determine what functions the user can or cannot use when interacting with the proDAS Applications. The security levels, in all proDAS applications, are structured so that the rights (permissions) of a specific security level include the rights (permissions) of all lower security levels. Each user is assigned a permission level for each proDAS application they have access to.

Users can have different access levels for the different applications they use.

Resource Users

Resource users manage the passwords for different categories of resources:

  • DSN – for accessing the Oracle database
  • Computer – for accessing the Linux PC or an SFTP site
  • Config Server – for accessing the config server

Resource users are created in the USS application. When a resource user record is added, the user can be tied to a regular user in proDAS or created as just a resource user.

Access Levels

The access levels of each application determine what functions each user can or cannot use when interacting with it. The USS is used to assign access levels to individual users, while the Management GUI is used to determine what each access level has permission to do within each application. Users with administrator privileges can use Management GUI to alter what the access levels for each application.

How to Add, Remove, or Modify User Accounts on proDAS

The following procedures detail the primary functions of the User Security System GUI. For more information on further functions of the system, see the descriptions in User Security System GUI.

  • Adding a New User to proDAS
  • Removing a User from proDAS
  • Changing a User's Password on proDAS
  • Changing a User's Access Level

Adding a New User to proDAS

Only the System Administrator (permission level 5 for the USS) can add new users to proDAS and assign user access levels for the proDAS applications.

To add a new user to proDAS:

  1. From the Users tab in the User Security System GUI, click Add (). A new line item appears in the list of users.
  2. Type in a username in the Name field and any notes required for the user.

    3. The user name must be at least three characters long.

  3. If the new user is to be a resource user, select the Resource User checkbox. If this checkbox is selected, the checkbox in the Resource User? column in the user list is automatically selected.
  4. Input the password for the new user in both Password fields. Ensure the passwords are identical. Click Set Password. The checkbox in the Password Set? column is automatically selected.
  5. Save the changes. The checkbox in the Password Set? column is deselected when changes are saved.

Removing a User from proDAS

User accounts should be removed if the users no longer need access to the system. Only the System Administrator (permission level 5 for the USS) can remove a user from proDAS.

To remove a user from proDAS:

  1. From the Users tab in the USS GUI, select the user to be removed in the user list.
  2. Click Remove (). The user is deleted from the system.
  3. Save the changes when required.

    4. The user will remain in the user list until the next save.

    When a user is deleted from the USS, all related user-application records are also removed from the Application Users tab.

Changing a User's Password on proDAS

Only the System Administrator (permission level 5 for the USS) can change a user's access level on proDAS.

To change a user's password on proDAS:

  1. From the Users tab in the USS GUI, select the required user in the user list.
  2. Change the password as necessary in the password fields. Click Set Password.
  3. Save the changes when required. The checkboxes in the Password Set? columns are deselected when changes are saved.

Changing a User's Access Level

Only the System Administrator (permission level 5 for the USS) can change a user's access level on proDAS.

To change a user's access (permission) level on proDAS:

  1. From the Application Users tab in the USS GUI, select the requisite user in the user list.
  2. Use the slider bar to change the permission level of the user.

    3. The permission level is only applicable to the application selected for that user. If the user requires permission for a different application, a new record must be created. To create a new record, click Add (). A new record must be created for each application the user requires permission for.

  3. Save the changes.

User Security System GUI

The User Security System (USS) provides an account management GUI where the user can view, create, and modify user accounts. The account management GUI is the primary user interface for the USS.

To access the USS account management GUI, navigate to C:/proDAS/bin and open the USS.exe executable file.

Log In Screen

The log in screen allows the user to enter the requisite username and password to log in to the USS account management system.

The user can remain logged in to the USS for 60 minutes. When there are four minutes or less remaining in the session, the user can extend the session by selecting File > Extend Session. The extend session window opens and the user can enter their username and password to extend the session for another 60 minutes.
