Calibrating Channels

Overview

Analogue channels are calibrated by injecting simulated signals from a calibrated source. A variety of techniques are then used to calculate the relationship between the input and output of the channel. This relationship is called the calibration function.

Analogue channels are calibrated periodically. The time between calibrations is dependent on the stability of the signal conditioning, test environment and transducers (if applicable).

Input signals are injected into the channel, and the calibration function calculated is displayed on the Calibration screen. Once a channel calibration has been completed, the signals are monitored, and by comparing the expected values to the recorded values on the screen, the operator can confirm that the calibration was successful. This step is called auditing and is performed after a calibration is complete.

When calibrating channels, a calibration reference source is used to simulate the input. For example:

  • a frequency channel requires a precision frequency signal from a source such as the HP 3245A Universal Source or Agilent 8904 Frequency Source
  • a general analogue input channel requires a precision DC voltage signal from a source such as the HP 3245A Universal Source, or Time DC voltage Source
  • a Resistive Temperature Detector (RTD) channel requires a precision resistance from a decade box

A polynomial calibration produces a "Best Fit" linearisation conversion of raw signals into engineering units. The polynomial coefficients of the "Best Fit" line are calculated by the Sensor Calibration Application (see the Multi-point Linear Calibration Calculation illustration below) and are stored in the proDAS database.

End-to-End Calibrations are used when the measurement chain is not expected to vary from test to test over a period of time. If a nonlinear correction is necessary, then End-to-End Calibration is required. End-to-End Calibrations can result in polynomial or breakpoint calibrations. In an End-to-End Calibration, a signal is injected at the start of the measurement chain and calibrated via proDAS (see the End-to-End Calibration illustration below).

The calibration function is calculated by SCA and the values are stored in the proDAS database. At run time, the values stored in the database are loaded into proDAS, providing the operator with the EU converted and corrected equivalent of the signal being measured. The incoming sensor calibration function is combined with the facility calibration function. proDAS uses the resulting calibration function to produce the value that is displayed on the operator screens (the figure below illustrates transducer outputs on proDAS).

The following two calibration utilities are available:

  • Sensor Calibration Application (SCA)
  • Auto Calibration Utility (ACU)

Sensor Calibration Application (SCA)

The Sensor Calibration Application (SCA) acquires, processes, and stores data for one or more sensor calibrations at the same time.

The SCA allows any number of sensors to be calibrated simultaneously, although the screen can only display a maximum of 32 sensors at once. Sensors that are calibrated simultaneously must belong to the same group and use the same set of applied target points.

The Sensor Calibration Application (SCA) can perform the following functions:

  • display sensor calibration information
  • perform single sensor calibration
  • perform multiple sensor calibrations
  • audit sensor calibrations
  • activate a previous calibration
  • print calibration reports
  • display a graph of calibration data
  • offset adjustment

Calibrations must be performed by a qualified technician trained in instrumentation calibration and in the use of calibrated signal sources.

The SCA can be launched:

  • from proDAS through the Management GUI Control Panel, where the username and password are automatically passed
  • as a stand-alone application where the username and password are not automatically passed

The Sensor Calibration Application (SCA) is used for the calibration and auditing of sensors in proDAS. SCA performs the following functions:

  • manual calibration
  • manual audit
  • offset adjustment
  • automatic calibration/audit
  • variable target calibration
  • customisation

Colour Definitions

The colour definitions differ between calibration and audit. The table below indicates the defined colour and it's description.

To access the Select Sensors to Calibrate/Audit window, from the Sensor Calibration window View menu, select Select Sensors To Calibrate.

For manual calibration, only sensors from the same group can be selected for calibration. If sensors from more than one group are selected, only automatic calibration is available.

Subgroups of groups can be defined using the period character in the names of calibration groups. The period character is interpreted as a delimiter between group levels (i.e.: the calibration group name "A.B" is interpreted as subgroup B of group A). It is possible to have multiple levels of groupings.

Performing a Manual Calibration

Calibration is used to determine new conversion curves called calibration functions.

Manual calibration can only be performed when the RTE is scanning. Before a manual calibration can be started, sensors from the same group must be selected. Each step of a manual calibration consists of a special tab. For details on each tabs function, see Manual Calibration window.

To perform a manual calibration:

  1. From the Sensor Calibration window View menu, select Select Sensors To Calibrate.
    The Select Sensors To Calibrate/Audit window is displayed.
  2. Select the sensors to calibrate by either double-clicking on the sensors or right-clicking on the calibration type node and selecting all sensors under that node. Once the sensors have been selected the calibration process can be started.

    Only sensors from the same group can be selected for calibration.

  3. From the Sensor Calibration window Calibration menu, select Manual Calibration... or
    select the Manual Calibration button.

    If the menu item or button is disabled (greyed out) then either no sensor is selected or sensors from different groups are selected. Ensure that only sensors from the same group are selected.

    Depending on the subsystem of the sensors and the application configuration, a startup check message may be displayed. For details on configuring startup checks, see Startup Checks.
  4. To confirm the startup check, select Yes. The Manual Calibration window is displayed.

    When opened, the Pre-Calibration Settings tab is displayed. If a default calibration definition file for the selected calibration group is defined, that file is automatically selected. Depending on the application configuration the ambient conditions may have been read from the corresponding proDAS channels.

  5. From the Sensor Calibration Definition File panel, select the correct file or (if no file is to be used for calibration) make sure no file is selected. For details on how to select a calibration definition file, see Pre-Calibration Settings.

    A Sensor Calibration Definition File is used to store the settings from the previous calibration of the current sensor type, to enable the same settings to be used automatically for repetitive calibrations.

  6. From the Calibration Equipment panel, select the calibration equipment that is to be used for the calibration. For details on how to select calibration equipment, see Pre-Calibration Settings. For details on how to edit calibration equipment, see Edit Calibration Equipment.
  7. From the Ambient Conditions panel, specify the ambient conditions.

    If the values read from proDAS channels are incorrect or if the values cannot be read, select the Manual Override checkbox and insert values and units for the ambient conditions. For details on how to specify ambient conditions, see Pre-Calibration Settings.

    Ambient condition channels are specified in the Sensor Calibration ini file.

  8. To accept the settings and proceed to the Calibration tab, select Next. The Calibration tab is displayed.

    The target value is set automatically if a calibration definition file is used. The example screen below is for a fixed target calibration.

  9. From the Measured Value panel, check the Sampling Average and if necessary, correct using the up/down arrows.
  10. From the Measured Value panel, check the Display Format and if necessary, correct using the up/down arrows.

    The left box displays the resulting display format. The middle box displays the number before the decimal and the right box displays the number of digits after the decimal.

  11. From the Measured Value panel, check the Tolerance and if necessary, correct using the drop-down box for selecting the type of tolerance, and the text box to specify the corresponding value.
  12. From the Measured Value panel, check the Standard Deviation and if necessary, correct.

    The time must be specified in seconds.

  13. From the Target Value panel, specify the target in the EU Value field.

    The target value can be read from the calibration definition file or directly inserted by the user. If the values are to be read from the file, use the arrow keys to browse forward or backward. By default, the first value of the file is used.

    When the measurement is accepted, the Increment value is used to calculate the next EU Value.
  14. Inject the signal to be measured and wait until the signal is stable.
  15. To accept the measurement, select Accept. If a new value is required, go to step 13. The Next button is available when at least two measured values are accepted.
  16. To complete the calibration measurement, select Next. The Calibration Results tab is displayed.

    For details on specifying how curves are generated, see Calibration Results.

    For details on viewing the measured data and the generated curves, see Calibration Results.

  17. To use the calibration results, select Next. The Audit Settings tab is displayed.

    An initial audit of the generated curves is required after a calibration.

    It is possible to include a second conversion (defined by the fixed sensor) when performing an audit (i.e. a channel that is calibrated to convert volts to Ohms, could then have a PT100 fixed sensor attached to display the audit results as a temperature).

  18. To apply the fixed sensor to the audit, check the Apply Fixed Sensors checkbox.
  19. To apply the audit settings, select Next. The Audit tab is displayed.
  20. From the Measured Value panel, check the Sampling and if necessary, correct using the up/down arrows.
  21. From the Measured Value panel, check the Display Format and if necessary, correct using the up/down arrows.

    The left box displays the resulting display format. The middle box displays the number before the decimal and the right box displays the number of digits after the decimal.

  22. From the Measured Value panel, check the Tolerance and if necessary, correct using the drop-down box for selecting the type of tolerance, and the text box to specify the corresponding value.
  23. From the Measured Value panel, check the Standard Deviation and if necessary, correct.

    The time must be specified in seconds.

  24. From the Target Value panel, specify the target value.

    The target value can be read from the calibration definition file or directly inserted by the user. If the values are to be read from the file, use the arrow keys to browse forward or backward. By default, the first value of the file is used.

  25. Inject the signal to be measured and wait until the signal is stable.
  26. To accept the measurement, select Accept. If a new value is required, go to step 24 . The Next button is available when at least one measured value is accepted.

    Measurements within tolerance are displayed in green and measurements out of tolerance are displayed in red. To redo an audit point, right-click on the result and select Reject Point from the context menu. Go to step 24 and re-enter the target value.

  27. To finish the Audit measurement, select Next. The Audit Results tab is displayed.

    For details on how to view the measured data and audit results, see Audit Results.

  28. To apply the audit results, select Next. The Store Data tab is displayed.
    For details on saving calibration and audit results to the database, see Store Data.
  29. From the Save Data to Database panel, check the Calibration Date, Due Date and if the calibration is not permanent, check the Calibration Expires checkbox.
  30. From the Save Data to Database panel, verify the name of the calibrator and the calibration comment. Correct or update the information as required.
  31. To store only valid calibrations to the database, select Save Only Passed Audits checkbox.
  32. To get a printout when saving the data, select Print Report When Saving checkbox.
  33. From the Save Data to Database panel, select Save to store the calibration and audit data. Confirm the confirmation question and success messages. The Print window is displayed.
  34. Select the printer to print the reports and select OK. The reports are generated and printed.
  35. Confirm the message that the reports were sent to the printer. The Finish button is now available.
  36. To save the calibration and audit settings for future use, specify the filename in the Save Calibration Steps to File panel. The file is used for all sensors in the same group.
  37. To finish the manual calibration, select Finish. A confirmation message is displayed confirming the manual calibration is finished, the data are stored to the database and reports have been generated.

Audit is used to check conversion curves and extend the due date.

Performing an Offset Adjustment

Offset adjustment is used to reduce the offset of measured channels. The offset is measured and the conversion curves are shifted by the measured offset.

Performing an offset adjustment has a similar workflow as performing a manual calibration with the following differences:

  • the user can specify if fixed sensors are to be applied (via Pre-Calibration Settings tab)
  • instead of a calibration measurement there is an offset measurement
  • only one measured value is accepted for offset measurement
  • audit is skipped
  • no new record is created when saving data, instead the existing tables are changed (no due date changes)

To perform an offset adjustment:

  1. From the Sensor Calibration window View menu, select Select Sensors To Calibrate.
    The Select Sensors To Calibrate/Audit window is displayed.

Performing a Variable Target Calibration

Variable target calibration refers to a calibration process whereby the injected signal (test value) may not be equal to the intended target value and is measured by a trusted device. In comparison, a fixed target calibration involves a precisely known injected signal (test value) equal to the target value.

During a variable target calibration, three parameters are recorded for each point:

  • the Target Value (set by the user)
  • the Test Value (injected signal)
  • the Measured Value (measured channel output)

A variable target calibration is automatically enabled when calibrating sensors containing a Raw Data Table of the type 3-dimensional break point table (3D BPT).

To perform a variable target calibration:

  1. From the Selected Sensors for Calibration/Audit window , select one or multiple sensors to be calibrated by double-clicking on the appropriate entries.
    Once the sensors have been selected the calibration process can be started.

    Only sensors from the same group can be selected for calibration.

  2. From the Sensor Calibration window Calibration menu, select Manual Calibration... or
    select the Manual Calibration button.

    An automatic variable target calibration can be performed provided the automation script has been defined.

    The Manual Calibration window is displayed.
  3. Pre-Calibration Settings tab:
    1. If available, select the appropriate Sensor Calibration Definition File (.scdf) and ensure that the Ambient Temperature and Humidity channels are specified.

      The ambient temperature and humidity are typically measured directly but can be manually overridden to constant values.

    2. Select the Calibration Equipment entry by double clicking on the relevant row in the Available Equipment table. (i.e.: during a thrust calibration, the calibration equipment selected refers to the traceable calibration (master) load cell calibration data.)
    3. Select Next. The Calibration tab is displayed.
  4. Calibration tab:
    1. The Test Value is regarded as a trusted value in the calibration. The test value can be set manually, by one channel, by two channels or by a file or script. Set the maximum allowable standard deviation of the test value.
    2. From the Measured Value panel, check the sampling average and correct it if necessary by using the up-down-buttons.
    3. From the Measured Value panel, check the display format and correct it if necessary by using the up-down-buttons. The left text box shows the resulting display format. The number before the point shows the minimum number of digits before the point. The number after the point defines the precision of the display format fraction digits.
    4. Check the tolerance and correct it if necessary. Use the list box for selecting the type of tolerance used and the text box to specify the according value.
    5. From the Measured Value panel, check the Standard Deviation and if necessary, correct. The time has to be specified in seconds.
    6. If the first Target Value is 0, the Zero button can be selected if it is required to eliminate any residual offset of the Test Value before accepting the first point.
    7. Calibration test point:
      1. Inject a test Signal and measure this with a trusted device such that it populates the Test Value (either by using 1 channel, 2 channels or by measuring independently and entering this value manually).
      2. At the same time, measure the same signal with the Calibration Sensor (i.e.: Inject the channel associated with the Calibration Sensor with the same signal).
      3. Record the intended value of the test signal (Target Value) in the EU Value field, (i.e.: pressure regulator setting, PLC-demanded hydraulic actuator thrust force), wait until the measured value becomes stable and accept the test point.

        The Target Value can be entered manually, set by file (if a calibration definition file was specified in step 3a), or set by increment, but should always equal the intended test signal value.

    8. Repeat the above step for the rest of the Target Values and select Next. The Calibration Results tab is displayed.
    9. To use the calibration results, select Next. The Audit Settings tab is displayed.
  5. Audit Settings tab:
    1. To apply the fixed sensor to the audit, check the Apply Fixed Sensors and Customer Sensors checkbox in the Audit Settings tab.
    2. To apply the audit settings, select Next. The Audit tab is displayed.
    3. Audit tab (test point):
      1. Inject a test Signal and measure this with a trusted device such that it populates the Test Value (either by using 1 channel, 2 channels or by measuring independently and entering this value manually).
      2. At the same time, measure the same signal with the Calibration Sensor (i.e.: Inject the channel associated with the Calibration Sensor with the same signal).
      3. Record the intended value of the test signal (Target Value) in the EU Value field, (i.e.: pressure regulator setting, PLC-demanded hydraulic actuator thrust force), wait until the measured value becomes stable and accept the test point.

        The Target Value can be entered manually, set by file (if a calibration definition file was specified in step 3a), or set by increment, but should always equal the intended test signal value.

        1. Once the subset of points have all been accepted, select Next. The Audit Results tab is displayed.
        1. From the Audit Results tab, select Next. The Store Data tab is displayed.
  1. Store Data tab (save the calibration and audit results to the database):
    1. If the Audit passed (green checkmark at the top of the Audit Results tab); in the Save Data to Database panel, enter the calibrator name, comments and set the calibration due date.
      If the Audit failed (red cross at the top of the Audit Results tab); when saving, ensure to enter an Audit Failed comment, the calibrator name, when saving the calibration.
    2. Check the calibration due date. If necessary, set the calibration due date and if the calibration is not permanent, select the Calibration Expires checkbox.
    3. To get a printout when saving the data, select the Print Report When Saving checkbox.
    4. If Save Only Passed Audits is checked, only valid calibrations are saved to the database.
    5. To apply the calibration and create a new calibration record in the database, select Save.

      Without having to save the calibration, in the Calibration Report section you may choose the appropriate report format and select Create to create a report.

    6. Store the calibration steps (including the target value increments) to a file for future use. Saving the steps allows the calibrator to use these predefined target values during subsequent calibrations. To save the steps to a file, specify the path and file name in the Store Calibration Steps to File section and select Save. This file could be selected in Pre-Calibration Settings to repeat the procedure for future calibrations.

      It is possible to update an existing file if changes are made during the calibration that are to be kept for future calibrations.

    7. To finish the calibration, select Finish.

Application INI File

The application INI file is called SensorCalibration.ini.

GUI Language

The GUI language is specified in the application INI file. To set the GUI to English use the following:

  [Localisation] Language=en-CA

To set the GUI to German use the following:

  [Localisation] Language=de-DE

Channels for Reading Ambient Conditions

Channels used for reading the ambient conditions are specified in the application INI file in the following section:

  [Calibration]
  …
  AmbientConditions.Temperature.ChannelName=
  CategoryName.Temperature=Temperature
  AmbientConditions.Temperature.Unit=DegC
  AmbientConditions.Humidity.ChannelName=
  CategoryName.Humidity=Humidity
  AmbientConditions.Humidity.Unit=grn
  …

When a channel name is specified, the application obtains the value from that channel and uses the units defined for that channel. When no channel name is defined, the user is prompted to enter the temperature and humidity, and specify the applicable units from a pull-down list based on the EU category name.


Extending Functionality Using Triggers

Sensor Calibration supports triggers whereby for each trigger, scripts can be invoked. The script can be written in either VBScript or VB.NET or C#. For scripts to work the application INI file must specify:

  • the folder where the scripts reside
  • the names of scripts to be invoked
  • the assemblies to be used (required for VB.NET or C#)
  [Triggers]
  RootFolder=D:\proDAS\Data\SensorCalibration\Scripts
  Invoke_OnProcessStepChanged = AtpSetDueDate.vb;TestTriggers.vb

  [ScriptExecution]
  NumberOfAssemblyReferences=4
  AssemblyReference0=C:\Windows\Microsoft.NET\Framework\v4.0.30319\System.dll
  AssemblyReference1=C:\Windows\Microsoft.NET\Framework\v4.0.30319\System.Data.dll
  AssemblyReference2=C:\Windows\Microsoft.NET\Framework\v4.0.30319\System.Drawing.dll
  AssemblyReference3=C:\Windows\Microsoft.NET\Framework\v4.0.30319\System.Windows.Forms.dll

To check whether a script can be compiled and executed:

  1. From the Data menu, select Check Script.

Calibration Logs and Reporting

The Sensor Calibration Application INI file can be modified to specify the root path where calibration logs are stored.

  [Report.Calibration]
  XML.RootPath=D:\proDAS\data\SensorCalibration

As a minimum, calibration logs (in a sub folder Logs) are stored under the root path.

The Sensor Calibration Application INI file can be modified to specify the location of:

  • group INI files for Multiple Channel Calibration
  • Sensor Calibration Definition Files
  [SensorCalibrationDefinitions]
  GroupIniPath = D:\proDAS\data\Calibration
  CalibrationDefinitionsRootPath = D:\proDAS\data\SensorCalibration\Calibration Definitions

Startup Checks

Whether a manual or an automatic calibration is being performed, the user can be forced to confirm a startup check prior to starting the calibration. A startup check is specific for each subsystem type and is configured in the application INI file in the section “Calibration”. The entry for the according subsystem follows the pattern “StartCheck.<Subsystem Type>”

  [Calibration]
  …
  StartCheck.GASSVXI=Auto calibration performed?
  StartCheck.HSV=Auto calibration performed?
  …

The startup checks are used to display a message on screen to remind the calibrator of something that should have been done before proceeding with the calibration. For example, prior to performing a GASSVXI calibration, the calibrator is reminded to calibrate the VXI cards using the ACU utility.

Excel Calibration Reports

The Excel export feature is configured using the ReportGenerator Excel.ini.

  [Excel Export]
  FolderGroupTemplates=D:\proDAS\data\SensorCalibration\Configuration
  FormatFile_Linear_FixedTarget=D:\proDAS\data\SensorCalibration\Reports\Templates\AtpPolyFixed.xlt
  FormatFile_Linear_VariableTarget=
  FormatFile_Polynomial_FixedTarget=D:\proDAS\data\SensorCalibration\Reports\Templates\AtpPolyFixed.xlt
  FormatFile_Polynomial_VariableTarget=
  FormatFile_Breakpoint_FixedTarget=D:\proDAS\data\SensorCalibration\Reports\Templates\AtpBpVariable.xlt
  FormatFile_Breakpoint_VariableTarget=D:\proDAS\data\SensorCalibration\Reports\Templates\AtpBpVariable.xlt
  FormatFile_Default=D:\proDAS\data\SensorCalibration\Reports\Templates\Default.xlt
  NameDataSheet=CalData
  NameAuditSheet=Audit
  NameAdjustmentSheet=Calibration
  CreateSheetWithReferences=True
  AddSerialNumberToFileNames=True

The AddSerialNumberToFileNames=True parameter determines where the serial number is placed in the report file. Setting it as false places it at the end of the file name. Setting it as true places it at the beginning of the file name.

When exporting sensor data into an Excel sheet, the "Excel" export uses a template. The template is selected by using the following order:

  • look for group template (name should be Template.<calibration group>.xlt, e.g. Template.NiCrNi.xlt for group NiCrNi) – the folder of those templates is set by the INI file entry "FolderGroupTemplates"
  • combination of curve type (linear, breakpoint, polynomial) and calibration type (fixed target or variable target)
  • default template

Specify the default template when the same template is required for all calibrations.

When saving an Excel report file, the default directory for selecting SCDF files to open gets updated to the path where the Excel report is saved.

Diagnostic and Maintenance Requirements

The Sensor Calibration Application uses the system.txt file as a trace file to track all necessary information, such as:

  • System Trace - a unified file of all the other logs
  • SQL Trace - records all SQL statements, with return values

The Sensor Calibration Application incorporates extensive error checking. Wherever possible, the Sensor Calibration Application attempts to continue operation if an error occurs.

Error checking, in general, is performed when changes are applied.

All error messages include the time, the module ID, the source of the error and a description of the error. These error messages are logged to the trace file indicated in the calibration.ini of the Management Interface Computer.

Common Oracle Error messages are mapped to user-friendly English text messages. These text messages are displayed whenever a defined error occurs. If an error message is not mapped to an English text message, then the Oracle Error is displayed.

A consistency check is done only when changes are applied. Errors, detected during a consistency check, are displayed in a single message box.

Security Requirements

Security for the Sensor Calibration Application is modeled after the security for the Management Interface GUI.

All functions have unique security keys and associated levels.

The security key for a function key is not found in the security list, then the item is disabled.

All users are assigned a security level from 1 to 4, with level 4 providing the highest access.

If the security level of a user, currently logged into the Sensor Calibration Application, matches or is greater than the security level of an item, then the function is enabled, otherwise the item is disabled for that user.

Auto Calibration Utility (ACU)

The Auto-Calibration Utility (ACU) resides on the Host Computer and communicates with the VXI chassis.

The utility can either trigger the internal calibration of the reference voltage and resistance on the E1413, E1415 and E1419 VXI cards or, an internal calibration or zero.

The application is a stand-alone program that does not interface with proDAS and can only be run when proDAS is not configured.

ACU should not be started/used while proDAS is configured.

The ACU communicates with a multimeter via a IEEE 488 to communication interface. The ACU can command an internal calibration of the multimeter. The date and time of the last internal multimeter calibration is stored.

The ACU application:

  • detects if a card is being used by proDAS
  • scans the VXI chassis for applicable hardware (VXI Technologies E1413, E1415 and E1419)
  • obtains card type and identification information (serial number) from each E1413, E1415 and E1419
  • allows the user to interact with the utility via selectable menu options
  • provides access to a particular card, or all cards, for calibration and configuration
  • reads the existing calibration data from onboard flash memory
  • allows the user to perform either an auto-zero or full calibration
  • saves the new calibration values into flash memory on each VXI Technologies E1413, E1415 and E1419 cards
  • produces a test report, that is saved on the Host Computer

The Auto-Calibration Utility (ACU) is located in the standard MDS applications directory (as shown below):

/users/RTE/bin/exe/acu

The ACU program also requires an associated configuration file (e.g.: config.acu) representing the VXI hardware configuration (as shown below):

/users/RTE/bin/exe/config.acu

Reports and application traces, produced when the program has completed a calibration, are stored in the standard MDS traces directory (e.g.: trace.acu) as shown below:

/users/RTE/bin/tmp/trace.acu

/users/RTE/bin/tmp/cal_report.yyyymmddhhmmss

where:

  • yyyy is the year (e.g.: 2003)
  • mm is the month of the year (e.g.: 04)
  • dd is the day of the month (e.g.: 20)
  • hh is the time in hours - 24-hour clock (e.g.: 15)
  • mm is the time in minutes (e.g.: 24)
  • ss is the time in seconds (e.g.: 10)

For example:

/users/RTE/bin/tmp/cal_report.20030420152410

ACU Configuration File

The ACU application, when run, scans the hardware in the VXI chassis and configures all of the VXI cards with the information contained in the configuration file.

The configuration file must match the installed Signal Conditioning Plug-On (SCP) modules on each VXI card.

For each VXI card in the chassis, the proDAS configuration file must contain the following information:

  • chassis number
  • slot number
  • VXI base addresses for each card (A16 and A24)

Do not change the base addresses.

For each SCP installed on the VXI card, the ACU configuration file must contain:

  • SCP number (0-7)
  • SCP type (HP 1503, HP 1502 or HP 1510)
  • channel gain and filter cutoff frequency (applicable to the HP 1503 and HP 1510 only)

Example of an ACU Configuration File

The ACU configuration file is located in the /user/RTE/bin/exe directory and is called config.acu (see the example ACU configuration file below).

The Report File (e.g.: cal_report) and the Trace File (e.g.: trace.acu) names are defined in the ACU configuration file (e.g.: config.acu). The cal_report and trace.acu files are stored in the tmp directory (see the preceding excerpt of the ACU Configuration File). For example:

CI report_path/users/RTE/bin/tmp
CI report_name cal_report
CI trace_dest trace.acu

  • DD is the day of the month (e.g.: 20)
  • hh is the time in hours - 24-hour clock (e.g.: 15)

Manual Reference Calibration

The Manual Reference Calibration has two options for calibration: resistance calibration and voltage calibration. The resistance calibration is accessed first with the voltage calibration coming after. They can be performed individually or together. To initiate a Manual Reference Calibration of a specific card:

  1. Select Manual Reference Calibration from the main ACU menu.
  2. Select Calibrate resistance for the module and press y to continue. A 4-wire measurement of the resistor will be made by connecting to the V H, V L, Ohm H and Ohm L terminals on the calibration bus.

    3. Pressing n will bypass the resistance calibration and begin voltage calibration, described starting at step 6.

  3. Enter the resistance for the module in ohms.
  4. Verify the resistance percentage for the module and press y to continue. If the resistance percentage needs to be altered, press n to re-enter the value.

    5. The value can be re-entered up to 3 times.

  5. Press y to continue to the voltage calibration. Press n to skip the voltage calibration. If skipping voltage calibration, go to step 9..

    6. Zero and full-scale measurements will automatically be made for the following ranges: 0.0625 V, 0.250 V, 1.000 V, 4.000 V and 16.000 V. The full-scale output voltage of the calibration source will be approximately 90% of the nominal value for each range, except for the 16 V range where the output will be approximately 10 V.

  6. Enter the measured zero voltage range for the module (this is measured in V).
  7. Verify the zero voltage %f for the module and press y to continue. Press n to re-enter the measured value.
  8. Enter the full-scale voltage range and confirm by pressing y. Press n to re-enter the measured value.

    9. The value can be re-entered up to 3 times.

  9. The ACU will prompt the operator to save the calibration. To save, press y. To not save, press n.
