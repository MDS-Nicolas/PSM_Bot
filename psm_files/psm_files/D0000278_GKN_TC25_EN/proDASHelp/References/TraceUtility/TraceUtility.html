proDAS Trace Utility

Overview

The trace utility logs informational messages to a trace file that can be used to troubleshoot problems encountered. All clients of the Trace Utility on the same computer may use the same trace file for logging purposes, ensuring that all messages can be viewed in the same file. The trace files are stored to the proDAS\data\Trace directory and may be reviewed using any text editor (e.g.: Notepad).

The trace information written to the trace file consists of:

  • date and time
  • module identification
  • source
  • description
Trace Information Description
Date and Time Automatically added by the Trace Utility. The date and time are in the format:

DD-MM-YYYY HH:MM:SS
Module Identification A global module identifier, indicating the module that is generating the trace message. An applications identifier can be found in its initialisation file.
Source An identifier of the method (function) within the module that is generating the trace message.
Description The detailed trace message.

The Trace Utility provides automatic backup of the trace file. When the trace file exceeds the default value (i.e.: 0x80000) set by the Trace Utility, the Trace Utility searches the %temp%, %tmp%, and then Temp directory in the root directory on each logical hard drive. The first existing directory found is used as the default value of the BackupPath key. If none of the directories exist: the key is left empty, the Backup feature is disabled, and the operator is notified.

The backup destination file name is SYSTEM<yyyymmddhhiiss>.txt, where:

  • SYSTEM - the word system in all upper-case
  • yyyy - is a four digit year (e.g.: 2011)
  • mm - is a two digit month (e.g.: March would be 03)
  • hh - is a two digit hour (e.g.: 2 pm would be 14 and 3 am would be 03)
  • ii - is two digit minutes (e.g.: 1 minute would be 01)
  • ss - is a two digit second (e.g.: 3 seconds would be 03)
  • txt - is the file extension

The backup feature is disabled if either the initialisation file is missing or the necessary information in the initialisation file is incorrect.

Further information on the Trace Utility can be found in ICD78031.2698, COM Interface Control Document for Trace Utility.
