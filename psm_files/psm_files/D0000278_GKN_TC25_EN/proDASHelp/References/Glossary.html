Definition of Terms

Term Definition (Items in bold type are also defined in the table)
Action An action defines the RTE automatic response when the value of a limit-checked channel exceeds a user-defined limit. For example, the action could be to write a message to the Event Log, and to start a User Log.
Alarm An alarm notifies the Operator that the value of a limit-checked channel has reached or exceeded a predefined limit. Alarms are defined in the channel editor, and must be acknowledged by the operator. Examples of alarms include High Alarm (HIHI), High Warning (HI), Low Warning (LO) and Low (LOLO).

See also Action and Limit.
Analogue Channel An analogue channel receives continuous electrical signals from a signal source (transducer). The information supplied by the signal varies in direct proportion to the intensity of the signal.
Analogue-to-Digital Conversion Analogue-to-digital (A/D) conversion occurs when a signal from an analogue channel, monitored for voltage, frequency, or resistance is converted to a digital signal that can be interpreted by proDAS.
Autostart Log Autostart Logs are transient logs that start recording automatically when the RTE begins monitoring an engine test.
Break-point Calibration A calibration method in which a transducer's characteristic is represented by several linear segments over a calibration range.
Break-point Table A breakpoint table contains sets of corresponding values to allow conversion of values and units. The table may be stored in the TRSCDB or the BPT Editor. For example, a breakpoint table could be used to convert RTD voltages to temperatures. This voltage could then be recorded or used in a calculation.
Calibration The operation of applying a physical phenomenon to a transducer or simulating a transducer's output by electrical means over the full range of the transducer to convert an electrical signal to engineering unit values.
Channel A channel is the basic structure for collecting, storing and processing data. Normally, it consists of an instrumentation signal source (e.g. transducer), a storage location in the current value table (CVT) for the data, and all of the hardware (cables, signal conditioners, converters...) and associated software between them.

Channels can be used to transmit data to other systems. In this case, the channel signal source is the CVT, and the final destination is a storage location in the receiving system.

Channels can be used to provide communications between the RTE, other systems such as a Facility Control System or a Engine Management System.
Channel Definition The channel definition describes how data is received, stored, processed, and/or output by a proDAS channel. Each channel has a unique channel definition.

Channel definitions are stored in the database and can be edited with the Channel Editor and Subsystem Editor.
Channel Pinout Pinouts allow a physical configuration of input and output data channels on a piece of hardware. Pinouts are configured to ensure consistent connectivity between hardware sub-assemblies within a system.
Computer Data Acquisition System A computer system that receives digital signals and converts them into engineering unit values that can be stored and logged.
Critical Channel A critical channel contains data that is extremely important to the test, or which has safety implications. Any channel may be defined as a critical channel.

Data from critical channels is continuously recorded in the critical log.
Critical Log The Critical Log stores data from all the critical channels in a revolving (first-in, first out) buffer. At any time the operator can save the log, and the data in the buffer will be transferred to permanent storage.

The log can be set to save data from before and after an event. For example, a log could save the three minutes of data before the save command is issued, and the next two minutes of data after the save command. This allows analysis of data before and after a particular event or problem.
Current Value Table (CVT) The CVT contains the current values of all channels, both physical hardware channels and calculated channels.

The CVT stores these values and displays them on the Primary and Secondary screens, uses them for calculations or alarm handling, and saves them as Fullsets, transient logs or critical logs.
Data Acquisition Data acquisition is the process of receiving data from instrumentation, processing and displaying the data, and recording it for analysis.
Data Acquisition System A data acquisition system is a computer system that collects, processes, stores, retrieves, and analyses data from instrumentation systems.
Deadband Deadband stops repeated alarms caused by fluctuations in the value of a critical channel, when the value is near the trigger point of an alarm. The deadband is half the size of the amplitude that the signal is allowed to fluctuate before it will trigger a new alarm.
Digital Signal A signal that can be interpreted by a data acquisition system as a digital value.
Discrete Channel A Discrete Channel is a channel that registers only an on/off, zero/one, or true/false value.
proDAS A software and hardware system for collecting, storing and analysing engine test data.
Engine Test Log Sheet The Engine Test Log Sheet is a viewable and printable record of Fullsets and log comments recorded during an engine test.
Engineering Unit In general, an engineering unit could be defined as any human-readable format resulting from a technical measurement. An engineering unit (EU) is a unit that would be recognised by the user in terms of engine testing or performance (e.g.: temperature in degrees or pressure in psi).
Event Log The Event Log displays and records all proDAS events such as operating messages, warnings, alarms, operator acknowledgments of alarms, and operator comments.
Float Channel A float channel registers a continuous range of values (as opposed to a discrete channel).
Fullset A Fullset is a single point of averaged data from all the channels being monitored during the current test, including all calculated channels. The operator can define the length of time over which the data is averaged as well as a test step identifier for easy retrieval.
Host Computer The Host Computer is the computer on which a software application is installed.
Limits The values that define a range of normal operation within a system. Data inputs that exceed limit values will initiate warnings or alarms. Limits may be variable or sliding by associating the value with a value from another channel.
Log Refer to Transient log.
Master Configuration A Master Configuration is a stored set of channel configurations. Selecting a master configuration defines the subsystems, channels, logs and display pages that the RTE monitors during an engine test.
Multi-point Linear Calibration A calibration method of acquiring two or more points and performing a linear regression analysis to generate a slope, offset and quality of fit statistics: mean squared error and R2.
Operator Page Operator Pages are dynamic graphical displays of engine test data monitored by the RTE during an engine test. Operator pages may also contain controls for changing the state of test equipment.
Scanning During an engine test, the RTE continually scans (monitors and records) engine test data.
Acquisition Rate The frequency (in Hz) at which the channel data in the CVT is updated.
Servicing The inspection and maintenance of equipment, usually on a periodic or scheduled basis.
Signal Conditioning The preparation (excitation, amplification, filtering and analogue-to-digital conversion) of raw data for interpretation by the RTE.
Signal Source A device that generates an electrical signal in response to measuring a physical quantity (such as a temperature or pressure) for interpretation by the RTE.
Subsystem A Subsystem is a discrete group of instrumentation, such as the General Analogue Subsystem, that supplies engine test data, support facility, or infrastructure data to proDAS (see also Subsystem Configuration). Subsystems are normally concerned with a specific type of related data, such as temperature or vibration. Note: in general discussions, "system" and "subsystem" are frequently used interchangeably, depending on the context of the conversation.
Subsystem Configuration A Subsystem Configuration consists of a set of channel definitions that define how the RTE processes the data from a subsystem. In this case, "subsystem" refers specifically to a subset of a master configuration.
Transient Log Transient logs are user-defined time-based records of data acquired by selected channels. The Transient Log definition specifies the name of the log, the time that the log runs, the sampling frequency, and the channels recorded by the log.
Two Point Linear Calibration A calibration method of acquiring two points to generate a slope and offset for engineering unit conversion.
User Log Refer to Transient Log.
Warning A warning notifies the operator that the value of a channel has reached a warning limit. Warning limits are defined in the channel definition and are recorded in the Event Log. Warnings may have to be acknowledged by the operator depending on how the system was configured.
Zero The operation of removing an offset of an analogue input signal by adjusting the engineering unit conversion at the static zero condition.
