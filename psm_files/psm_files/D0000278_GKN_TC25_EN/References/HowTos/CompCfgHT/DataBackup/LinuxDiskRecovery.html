How To – Restore an Entire Disk Partition That Does Not Contain the Operating System

This How To provides the steps required to restore an entire disk partition on the Linux Host Computer.

Restore an Entire Disk Partition That Does Not Contain the Operating System

In the event that the user is able to start up the Linux Host Computer but the proDAS data contained on the non-system disk has been lost or corrupted, it is possible to restore an entire disk partition.

  1. Log in as the root user on the Linux Host Computer.
  2. Start Acronis True Image Echo Server for Linux by one of the following methods:
    • click the Acronis icon on the Linux Host Computer
    • from the command prompt, type trueimage

    3. The Acronis GUI starts.

  3. Click the Recovery task from the Acronis main window.
    This launches the Acronis Restore Data Wizard.
  4. Select the Sunday full backup archive file from where the data files need to be restored.
    The Acronis Restore Data Wizard permits full backup, or any of the associated incremental backup archives from which to restore the data.
  5. Select Restore Disk or Partitions.
  6. Select the partition to be restored.
  7. Select a target partition for the restore.
  8. Select Logical for the restored partition type.
  9. Select Linux Ext 3 as the restored partition file system.
  10. Specify the size and location of the restored partition.
  11. Select the option No, I do not to restore the selected partition only.
  12. Choose the restore options to either Use default options or Set the options manually.
  13. Click Proceed to start the restore job.
    The data is restored.

For more information see, the Acronis True Image Echo Server for Linux User Guide.
