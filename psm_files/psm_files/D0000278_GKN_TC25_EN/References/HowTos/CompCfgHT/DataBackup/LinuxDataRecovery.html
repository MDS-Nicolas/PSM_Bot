How To – Recover Data on the Linux Host Computer

This How To provides the steps required to perform data recovery on the Linux Host Computer.

Recover Data Files

In the event that a file is accidently deleted, use the following procedure to recover the data.

This procedure should be repeated for every group of files that is located in a different target directory.

  1. Log in as the root user on the Linux Host Computer.
  2. Start Acronis True Image Echo Server for Linux by one of the following methods:
    • click the Acronis icon shortcut located on the main display
    • from the command prompt, type trueimage

    The Acronis GUI starts.

  3. Select the Recovery task from the Acronis main window.
    This launches the Acronis Restore Data Wizard.
  4. Navigate to the archive files for the missing data.
  5. Select the Sunday full backup archive file from where the data files need to be restored.

    6. If the file to be recovered was deleted between Saturday and Sunday, select the Saturday full backup. If the file was deleted at another time during the week, select the Sunday full backup to obtain the list of available files.

    The Acronis Restore Data Wizard lets you select the full backup or any of the associated incremental backup archives from which to restore the data.

  6. Select the option to restore specific files or folders.
  7. Select the target directory to restore to.
  8. Select the specific files to restore.
  9. Select the remaining task options based on user preference and then select the Restore Data job.
    The data is restored.

To restore a file from an incremental backup, the referenced full backup is also required.

An error is reported by Acronis if an incremental backup archive is selected but the corresponding full backup archive is no longer available. For example, once a new full backup is performed on Sunday, all the previous incremental backup archives become invalid.

For more information see, the Acronis True Image Echo Server for Linux User Guide.
