How To – Switch to the Spare Host Computer (If Available)

This How To provides the steps required to switch to a spare Host Computer if one is available in the facility.

If all the hard disks are functional but the main Host Computer still fails to start, the problem may be the motherboard. In this case, switch to the spare Host Computer by replacing all the hard disks on the spare Host Computer with all the disks on the main Host Computer.

Label all disks on both computers before doing the replacement.

The disk replacement must be done one by one on the same slot on both computers. Do not mix them up.

After switching to the spare Host Computer, the four network interfaces need to be re-activated as their MAC addresses are different from the main Host Computer.

To reactivate the four network interfaces on the spare Host Computer:

  1. In the Linux Command Line window, log in as root.
  2. Make a copy of the /etc/hosts file (for example, use date as part of the file name):
    cd /etc
    cp hosts hosts.Jan052009
  3. Reactivate network interfaces:
    1. Click Applications->System settings ->Network.
    2. Select eth3.bak, then click Delete to delete the eth3.bak.
    3. Select eth2.bak, then click Delete to delete the eth2.bak.
    4. Select eth1.bak, then click Delete to delete the eth1.bak.
    5. Select eth0.bak, then click Delete to delete the eth0.bak.
    6. Select eth0, click Edit.
    7. Select Statically set IP addresses.
    8. Fill in the fields with the following addresses:
      Address:10.86.102.2
      Subnet mask: 255.255.255.0
    9. Click DNS, type in acae1-rte in the Hostname field.
    10. Select eth1, click Edit.
    11. Select Statically set IP addresses.
    12. Fill in the fields with the following addresses:
      Address: 10.86.101.2
      Subnet mask: 255.255.255.0
    13. Select eth2, click Edit.
    14. Select Statically set IP addresses.
    15. Fill in the fields with the following addresses:
      Address: 10.86.103.2
      Subnet mask: 255.255.255.0
    16. Select eth3, click Edit.
    17. Select Statically set IP addresses.
    18. Fill in the fields with the following addresses:
      Address: 10.86.104.2
      Subnet mask: 255.255.255.0
    19. Select eth0, click Activate.
    20. Select eth1, click Activate.
    21. Select eth2, click Activate.
    22. Select eth3, click Activate.
    23. Click File > Save.
    24. Click File > Quit.
  4. In the Linux Command Line window, restore the hosts' file using the following commands:
    cd /etc/
    cp hosts.Jan052009 hosts
  5. Reboot the system using the following commands. This will verify that the network interfaces are reactivated and the system is functioning properly.
    cd /
    shutdown -r now
