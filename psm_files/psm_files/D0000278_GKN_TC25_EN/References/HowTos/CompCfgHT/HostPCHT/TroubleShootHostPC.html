Troubleshooting

This section provides common troubleshooting steps for the Host Computer.

It contains procedures for:

  • If the Host Computer Freezes
  • If the proDAS Software on the Host Computer Freezes
  • Recovering from a Component Failure
  • Recovering from a Communications Failure
  • Error Message - Socket Bind Error

 

If the Host Computer Freezes

  1. Reboot the Host Computer.
    To reboot the Host Computer, see How To – Reboot the Host Computer.
  2. If the Host Computer does not reboot, there is a system problem, such as a loose or damaged component.
    For system problems, see the Red Hat Enterprise Linux vendor documentation:
    • Red Hat Enterprise MRG 1.2 Realtime Installation Guide
    • Red Hat Enterprise MRG 1.2 Realtime Reference Guide
    • Red Hat Enterprise Linux 5.2 Installation Guide
    • Red Hat Enterprise Linux 5.2 Deployment Guide

If the proDAS Software on the Host Computer Freezes

If the proDAS software freezes, proDAS should restart automatically and the Management GUI should reconnect.

If proDAS does not restart automatically:

  1. Log in as root.
  2. Enter ps -el | grep proDAS. A list of processes is displayed.
    If the proDAS session is one of the processes that is displayed on the list, contact MDS.

If the proDAS session is not one of the processes that is displayed on the list, restart the session with the following procedure:

  1. Log in as root.
  2. Enter /etc/init.d/runprodas start on a command line.
  3. Enter ps -ef | grep proDAS.
  4. If proDAS does not start, contact MDS.

Recovering from a Component Failure

If there is a component failure on the Host computer, a message is displayed on the System Console.

To recover from a component failure:

  1. Troubleshoot the failed component.
  2. After the Host computer is repaired, it is recommended that the entire proDAS system be powered down and restarted. For details, see How To – Power Down the Instrumentation System.

Recovering from a Communications Failure

If a communication error message is displayed on the Management GUI Computer as a result of a communications failure between the Management GUI Computer and proDAS:

  1. On the Management GUI Computer, return to the Management GUI Home screen and select Quit.
  2. Close all other running applications.
  3. Reboot the Management GUI Computer. proDAS exits and then restarts automatically within 10-20 seconds.
  4. On the Management GUI Computer, restart the proDAS Management Interface application and login to proDAS.
    Communications between the Management GUI Computer and the Host Computer should now be restored and the engine test can be continued.

Error Message - Socket Bind Error

If the server exits (Host Computer) while the client(s) (Windows computers) are still connected, the operating system places the server socket for the connection to the client into a transition state: fin_wait_2.

The time out for the transition state is 13 - 14 minutes.

If a socket bind error message is displayed when starting proDAS or any server, either reboot or wait 15 minutes and retry.
