Running an Engine Test

Overview

This module is designed to be used as a reference guide in the everyday work of the Test Cell operator. The different operator functions that can be performed while nxDAS is scanning and acquiring data are explained through step-by-step procedures.

This section contains information on the following topics:

  • Preparation Tasks
  • Master Configuration
  • Pre-Test Checks
  • Configurable Displays
  • Logging Events
  • Test Page
  • Test Procedure Scripts
  • Timer
  • Data Logging
  • Modifying Channel Parameters
  • Reports

Preparation Tasks

Before an engine test can be started and data acquired, there are several set up tasks that may need to be performed.

The following table identifies a list of preparation tasks that might apply and the associated sections that describe the desired task:

Task Description References

Performing Thrust Calibration Perform an audit of Thrust Measurement System calibration.  • Thrust Calibration Application

Master Configuration

After all the appropriate tasks have been completed, nxDAS must be configured with a master configuration. The master configuration is selected using the Operations Manager (See How To – Set a Configuration as Active). The active configuration is loaded using the Operations Manager in Acquisiton Mode.

Pre-Test Checks

nxDAS gives the operator the ability to run various pre-test checks prior to acquiring data. These tests allow the user to perform functions such as confirming that channel readings are within the appropriate ranges and purging of the pressure scanner lines.

The execution of the pre-test checks depends on Customer hardware and procedures. For details on the pre-test checks, see Instrument Controls.

Please refer to the "Home" section of the Product Support Manual for details on your applicable pre-run checks.

Configurable Displays

nxDAS provides several user configurable displays that can be used to view the current data during an engine test, including:

  • nxViz Runtime
  • Spotlights
  • Tabular Channel Display (TCD)

Logging Events

nxDAS performs logging (data recording) of all system event messages including operator interactions, alarm conditions, warning messages, and error messages.

Event messages are logged using the following:

  • Event Log
  • Event Viewer
  • Operations Manager – Status Pane

Test Page

During an engine test, most of the testing tasks can be performed using the Test Page.

Test Procedure Scripts

nxDAS can run predefined engine test procedure scripts that instruct the operators on how to test the engine as well as automatically record data such as fullsets or logs. For more information, see Operations Manager – Acquisition Mode and Script Editor.

Timer

nxDAS has a built in Timer that operators can use during the engine test. For more information, see Operations Manager – Acquisition Mode – Timer Panel.

Data Logging

If required, the operators may also record test data independent of an engine test script.

nxDAS can log (record) the following:

  • Transient Logs
  • Fullsets
  • Critical Logs
  • Continuous Logs (if configured)
  • Test Comments

Modifying Channel Parameters

Modifying channels is local to the CVT only. The true value of the channel is always published over iDDS. For example, if the user overrides a PBS channel value, the CVT value is set, however the true channel value is published over iDDS. Likewise, if the user overrides a channel subscribed to by the RTE over iDDS, the CVT is set with the user-defined value, but the published value on the network is not changed.

During an engine test the operator has the ability to modify the following channel parameters:

  • Set/Redirect Channel Value
  • Set Channel Quality
  • Set Channel Limit

Reports

nxDAS gives the operator access to several reports and other post test applications that may be useful during an engine test. Reports are accessed through the Real-Time (RT) Reports GUI.
